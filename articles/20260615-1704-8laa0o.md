## 背景・前提  
Tailwind CSS は「クラス名をそのまま意味として使う」アプローチを採用しています。  
- **クラス名がデザインを表す**  
  例: `text-xl`, `bg-blue-500`, `p-4` など  
- **レイアウトやスタイルを分離せずに、HTML に直接書く**  
  これにより、CSS ファイルを別途作成せずに済み、スタイルの変更も即時に確認できます。  

ただし、クラス名が増えると可読性が下がることもあります。実務では「コンポーネント化」「カスタムユーティリティ」などで対策します。  
本記事ではその「対策」も含めて紹介します。

## 本文  
### 1. Tailwind のインストールと基本設定  
まずは Tailwind をプロジェクトに追加する手順です。  

```bash
# npm でインストール
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

生成される `tailwind.config.js` は以下のようになります。  

```js
module.exports = {
  content: ['./src/**/*.{html,js,jsx,ts,tsx}'],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

- `content` に Tailwind がクラスを探索するファイルを指定  
- `theme.extend` で既存のデザインシステムを拡張  

### 2. クラス名の考え方  
Tailwind のクラス名は「機能 → スタイル」の形で構成されます。  
| 機能 | スタイル | 例 |
|------|-----------|---|
| テキストサイズ | `text-` | `text-sm`, `text-lg` |
| 背景色 | `bg-` | `bg-red-500`, `bg-gray-200` |
| パディング | `p-` | `p-4`, `px-2` |
| マージン | `m-` | `m-8`, `mt-4` |
| フレックス | `flex`, `justify-`, `items-` | `flex`, `justify-center`, `items-end` |

クラス名は短く、意味が直感的に分かるように設計されています。  
> **Tip**: 変数名のように `text-` の後にサイズを付けるだけで、必要なデザインを瞬時に把握できます。

### 3. レスポンシブデザインの書き方  
Tailwind は「ブレークポイント」を先頭に付けてクラス名を書きます。  
| ブレークポイント | 表記 | 画面幅 |
|-------------------|------|--------|
| sm | `sm:` | 640px 以上 |
| md | `md:` | 768px 以上 |
| lg | `lg:` | 1024px 以上 |
| xl | `xl:` | 1280px 以上 |

#### 例：カードレイアウト  
```html
<div class="bg-white shadow rounded p-4 sm:p-6 lg:p-8">
  <h2 class="text-xl font-semibold mb-2">カードタイトル</h2>
  <p class="text-gray-600">カード本文。レスポンシブでパディングが変わります。</p>
</div>
```

- デフォルトは `p-4`、`sm` 以上で `p-6`、`lg` 以上で `p-8` に切り替わります。  
- こうした書き方で、1行のコードで複数のデバイスに対応できます。

### 4. カスタム設定の入口  
実務ではプロジェクトごとに色やフォントを統一する必要があります。  
`tailwind.config.js` の `theme.extend` を使ってカスタム設定を追加します。  

#### 例：オリジナルカラーとフォント  
```js
module.exports = {
  content: ['./src/**/*.{html,js,jsx,ts,tsx}'],
  theme: {
    extend: {
      colors: {
        primary: '#1E40AF', // 独自カラー
        secondary: '#F59E0B',
      },
      fontSize: {
        '2xl': '1.75rem',
      },
    },
  },
  plugins: [],
}
```

これでクラス名は次のように使えます。  

```html
<h1 class="text-primary font-semibold text-2xl">見出し</h1>
<p class="text-secondary">説明文</p>
```

さらに、**カスタムディレクティブ**（`@apply`）を使えば共通クラスを作成できます。  

```css
/* src/styles/global.css */
.btn-primary {
  @apply py-2 px-4 bg-primary text-white rounded hover:bg-blue-600;
}
```

```html
<button class="btn-primary">クリック</button>
```

### 5. コンポーネント化のベストプラクティス  
Tailwind で大量のユーティリティクラスを書き続けると、HTML が長くなります。  
**React** や **Vue** などのフレームワークでは、**コンポーネント単位**でクラスをまとめることで可読性を保ちます。  

#### 例：React でボタンコンポーネント  
```tsx
// Button.tsx
type Props = {
  variant?: 'primary' | 'secondary';
  children: React.ReactNode;
};

const Button = ({ variant = 'primary', children }: Props) => {
  const base = 'py-2 px-4 rounded font-medium';
  const variants = {
    primary: 'bg-primary text-white hover:bg-blue-600',
    secondary: 'bg-secondary text-black hover:bg-yellow-700',
  };
  return <button className={`${base} ${variants[variant]}`}>{children}</button>;
};

export default Button;
```

```tsx
// App.tsx
import Button from './Button';

function App() {
  return (
    <div className="p-4">
      <Button variant="primary">Primary</Button>
      <Button variant="secondary">Secondary</Button>
    </div>
  );
}
```

こうすることで、コンポーネントごとにクラスを管理でき、再利用性が高まります。

## まとめ  
- **Tailwind CSS はユーティリティファースト**で、クラス名がそのままデザインを表すため、迅速な開発が可能。  
- **レスポンシブはブレークポイントをプレフィックスで表現**し、1行で複数デバイスに対応。  
- **`tailwind.config.js` の `extend`** でプロジェクト独自の色・フォントを追加でき、`@apply` で共通クラスを定義。  
- **フレームワークと組み合わせてコンポーネント化**することで、可読性と再利用性を保ちながら開発を進められます。  

実務では「やりたいこと」を素早く形にしたい場面が多いです。Tailwind を上手く活用すれば、デザインと実装を同時に進めることができ、開発サイクルを大幅に短縮できます。ぜひプロジェクトに取り入れて、実際に体感してみてください。
