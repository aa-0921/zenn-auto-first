## はじめに：AI ツール、本当に役立つの？

こんにちは、開発者の皆さん。皆さんは最近、Claude Code や GitHub Copilot 、その他の AI コーディング支援ツールを使っていましょうか？ 特に Claude Code は、CLI 操作が可能なため「 AI に対話形式でコードを書かせる」という点で、開発現場での導入が爆速です。

しかし、気が付くと「 AI に丸投げしている」「 全然理解できていないコード」「 自分のスキルが上がらないのではないか？」という不安に繋がっちゃうことがありますよね。

本記事では、駆け出しエンジニア目線で、AI ツールを開発に取り入れる上での「 役立つコツ」を、「 使うべき場面」「 聞き方のコツ」「 自分との使い分け」の 3 つの視点から解説していきます。

## 背景：なぜ AI ツールが開発現場で話題なのか

### AI ツールの利点は？

開発現場で AI ツールが注目されるのは、単純な効率化だけではありません。以下のようなメリットがあるからです：

- **定型作業の高速化**：API 呼び出しや boilerplate の作成が数秒で完了
- **バゲット駆出しの是正**：「 こういう書き方ってどう？」と聞けば即座に改善案が返ってくる
- **学びの機会**：自分が書きたいけど、どう書けばいいのか分からないパターンを、AI がサンプルを提供してくれる

ただし、これらの利点を最大化するためには「 AI への質問の仕方」や「 どの部分にお任せするか」を考える必要があります。

### 駆け出しエンジニeer が陥りやすい pitfalls

- **丸投げ症状**：要件だけ言って、実装を全て AI にお任せ
- **黒ボックス回避**：AI が生成したコードをそのままコミット
- **進捗の否定**：AI のおかげで「 自分は何もしていない」と感じてしまう

これらを防ぐための「 ツールの使い方」を次のセクションから見ていきましょう。

## 本文：Claude Code を効果的に使う 3 つのコツ

### 1. AI への質問（プロンプト）のコツ

まず、AI ツールであるとき、「 どのように質問するか」が結果に大きく影響します。以下の 3 つのポイントに着目してください。

#### 1.1. コンテキストを与える

**悪い例**：
```
フロント-end でフォームを作って
```

**良い例**：
```
React + TypeScript のフロントエンドで、ユーザー登録フォームを作成してください。
必須バリデーション：メールアドレス、パスワード（8 文字以上）
UI ライブラリは Material-UI を使用。バリデーションエラーはフォーム下に赤文字で表示
```

#### 1.2. 制約を明示する

AI は自由な発想をしてくれる一方で、「 自分が想定していないスタックだったり、依存関係のないコード」を生成することもあります。そのため、使用技術やバージョンを明示しましょう。

```
Next.js 14 app router + Prisma + PostgreSQL で、
ユーザー一覧ページをサーバーコンポーネントとして作成してください。
Prisma は既に設定済みです。
```

#### 1.3. フィードバックループを作る

AI の出力を見て「 これ、他にもっといい書き方あるんじゃない？」と思ったら、その場で修正を依頼しましょう。

```
このコード、コンポーネント分割の観点からもう少しキレイにできそうですね。
もう少し再利用性の高い構成にリファクタリングしてください
```

### 2. AI と自分の仕事を振り分ける

次に、AI と自分、どのような役割でコードを書くかを決めましょう。以下の判断基準を参考にしてください。

| 作業の種類           | AI にお任せ                             | 自分で書く                             |
|---------------------|--------------------------------------|-------------------------------------|
| 仕様の検討           | ○（アイデア出し）                      | ×                                   |
| Boilerplate/雛形    | ○（CLI で生成、API 呼び出しスケlette）  | △（必要な部分だけ手書き）              |
| ビジネスロジック    | △（方針を聞く→自分で実装）               | ○                                   |
| テストコード        | ○（初期候補を生成）                     | △（アサーション内容は自分で考える）    |
| デバッグ            | △（ヒントを得る→自分で解決）             | ○                                   |
| コミットメント      | ×                                    | ○                                   |

#### 具体的な流れの一例

1. **要件定義**：AI に簖繕を検討してもらい、アウトプットのイメージを固める
2. **雛形生成**：CLI でディレクトリ構成や基本ファイルを.generage
3. **実装**：ビジネスロジックは自分で書き、つまずいた場所で AI にヒントを求める
4. **レビュー**：AI にコードレビューを依頼、改善点があれば都度修正

このようにすることで、「 AI の利便を享受しつつも、スキルアップを止めない」ことが可能です。

### 3. AI が生成したコードを理解する方法

多くの人が陥るのが、「 コードが動いたから次に進む」というところです。しかし、後のメンテナンスやチーム内でのコードレビュー、さらにはオリジナルな拡張をしようとしたとき、「 自分が書いたのかどうか分からない」という問題が出てきます。

#### 3.1. 出力されたコードにコメントを追加しながら読む

AI が生成したコードには、必ずしも「 なぜこの書き方？」「 なぜこのライブラリ？」という理由が書かれていません。そのため、一度自分の言葉でコメントを追記しながら読み込んでみましょう。

```javascript
// TODO: なぜこのように書いたらダメなのか？
// → このように書くと、コンポーネントの再利用性が下がる
const UserForm = ({ onSubmit }) => {
  // ...
}
```

#### 3.2. 似たようなコードを自分で書き直す

同じ機能を、AI に頼らず自分の手で書き直す練習をしましょう。これにより、「 どのように書けばいいのか」という感覚がついてくるのです。

#### 3.3. どの部分を AI が書いたのか、Git の history で確認する

Git を使っていれば、`git blame` や `git log -p` で、どの行がいつの間に書き込まれたかが分かります。この情報を元に、「 自分の責任範囲」「 AI の責任範囲」を明確にしておきましょう。

## 実践編：Claude Code で簡易 TODO アプリを開発する

ここからは、Claude Code を例に、実際の開発フローを体験していきましょう。

### ステップ 1：要件を Claude に伝える

まずは、Claude Code を起動して、簡易 TODO アプリの要件を伝えてみましょう。

```
Next.js 14 の TODO アプリを作ってください。
機能：タスクの追加/削除/完了切り替え
データ保存：ローカルストレージ
スタイリング：Tailwind CSS
```

### ステップ 2：出力されたコードを確認する

Claude が生成したコードは、ほぼ完成した React コンポーネントとして出力されます。ここで気を付けるのは、「 依存関係が正しいか」「 型定義が適切か」という点です。

### ステップ 3：自分の手で改良する

次は、自分の手でコードを改良してみましょう。たとえば、「 タスクの編集機能」を追加してみる、というのが良い練習です。

```typescript
// 自分が追加した編集機能
const [editingId, setEditingId] = useState<string | null>(null);
const [editText, setEditText] = useState("");
```

このように、AI の出力をベースにして、自分の手で「 プラス」を加えていくのが、成長のコツです。

## まとめ：AI ツールを活用するための 3 つの原則

いかがでしたか？本記事では、AI ツール（Claude Code など）を開発に取り入れるに当たって、次の 3 つの原則を心に留めて活用してください。

1. **質問は具体的かつ制約付きに**：AI は「 質問の品質」によって結果が変わります
2. **仕事の振り分けを意識する**：面倒な定型作業はお任せして、本質的なロジックは自分で書く
3. **出力されたコードは「 理解する」ことを忘れずに**：コメントを書いたり、自分で書き直したりして定着させる

AI ツールは「 開発を速くする」だけでなく、「 開発者を次のステージへ進める」力を持っています。ただし、その鍵を握るのは「 どのように使うか」です。

皆さんの開発現場で、AI ツールをどのように活用していますか？ そのノウホウを、是非とも教えて下さい！
