## Next.jsの基本とバージョンアップによる変化

Next.jsは、Vercelが開発するReactのフレームワークで、サーバーサレンダリング、静的サイト生成、APIルートなどの機能を提供します。2022年にリリースされたNext.js 13では、従来のPages Routerに加えて、新しいApp Routerが導入され、大幅な機能追加と変更が行われました。

### Pages Router vs App Router

**Pages Router**はNext.js 12以前から存在する従来のルーティング方式です。`pages`ディレクトリ内のファイル構造に基づいてルーティングが決定されます。

```
pages/
  index.js        # /
  about.js        # /about
  blog/
    index.js      # /blog
    [slug].js     # /blog/:slug
```

一方、**App Router**はNext.js 13で導入された新しいルーティング方式で、React Server Componentsをベースとしています。`app`ディレクトリ内のファイル構造に基づきます。

```
app/
  page.js         # /
  about/
    page.js       # /about
  blog/
    page.js       # /blog
    [slug]/
      page.js     # /blog/:slug
```

### 主な違い

1. **レイアウトの概念**：App Routerでは、ルートレイアウトやネストされたレイアウトを簡単に定義できます。

2. **サーバーコンポーネントがデフォルト**：App Routerでは、コンポーネントはデフォルトでサーバーコンポーネントとして扱われます。

3. **データ取得の方法**：App Routerでは、サーバーコンポーネント内で直接データ取得が可能です。

## サーバーコンポーネントとクライアントコンポーネント

Next.js 13以降の最大の変更点の一つが、サーバーコンポーネントとクライアントコンポーネントの明確な区別です。これらを正しく理解し使い分けることが、Next.js開発における最初の大きなハードルとなります。

### サーバーコンポーネント

サーバーコンポーネントは、サーバーサイドでレンダリングされるコンポーネントです。特徴は以下の通りです：

- クライアントサイドでのJavaScriptは送信されない
- `useState`、`useEffect`などのReactフックは使用できない
- 直接送信可能なデータを扱うことができる
- ルーティング時に自動的にサーバーコンポーネントとして扱われる

```jsx
// app/posts/page.js - サーバーコンポーネント
async function PostsPage() {
  // サーバーサイドでデータ取得が可能
  const posts = await fetch('https://api.example.com/posts');
  const postsData = await posts.json();
  
  return (
    <div>
      <h1>ブログ記事一覧</h1>
      <ul>
        {postsData.map(post => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  );
}
```

### クライアントコンポーネント

クライアントコンポーネントは、クライアントサイドでレンダリングされるコンポーネントです。特徴は以下の通りです：

- Reactフック（`useState`、`useEffect`など）を使用できる
- イベントハンドラや状態管理が必要な場合に使用
- クライアントサイドでのみ実行可能なコードを含めることができる
- ファイルの先頭に`"use client";`を記述する必要がある

```jsx
// app/components/Counter.jsx - クライアントコンポーネント
"use client";

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>増やす</button>
    </div>
  );
}
```

### どちらを使うべきか？

コンポーネントの種類を選択する際の基本的なガイドライン：

- **サーバーコンポーネント**：
  - データ取得が必要な場合
  - セキュリティが重要な情報を扱う場合
  - クライアントサイドでのJavaScriptを減らしたい場合

- **クライアントコンポーネント**：
  - インタラクティブなUI（フォーム、ボタンなど）
  - 状態管理が必要な場合
  - ブラウザAPI
