## はじめに：なぜTailwind CSSが注目されるのか？

Tailwind CSSは、従来型のCSSフレームワーク（BootstrapやFoundationなど）とは異なる「ユーティリティ優先型」のCSSフレームワークです。クラス名に直接スタイルを定義することで、HTMLに近い形でスタイリングが可能になり、開発速度の向上が目指せます。本記事では、実務で役立つTailwind CSSの基本コンセプトとよく使うユーティリティを解説します。

---

## クラス名の考え方：ユーティリティ優先アプローチ

### 基本的なクラス構造
Tailwindでは、クラス名が「プロパティ」＋「値」の組み合わせで構成されます。例えば：
```html
<div class="text-center text-gray-800 font-bold py-4">
  見出しテキスト
</div>
```
このクラスは以下のスタイルを適用します：
- `text-center`：中央揃え
- `text-gray-800`：グレーの文字色（#374151）
- `font-bold`：太字
- `py-4`：上下のパディング4ユニット分

### ユーティリティのチェイン化
複数のユーティリティをスペースで連結して適用可能です。レスポンシブ修飾子も同様に連結できます：
```html
<div class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
  ボタン
</div>
```

---

## レスポンシブデザイン：breakpointの書き方

### 基本的なレスポンシブ構文
Tailwindでは、スクリーン幅ごとにスタイルを切り替えるための修飾子を使用します。基本的な書き方は：
```html
<div class="text-sm md:text-base lg:text-xl">
  ラベル
</div>
```
これは以下のように解釈されます：
- スマホ（<768px）: 小文字
- タブレット（≥768px）: 基本サイズ
- パソコン（≥1024px）: 大字

### 実際の例：レスポンシブカードコンポーネント
```html
<div class="bg-white rounded-lg shadow-md overflow-hidden sm:flex">
  <img class="w-full h-48 object-cover sm:h-32" src="..." alt="...">
  <div class="p-4 sm:flex-1">
    <h2 class="text-lg font-medium sm:text-xl">タイトル</h2>
    <p class="text-gray-700 text-sm sm:text-base">内容</p>
  </div>
</div>
```

## カスタム設定：tailwind.config.jsの活用

### 基本的な設定構造
`tailwind.config.js`でデフォルトのスタイルをカスタマイズ可能です。例：
```js
module.exports = {
  theme: {
    colors: {
      primary: '#2563eb',
      secondary: '#154064',
      // カスタムカラーを追加
      custom: '#ff6b6b',
    },
    fontSize: {
      // カスタムフォントサイズを追加
      'custom-lg': '1.5rem',
    },
  },
};
```

### カスタムプロパティの利用
設定したカスタム値はCSS変数経由で利用可能です：
```html
<div class="bg-primary text-secondary">
  <!-- 青色背景と白文字 -->
</div>

<div class="text-custom-lg">
  <!-- カスタムフォントサイズ適用 -->
</div>
```

## 実務で役立つユーティリティまとめ

| カテゴリ       | よく使うユーティリティ                     | 例の使用法                          |
|----------------|--------------------------------------------|------------------------------------|
| レイアウト     | `flex`, `grid`, `flex-wrap`, `justify-between` | レスポンシブグリッドレイアウト    |
| 文字スタイル   | `text-*`, `font-*`, `opacity-*`             | テキストの色・サイズ変更          |
| 影とエフェクト | `shadow-*`, `hover:shadow-*`                | ホバーエフェクト                  |
| レスポンシブ   | `sm:`, `md:`, `lg:`                         | ブレークポイント修飾              |
| ユーティリティ連結 | `class="bg-blue-500 hover:bg-blue-700"` | 状態ごとのスタイル変更            |

## まとめ：Tailwind CSSの魅力と活用ポイント

Tailwind CSSは、以下のような利点があります：
1. **開発速度の向上**：HTML内で直接スタイルを定義できるため、CSSファイルの維持が簡単
2. **一貫性の高いデザイン**：カスタム設定でデザインシステムを構築可能
3. **パフォーマンス最適化**：デフォルトでは不要なCSSを排除
