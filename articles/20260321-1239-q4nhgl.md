# REST API とは？

REST API（代表表現アプリケーションプログラミングインターフェース）は、Webサービス間の通信に広く使われるアーキテクチャパターンです。フロントエンドとバックエンドが通信する際、HTTPプロトコルを使ってリクエストとレスポンスをやり取りします。特に、JSON形式のデータが主流のため、フロントエンド開発者にとって理解しやすく、実装しやすい特徴があります。

## REST API の基本原則

REST API は、以下の4つの原則に基づいています：
1. **クライアント-サーバーアーキテクチャ**：クライアント（フロントエンド）とサーバー（バックエンド）が独立して動作。
2. ** Stateless（状態を持たない）**：サーバーはリクエストごとに状態を保持しません。
3. **一貫性**（Uniform Interface）：リクエストとレスポンスの形式が統一されている。
4. **階層型**（Hierarchical System）：リソースをURI（Uniform Resource Identifier）で表現。

### 例：ユーザー管理API
ユーザーを管理するAPIの場合、以下のようにリソースを定義します。
- GET `/users`：全ユーザーを取得
- POST `/users`：新しいユーザーを作成
- GET `/users/1`：ID=1のユーザーを取得
- PUT `/users/1`：ID=1のユーザーを更新
- DELETE `/users/1`：ID=1のユーザーを削除

## HTTP メソッドとステータスコード

### HTTP メソッド
HTTPメソッドは、リクエストの目的を示します。主なメソッドは以下の通りです。

#### GET
- **用途**：リソースの取得
- **例**：`GET /api/users` → 全ユーザー情報を取得

#### POST
- **用途**：リソースの作成
- **例**：`POST /api/users` → 新規ユーザーを作成（リクエストボディにデータを送信）

#### PUT/PATCH
- **PUT**：リソースの完全更新
- **PATCH**：リソースの部分更新
- **例**：`PUT /api/users/1` → ID=1のユーザー全体を更新

#### DELETE
- **用途**：リソースの削除
- **例**：`DELETE /api/users/1` → ID=1のユーザーを削除

### ステータスコード
レスポンスの成功・失敗を数値で表現します。

- **2xx**：成功（例：200 OK）
- **4xx**：クライアントエラー（例：404 Not Found）
- **5xx**：サーバーエラー（例：500 Internal Server Error）

#### 具体的な例
```javascript
// 成功例：200 OK
fetch('https://api.example.com/users')
  .then(response => {
    if (response.status === 200) {
      return response.json();
    }
  })
  .catch(error => console.error(error));

// 失敗例：404 Not Found
fetch('https://api.example.com/users/999')
  .then(response => {
    if (response.status === 404) {
      console.log('ユーザーが見つかりません');
    }
  });
```

## JSON のやり取り

APIは通常、JSON形式でデータを送受信します。JSONは構造化されたデータをテキストで表現できるため、JavaScriptとの互換性が高いです。

### JSON の例
```json
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com"
}
```

### JavaScriptでのパース
```javascript
// レスポンスからJSONをパース
fetch('https://api.example.com/users/1')
  .then(response => response.json())
  .then(data => console.log(data));
```

## fetch と axios での呼び出し

### fetch（ブラウザ内蔵API）
`fetch`はブラウザに内蔵されたAPIで、シンプルな呼び出しが可能です。

#### GETリクエスト
```javascript
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(users => {
    // users配列を処理
  })
  .catch(error => console.error('エラー:', error));
```

#### POSTリクエスト
```javascript
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Alice', email: 'alice@example.com' })
})
  .then(response => response.json())
  .then(newUser => console.log('新規作成:', newUser));
```

### axios（ライブラリ）
`axios`は依存関係が必要ですが、より柔軟な機能を提供します。

#### GETリクエスト
```javascript
axios.get('https://api.example.com/users')
  .then(response => {
    console.log('ユーザー一覧:', response.data);
  })
  .catch(error => console.error('エラー:', error));
```

#### POSTリクエスト
```javascript
axios.post('https://api.example.com/users', {
  name: 'Bob',
  email: 'bob@example.com'
})
  .then(response => {
    console.log('新規作成:', response.data);
  })
  .catch(error => console.error('エラー:', error));
```

## まとめ

REST API は、フロントエンドとバックエンドの通信を効率的に行うための標準的な方法です。HTTPメソッドとステータスコードを正しく理解し、JSON形式でデータをやり取りするスキルは、フロントエンド開発者にとって必須です。`fetch`と`axios`のどちらを使うかは、プロジェクトの要件や個人の好みに応じて選択します。実際の開発では、エラーハンドリングや認証処理も含めて、頑健なAPI呼び出しロジックを実装することが重要です。
