## 前提条件とサンプルデータ

まず、以下の2つのテーブルを使用して解説します：

```sql
-- 顧客テーブル
CREATE TABLE customers (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT,
    city VARCHAR(50),
    created_at DATE
);

-- 注文テーブル
CREATE TABLE orders (
    id INT PRIMARY KEY,
    customer_id INT,
    product_name VARCHAR(100),
    amount INT,
    order_date DATE
);
```

サンプルデータ：
```sql
INSERT INTO customers VALUES
```

## SELECT文とWHERE句の基本

### SELECT文の基本構文

```sql
SELECT 列名1, 列名2 FROM テーブル名;
```

すべての列を取得する場合は：
```sql
SELECT * FROM customers;
```

結果：
```
id | name    | age | city  | created_at
1  | 田中太郎  | 28  | 東京  | 2023-01-15
2  | 鈴木花子 | 34  | 大阪  | 2023-02-20
3  | 佐藤健太 | 22  | 東京  | 2023-03-10
```

### WHERE句による条件絞り

特定の条件に一致するデータのみ取得できます：

```sql
-- 年齢が28歳以上の顧客
SELECT name, age FROM customers WHERE age >= 28;

-- 東京在住の顧客
SELECT * FROM customers WHERE city = '東京';
```

複合条件の使用：
```sql
-- 東京在住かつ年齢25以上
SELECT * FROM customers WHERE city = '東京' AND age >= 25;
```

## JOINの基礎と応用

### INNER JOINの基本

関連する2つのテーブルを結合する最も基本的な方法：

```sql
SELECT 
    c.name AS 顧客名,
    o.product_name AS 商品名,
    o.amount AS 数量
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id;
```

結果：
```
顧客名   | 商品名        | 数量
田中太郎  | ノートパソコン | 1
田中太郎  | ワイヤレスマウス | 2
鈴木花子 | モニター      | 1
佐藤健太 | キーボード    | 3
```

### LEFT JOINの活用

左テーブルにすべてのレコードを含めたい場合：

```sql
-- 注文のない顧客も含める
SELECT 
    c.name,
    COUNT(o.id) AS 注文数
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id
GROUP BY c.id, c.name;
```

## 集計関数とGROUP BY

### 基本的な集計関数

```sql
-- 総顧客数
SELECT COUNT(*) FROM customers;

-- 年齢の平均値
SELECT AVG(age) FROM customers;

-- 最高年齢
SELECT MAX(age) FROM customers;
```

### GROUP BYを使った集計

同じ値を持つ行をグループ化して集計：

```sql
-- 都市別の顧客数
SELECT city, COUNT(*) AS 顧客数
FROM customers
GROUP BY city;

-- 都市別の平均年齢
SELECT city, AVG(age) AS 平均年齢
FROM customers
GROUP BY city;
```

### HAVING句で集計結果をフィルタリング

GROUP BYの結果に対して条件を指定：

```sql
-- 顧客数が2人以上の都市
SELECT city, COUNT(*) AS 顧客数
FROM customers
GROUP BY city
HAVING COUNT(*) >= 2;
```

## サブクエリの活用術

サブクエリ（副問い合わせ）は、クエリの中に別のクエリをネストさせる技術です。

### 基本的なサブクエリ

```sql
-- 平均年齢以上の顧客
SELECT name, age FROM customers
WHERE age > (SELECT AVG(age) FROM customers);
```

### IN句でサブクエリを使用

```sql
-- 注文をした顧客のみ表示
SELECT name FROM customers
WHERE id IN (SELECT DISTINCT customer_id FROM orders);
```

### 相関サブクエリ

外部の行に依存するサブクエリ：

```sql
-- 各顧客の注文総額を表示
SELECT 
    c.name,
    (SELECT SUM(amount) FROM orders WHERE customer_id = c.id) AS 注文総額
FROM customers c;
```

## 実務でよく使うパターン

### 1. ランキング表示

```sql
-- 注文総額でランキング
SELECT 
    c.name,
    SUM(o.amount) AS 総注文額,
    RANK() OVER (ORDER BY SUM(o.amount) DESC) AS ランク
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id
GROUP BY c.id, c.name
ORDER BY 総注文額 DESC;
```

### 2. 最新のデータ取得

```sql
-- 各顧客の最新注文を表示
SELECT DISTINCT
    c.name,
    FIRST_VALUE(o.product_name) OVER (
        PARTITION BY o.customer_id 
        ORDER BY o.order_date DESC
    ) AS 最新商品
FROM customers c
JOIN orders o ON c.id = o.customer_id;
```

### 3. 条件付き集計

```sql
-- 男性と女性の平均年齢を別〜に表示（仮想の性別カラムを追加）
SELECT 
    SUM(CASE WHEN age >= 30 THEN 1 ELSE 0 END) AS 30代以上人数,
    SUM(CASE WHEN age < 30 THEN 1 ELSE 0 END) AS 30代未満人数
FROM customers;
```

## まとめ

この記事ではSQLの基本から実務でよく使用するパターンまで解説しました：

- **SELECT/WHERE**: データの取得と条件絞り
- **JOIN**: 複数テーブルの結合方法
- **集計関数**: COUNT、SUM、AVGなどの基本集計
- **GROUP BY/HAVING**: グループごとの集計とフィルタリング
- **サブクエリ**: 複雑な条件や計算のための問い合わせの入れ子

これらのテクニックを組み合わせることで、実務のほぼすべてのシナリオをカバーできます。定期的に練習を続け、実際のプロジェクトでの経験を積むことがSQLマスターへの近道です。
