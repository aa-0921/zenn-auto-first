## 1. REST APIの概念とHTTPの基本
REST APIは、HTTPメソッド（GET, POST, PUT, DELETE など）を利用してリソースを表現し、ステータスコードで処理結果を示すインターフェースです。サーバー側ではURI（例: /users/123）でリソースを特定し、クライアントはこのURIに対して適切なメソッドとペイロードを送信します。重要なのは「リソースの表現」と「無状態（stateless）」という設計原則です。これにより、キャッシュやロードバランサが扱いやすくなります。

## 2. HTTPメソッドとステータスコードの意味
| メソッド | 主な用途 | 例 |
|----------|----------|----|
| GET      | リソースの取得 | ユーザーリスト取得 |
| POST     | リソースの作成 | 新規ユーザー作成 |
| PUT      | リソースの置換 | 既存ユーザー情報更新 |
| DELETE   | リソースの削除 | ユーザー削除 |

ステータスコードは2xx（成功）、4xx（クライアントエラー）、5xx（サーバーエラー）に分かれ、具体的な番号（200, 201, 400, 404, 500 など）で詳細を伝えます。フロントエンド側はこのコードを元にエラーメッセージや再試行ロジックを実装します。

## 3. JSONデータのやり取り
REST APIは通常、リクエストとレスポンスのボディにJSONを使用します。リクエスト例：

```json
{
  "name": "Alice",
  "email": "alice@example.com"
}
```

レスポンス例（201 Created）:

```json
{
  "id": 42,
  "name": "Alice",
  "email": "alice@example.com",
  "createdAt": "2025-11-02T10:00:00Z"
}
```

サーバーは受け取ったJSONをパースし、ビジネスロジックに応じて適切なJSONResponseを返します。

## 4. フロントエンドでAPIを呼び出す例（fetch）
```javascript
async function fetchUser(id) {
  const response = await fetch(`https://api.example.com/users/${id}`, {
    method: 'GET',
    headers: {
      'Accept': 'application/json',
    },
  });

  if (!response.ok) {
    // ステータスが200未満または400以上の場合はエラーとして処理
    const errorBody = await response.text();
    throw new Error(`HTTP ${response.status}: ${errorBody}`);
  }

  const data = await response.json();
  return data; // { id, name, email, ... }
}

// 使用例
fetchUser(123)
  .then(user => console.log('取得成功:', user))
  .catch(err => console.error('取得失敗:', err));
```

ポイントは `response.ok` でステータスコードが成功系か判定し、`await response.json()` でJSONパースを行うことです。

## 5. フロントエンドでAPIを呼び出す例（axios）
axiosは Promise を自動で処理し、エラーハンドリングが簡潔です。

```javascript
import axios from 'axios';

async function fetchUserAxios(id) {
  try {
    const response = await axios.get(`https://api.example.com/users/${id}`, {
      headers: { Accept: 'application/json' },
    });
    // ステータスコードは自動でチェックされ、失敗すると例外が投げられます
    return response.data;
  } catch (error) {
    // エラーオブジェクトには response.status が含まれる
    if (error.response) {
      throw new Error(`API Error ${error.response.status}: ${error.response.data.message}`);
    } else if (error.request) {
      throw new Error('ネットワークエラーまたはタイムアウト');
    } else // 設定エラー等
      throw new Error('リクエスト設定エラー');
  }
}

// 使用例
fetchUserAxios(123)
  .then(user => console.log('取得成功:', user))
  .catch(err => console.error('取得失敗:', err));
```

axios では `response.data` にサーバーからの JSON が直接格納され、ステータスコードのチェックは内部で行われます。

## 6. エラーハンドリングとローディング状態の実装
実務では、ユーザー操作に対して「読み込み中」「エラー発生」などのステートをUIに反映させます。React 例：

```jsx
function UserDetail({ userId }) {
  const [user, setUser] = React.useState(null);
  const [loading, setLoading] = React.useState(true);
  const [error, setError] = React.useState(null);

  React.useEffect(() => {
    const abortController = new AbortController();
    async function load() {
      try {
        const res = await fetch(`https://api.example.com/users/${userId}`, {
          signal: abortController.signal,
        });
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const data = await res.json();
        setUser(data);
      } catch (e) {
        if (e.name !== 'AbortError') setError(e.message);
      } finally {
        setLoading(false);
      }
    }
    load();
    return () => abortController.abort();
  }, [userId]);

  if (loading) return <p>読み込み中...</p>;
  if (error)   return <p>エラー: {error}</p>;
  return <pre>{JSON.stringify(user, null, 2)}</pre>;
}
```

このように、ステータスコードやネットワークエラーを適切に捕捉し、ローディングフラグでユーザー体験を向上させます。
