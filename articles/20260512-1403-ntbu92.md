## Git の基本コマンド概要

Git の操作は大きく「リポジトリの取得」「変更の記録」「共有」の3つのフェーズに分けられます。まずはリポジトリをローカルに取得する「clone」から始め、以降の作業はローカルブランチで開発し、適切なタイミングでリモートへ反映します。このサイクルを回すことで、コードの整合性と履歴の可視性が保たれます。

## 代表的コマンドの使い方

以下に、実務で最も頻繁に使用するコマンドとその具体例を示します。

```bash
# リポジトリのクローン
git clone https://github.com/user/project.git
```

```bash
# 作業ブランチの作成と切り替えgit checkout -b feature/login
```

```bash
# 変更内容のステージングとコミット
git add .
git commit -m "Add user authentication"
```

```bash
# ローカルブランチをリモートにプッシュ
git push origin feature/login
```

```bash
# リモート変更を取得（ローカルに反映）
git pull origin main
```

これらの手順を繰り返すことで、開発フローが「開発 → コミット → 共有」の循環に自然に組み込まれます。

## コンフリクト解決の実例複数人が同じファイルを編集した場合、Git は自動的にマージできない箇所を「コンフリクト」として警告します。以下は実際のコンフリクト解決手順です。

```bash
# コンフリクトが起きたブランチへ切り替え
git checkout feature/login
```

```bash# コンフリクトファイルを確認
git status
```

```bash
# 手動で衝突部分を修正した後、ステージング
git add conflicted_file.txt
```

```bash
# コンフリクト解消のコミット
git commit -m "Resolve merge conflict in conflicted_file.txt"
```

コンフリクト解消時は、必ず変更内容を確認し、意図しない削除がないかチェックすることが重要です。また、マージ前に「git fetch」で最新のリモート状態を取得しておくと、予期せぬコンフリクトを減らすことができます。

## まとめ

Git の基本コマンドは「clone」「branch」「commit」「push」「pull」の5つに集約できます。これらをマスターし、コンフリクト時の対処法を理解すれば、現場での協働がスムーズに進みます。日常的に「git status」や「git log」で状態を確認し、適切なタイミングでブランチを切り替える習慣を身につけることが、品質の高いコードbase維持につながります。ぜひ実務での開発フローに組み込み、実践的なスキルへと昇華させてください。
