## 背景・前提：なぜキャリアチェンジを決意したのか

### 1. 仕事の「壁」に直面した

前職は営業職。ノルマ達成に追われる日々の中で、**「自分の手で価値を創造したい」**という漠然とした思いが芽生えました。特に、顧客の要望を受けて社内でシステム改善を提案したときに、自分がコードを書ければもっと迅速に解決できると実感したのが転機です。

### 2. 市場の需要をリサーチ

転職サイトやエンジニア向けのコミュニティで「**未経験からでも採用されやすい言語**」を調査。結果、**JavaScript（特にフロントエンド）** と **Python** が案件数・年収ともに安定していることが分かり、どちらかに絞ることにしました。

### 3. 時間的制約と学習リソース

フルタイムで働きながらの学習は必至。1 日あたり 1〜2 時間、週末は 4〜6 時間確保できると見積もり、**「アウトプット中心の学習」**を方針にしました。

## 学習の進め方：実践的かつ継続可能なステップ

### 1. 基礎固めは教材 1 つに絞る

- **教材例**: Udemy の「The Complete JavaScript Course」＋公式 MDN ドキュメント  
- **理由**: 複数教材を併用すると情報が散らばり、復習が困難になるため。1 本の教材を完走させることで、概念の一貫性が保たれます。

```bash
# 学習計画の例（1 週間単位）
Week 1-2: JavaScript 基本文法（変数・関数・配列・オブジェクト）
Week 3-4: DOM 操作とイベントハンドラ
Week 5-6: 非同期処理（Promise, async/await）
Week 7-8: 小規模 SPA（React 入門）実装
```

### 2. 手を動かす「ミニプロジェクト」を週 1 回実施

学んだ概念は必ずコードに落とし込む。以下のような **「スモールスケール」** の課題を設定しました。

| 週 | プロジェクト例 | 主な学習ポイント |
|---|---|---|
| 1 | TODO リスト（ローカルストレージ） | 基本的な CRUD とローカルストレージ |
| 2 | 天気取得アプリ（OpenWeather API） | fetch と非同期処理 |
| 3 | ポートフォリオサイト（React + Vite） | コンポーネント設計とビルドツール |
| 4 | ブログエディタ（Markdown エディタ） | カスタムフックと状態管理 |

#### コード例：fetch で天気情報を取得

```javascript
async function getWeather(city) {
  const apiKey = "YOUR_API_KEY";
  const url = `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${apiKey}&units=metric`;

  try {
    const response = await fetch(url);
    if (!response.ok) throw new Error("Network response was not ok");
    const data = await response.json();
    console.log(`${city} の気温は ${data.main.temp}°C`);
  } catch (error) {
    console.error("Fetch error:", error);
  }
}
```

### 3. アウトプットは「ブログ」と「GitHub」

- **ブログ**: 学んだことを日本語でまとめる。検索エンジンからの流入が期待でき、復習にもなる。  
- **GitHub**: コードは必ずリポジトリにプッシュ。README に実装概要・学習ポイント・課題を書き、**ポートフォリオ** として活用。

### 4. コミュニティ参加でモチベーション維持

- **オンライン**: Slack のエンジニアコミュニティ、Discord の学習サーバー  
- **オフライン**: 勉強会（Meetup）やハッカソンに月 1 回参加  
- **効果**: 疑問点の即時解決、最新技術情報の取得、同年代の転職者との情報交換が可能。

## 転職活動の実践：書類選考から内定までの流れ

### 1. レジュメと職務経歴書の作り方

| 項目 | ポイント |
|---|---|
| ヘッドライン | 「未経験から 2 年でフロントエンドエンジニアに転職」などインパクトを持たせる |
| スキルセット | JavaScript, React, HTML/CSS, Git, CI/CD（学習中） |
| プロジェクト実績 | 上記ミニプロジェクトを 1 行で要約し、GitHub リンク添付 |
| 業務経験 | 前職の営業で培った **要件定義・顧客折衝** スキルを「エンジニア視点での要件整理」として強調 |

### 2. 書類選考突破のためのテクニック

- **キーワードマッチ**: 求人票に記載された技術キーワード（例: React, REST API）を必ずレジュメに入れる。  
- **数字で示す成果**: 前職で「売上 20% 増」など具体的数字を入れると、採用担当者の目に留まりやすい。  
- **カバーレターは必須**: 「未経験だけど、〇〇 の経験が活かせる」点を 3 行以内で簡潔に。

### 3. 面接対策：テクニカルとヒューマンの両面

| フェーズ | 対策内容 |
|---|---|
| コーディングテスト | LeetCode の Easy/Medium を 1 日 1 問解く。タイムアウト対策として「紙にアルゴリズムを書いてから実装」 |
| 技術面接 | 自作プロジェクトの設計思想を説明できるように準備。特に「なぜ React を選んだか」「状態管理はどうしたか」 |
| 行動面接 | 前職の営業経験で「顧客の要望をどう聞き出したか」「失敗から何を学んだか」など、エンジニアに転換できるエピソードを用意 |

### 4. 内定後の準備：スムーズなオンボーディングのために

1. **事前学習**: 入社先が使用しているフレームワーク（例: Next.js）の公式チュートリアルを 1 週間で終える。  
2. **ツールセットアップ**: VS Code の拡張機能、Docker、GitHub Actions など、社内で推奨される開発環境をローカルに構築。  
3. **メンター／ペアプログラミング**: 入社直後にメンターを決め、週 1 回のコードレビューを依頼。

## 現場で実感した「手応え」：成長を数字で測る

| 項目 | 具体例 |
|---|---|
| **コードレビューの通過率** | 入社 1 カ月でレビュー通過率 85%（業界平均 70%） |
| **タスク完了速度** | スプリント 2 週間で 5 件の UI 改修タスクを完遂 |
| **自動化スクリプト作成** | 手動で行っていたデプロイ手順を GitHub Actions で自動化、工数 30% 削減 |
| **顧客折衝** | 前職の営業経験を活かし、要件定義ミーティングで非機能要件（パフォーマンス・アクセシビリティ）を提案、採用された |

> **ポイント**  
> - **「未経験＝即戦力ではない」** が前提でも、**「学習速度」** と **「業務への適応力」** を可視化すれば、上司や同僚からの評価は上がります。  
> - 自分で **KPI**（例：コードレビュー通過率、タスク完了率）を設定し、定期的に振り返る習慣が成長を実感させてくれます。

## まとめ：30 歳からエンジニア転職を成功させるためのチェックリスト

1. **目的を明確化** → 「何を作りたいか」「どんな価値を提供したいか」を言語化。  
2. **学習計画を 1 本の教材に絞る** → 体系的に学び、アウトプットで定着。  
3. **ミニプロジェクトで実装経験を積む** → GitHub に公開し、ポートフォリオ化。  
4. **ブログやコミュニティでアウトプット** → モチベーション維持と情報共有。  
5. **レジュメは数字とキーワードで差別化** → 書類選考通過率が上がる。  
6. **面接は技術＋前職の経験を掛け合わせてアピール** → 「未経験」でも「即戦力感」を演出。  
7. **入社前にツールとフレームワークを予習** → オンボーディングがスムーズ。  
8. **KPI を設定し、定量的に成長を可視化** → 自己評価と上司評価が一致しやすくなる。

30 歳を超えてからのキャリアチェンジは決して遅くありません。**「学び続ける姿勢」** と **「実務に近い形でのアウトプット」** を軸にすれば、未経験でもエンジニアとしての第一歩を確実に踏み出せます。ぜひ本記事のステップを参考に、あなた自身の転職ストーリーを描いてみてください。
