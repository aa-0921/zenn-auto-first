## 1. 非同期処理の背景と実務上の課題
Web アプリケーションやサーバーサイドのコードでは、ネットワークリクエスト、ファイル読み込み、タイマー待機など非同期操作が不可欠です。これらをコールバックだけで扱うと、複数のステップがネストされ「コールバック地獄」と呼ばれる可読性の低いコードになりやすくなります。たとえば、次のようなコードはファイルを読み込み、JSON に変換し、API に送信する流れをコールバックで実装しています。

```javascript
fs.readFile('data.txt', 'utf8', (err, data) => {
  if (err) return handleError(err);
  const json = JSON.parse(data);
  fetch('https://api.example.com/submit', {
    method: 'POST',
    body: JSON.stringify(json)
  })
  .then(res => res.json())
  .then(result => console.log('Success', result))
  .catch(err => console.error('Error', err));
});
```

上記のコードは 3 層のネストが続いており、エラーハンドリングも各ステップに散らばっているため、保守性が低くバグが発生しやすいです。実務ではこのような構造を避け、Promise または async/await を使うことが一般的です。

## 2. Promise を用いた改善例と then の使い分け
Promise は非同期操作の結果をオブジェクトとして扱い、`.then` と `.catch` で成功・失敗をチェーンできます。上記の例を Promise に置き換えると次のようになります。

```javascript
fs.promises.readFile('data.txt', 'utf8')
  .then(data => {
    const json = JSON.parse(data);
    return fetch('https://api.example.com/submit', {
      method: 'POST',
      body: JSON.stringify(json)
    });
  })
  .then(response => response.json())
  .then(result => console.log('Success', result))
  .catch(err => console.error('Error', err));
```

ポイントは、`then` では戻り値に Promise を返すことで次のステップを待たせることです。これにより、コードは線形的に読めるようになり、エラーハンドリングは `.catch` の一箇所で完結できます。

## 3. async/await のメリットとエラーハンドリング
`async`/`await` は Promise を基礎にした構文糖で、非同期処理を同期的に見た目で書けます。`async` が付いた関数は常に Promise を返すため、`await` で結果を取得できます。エラーハンドリングは通常の例外処理と同様に `try`/`catch` を使用します。

```javascript
async function submitData() {
  try {
    const data = await fs.promises.readFile('data.txt', 'utf8');
    const json = JSON.parse(data);
    const response = await fetch('https://api.example.com/submit', {
      method: 'POST',
      body: JSON.stringify(json)
    });
    const result = await response.json();
    console.log('Success', result);
  } catch (err) {
    console.error('Error', err);
  }
}
submitData();
```

`try`/`catch` による一元的なエラーキャッチにより、コードはすっきりし、例外がどこで発生したかを明確に把握しやすくなります。

## 4. まとめ
- コールバック地獄は可読性と保守性を低下させるため、Promise または async/await に置き換えることが推奨です。  
- Promise はチェーンで処理を組み立て、`.then`/`.catch` で成功と失敗を分離でき、再利用性が高いです。  
- async/await はコードの可読性とエラーハンドリングの簡潔さが特長で、複数ステップを同期的に記述できます。  
- エラーハンドリングは `catch`（Promise）または `try`/`catch`（async/await）を使い、必ず中央で捕捉してロギングやリトライ処理を行うようにしましょう。  

このように、Promise と async/await を適切に使い分けることで、非同期処理の可読性と保守性を大幅に向上させ、実務で即座に役立つコードを書くことができます。
