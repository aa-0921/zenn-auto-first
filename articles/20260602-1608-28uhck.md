## 1. ルーティングエラー：URLが正しくマッピングされない  
### 問題の例  
- 新しく作成したリソースのルートが認識されない  
- 同じパスが複数のコントローラーで定義されてしまう  
- エンドポイントが存在せず、404エラーが発生  

### 具体的なデバッグ方法  
#### コマンド1: ルートの確認  
```bash
rails routes
```  
このコマンドで、すべてのルートが一覧表示されます。例えば、ブログ投稿のルートが定義されているか確認します。  

#### コマンド2: エンドポイントのテスト  
```bash
rails test test_that_should_respond_to_post
```  
RSpecやMinitestを使って、エンドポイントが正しく認識されるかテストします。  

#### 例：URLが不正な場合の修正  
```ruby
# config/routes.rb の修正例  
resources :articles, except: :show do
  member do
    get :preview  # カスタムアクションの定義
  end
end
```

## 2. マイグレーションエラー：データベース操作が失敗する  
### 問題の例  
- 同一名前の列を再定義しようとした場合のエラー  
- インデックスが存在しないとエラーになる操作  
- シーケンスのリセットが必要なカラムを変更  

### 具体的なデバッグ方法  
#### コマンド1: 現在のスキーマ確認  
```bash
rails db:schema:dump
```  
スキーマファイルを確認し、実際に存在するカラムやインデックスをチェックします。  

#### コマンド2: 遅延マイグレーションの実行  
```bash
rails db:migrate:down VERSION=20230901123400
rails db:migrate:up VERSION=20230901123400
```  
特定のマイグレーションを巻き戻し、再実行することで問題を再現・修正します。  

#### 例：列の名前変更時のエラー  
```ruby
# 間違ったマイグレーション  
def change
  remove_column :articles, :title, :string
  add_column :articles, :name, :string
end

# 正しい修正方法  
def change
  rename_column :articles, :title, :name
end
```

## 3. N+1 クエリ問題：パフォーマンス低下の根源  
### 問題の例  
- 10件の記事を取得する際、10回の追加クエリが発行される  
- ネストされたデータ（コメントやアタッチメント）の取得が非効率  

### 具体的なデバッグ方法  
#### コマンド1: クエリログの監視  
```bash
rails console
> Article.all
```  
コンソールから発行されたSQLログを確認し、N+1クエリを特定します。  

#### コマンド2: `bullet` gemの利用  
Gemfileに追加：  
```ruby
gem 'bullet'
```  
開発環境でN+1クエリを自動検出します。実行時に警告メッセージが表示されます。  

#### 例：クエリの最適化  
```ruby
# N+1クエリの発生コード  
@articles = Article.all
@articles.each do |article|
  puts article.comments.count  # N+1発生
end

# 最適化後のコード  
@articles = Article.includes(:comments)
@articles.each do |article|
  puts article.comments.size  # 1回のクエリで取得
end
```

## まとめ：Rails のデバッグをプロに昇華する  
Rails は慣習に基づくフレームワークですが、その柔軟性が初心者を混乱させることもあります。  
- **ルーティング**：`rails routes` とテストで確認  
- **マイグレーション**：スキーマダンプとバージョン管理で検証  
- **N+1クエリ**：コンソールログと `bullet` で最適化  

これらの技術を習得することで、開発速度が飛躍的に向上します。特に「エラーが発生したらまずプロンプトで確認」という基本姿勢を忘れず、実務で役立つツールを日常的に使いこなすことがゴールです。
