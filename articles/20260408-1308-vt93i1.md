## REST APIの基本原則
REST APIは、リソースをリソース名（URL）で表現し、HTTPメソッドで操作する仕組みです。主なメソッドは以下の通りです：
- `GET`：リソースを取得
- `POST`：新規リソースの作成
- `PUT`：既存リソースの更新
- `DELETE`：リソースの削除

たとえば、ユーザー管理APIでは、以下のように使用されます：
- `/users` に `GET`で全ユーザーを取得
- `/users` に `POST`で新しいユーザーを登録
- `/users/1` に `PUT`でユーザー1の情報を更新
- `/users/1` に `DELETE`でユーザー1を削除

## HTTPステータスコードの理解
API呼び出し後のレスポンスには、ステータスコードが含まれます。重要なコードは以下の通りです：
- `200 OK`：成功（通常の応答）
- `201 Created`：リソースの新規作成成功
- `404 Not Found`：リソースが見つからない
- `500 Internal Server Error`：サーバーサイドのエラー

フロントエンドでは、これらのコードを確認してエラーハンドリングを行います。例えば、`404`の場合は「データが存在しない」とユーザーに通知します。

## JSON形式のデータやり取り
REST APIでは、データは多くの場合JSON形式で送受信されます。JSONはテキストベースの構造化データ表現言語で、以下のように見えます：
```json
{
  "id": 1,
  "name": "山田太郎",
  "email": "taro@example.com"
}
```
- リクエストボディにデータを送る場合はJSONを文字列として渡す
- レスポンスでデータを受け取る場合はJSONをパース（例: `response.json()`）

## フロントエンドからのAPI呼び出し：fetchの使い方
ブラウザ上でAPIを呼び出すには、ブラウザ内蔵の`fetch` APIを使います。以下は`GET`リクエストの例です：
```javascript
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    if (!response.ok) throw new Error('APIエラー');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```
- `fetch`はPromiseを返すため、`async/await`で非同期処理を扱います
- `response.json()`でJSONをJavaScriptオブジェクトに変換
- エラー処理は`try/catch`で実装

## フロントエンドからのAPI呼び出し：axiosの使い方
`axios`は`fetch`より使いやすいライブラリです。インストール後、以下のように呼び出します：
```javascript
import axios from 'axios';

async function createUser(userData) {
  try {
    const response = await axios.post('https://api.example.com/users', userData);
    return response.data;
  } catch (error) {
    throw new Error(`APIエラー: ${error.message}`);
  }
}
```
- `axios.post()`で`POST`リクエストを送信
- リクエストボディは2番目の引数にJSONオブジェクトを渡す
- エラー処理は`try/catch`で統一可能

## まとめ
REST APIは、フロントエンドとバックエンドの通信を標準化した仕組みです。HTTPメソッド（GET/POSTなど）とステータスコード（200/404など）を理解し、JSON形式でデータをやり取ることで、フロントエンドからAPIを呼び出すことが可能です。`fetch`はブラウザ内蔵のシンプルな方法、`axios`は柔軟なライブラリとして選択肢があります。実務では、エラーハンドリングやリクエストのリトライ処理も考慮することが重要です。
