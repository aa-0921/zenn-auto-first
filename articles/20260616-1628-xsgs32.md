## 背景・前提
- **学習の目的**：業務で即戦力になることだけでなく、 fundamentals（基礎）を固め、将来的に自分で問題を解決できる力を身につけること。  
- **時間の確保**：フルタイムの業務がある中で、1日30分〜1時間の学習時間を確保する必要があった。  
- **情報源**：公式ドキュメント、技術ブログ、Qiita、GitHubのREADMEなど、多様な情報源を横断する必要があった。  

## 学習ステップ
### 1. インプットの質を高める
1. **公式ドキュメントを隅々まで読む**  
   - キーワード検索だけでなく、目次から体系的に読む。     - 用語が出てきたらメモを取り、後で検索できるようにする。  
2. **技術ブログの「実装例」だけではなく「背景」まで読む**  
   - なぜその技術が選ばれたのか、代替案は何かを意識すると理解が深まる。  
3. **毎日1つはアウトプットのタスクを設定**  
   - 読んだ内容をブログ記事にまとめる、コードを書き直す、質問を1つ作るなど。

### 2. アウトプットの具体的な方法
- **写経**：公式リポジトリやチュートリアルのコードをそのまま打ち込す。  
  - 1行ずつ手を動かしながら、コメントを自分の言葉で書き換える。  
- **個人開発**：小さな機能（例：REST APIの簡易クローラ）を1週間で完走させる。  
  - 完成後すぐにコードレビューを他者に依頼し、フィードバックを得る。  
- **ブログ・Qiitaへの投稿**：学んだことを1000字程度でまとめ、タグを付けて投稿。  
  - 読者からのコメントが新たな疑問を生み、次のインプットにつながる。

### 3. 具体的なコード例以下は、Pythonで「APIからデータ取得 → 加工 → ローカルに保存」の流れを写経しながら実装した例です。  

```python
import requests
import json
from datetime import datetime

def fetch_data(api_url: str) -> dict:
    """指定したAPIからJSONデータを取得する"""
    response = requests.get(api_url)
    response.raise_for_status()
    return response.json()

def transform(data: dict) -> list:
    """データを必要な形に整形する"""
    result = []
    for item in data.get('items', []):
        result.append({
            'id': item['id'],
            'title': item['title'],
            'created_at': datetime.fromtimestamp(item['created_at']).isoformat()
        })
    return result

def save_to_file(data: list, filepath: str) -> None:
    """整形したデータをJSONファイルに保存する"""
    with open(filepath, 'w', encoding='utf-8') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)

if __name__ == '__main__':
    API_URL = 'https://api.example.com/v1/items'
    FILE_PATH = f'output_{datetime.now().strftime("%Y%m%d")}.json'

    raw = fetch_data(API_URL)
    processed = transform(raw)
    save_to_file(processed, FILE_PATH)

    print(f'データ取得・加工・保存が完了しました: {FILE_PATH}')
```

このコードを写経した後、次のステップとして「エラーハンドリングを追加する」「単体テストを書く」など、自分で拡張機能を付け加えました。  
このように、**写経 → 拡張 → テスト**のサイクルを回すことで、実務で即活かせるスキルが身につきます。

## まとめ
- **インプットとアウトプットを同等に重視**：読むだけでなく、すぐに手を動かすことが上達への近道です。  
- **ドキュメントは「背景」まで読む**：キーワード検索だけでなく、目的や設計思想まで把握すると深い理解が得られます。  
- **写経は「コメント書き換え」**：自分の言葉で解説しながら進めると、知識が定着しやすくなります。  - **個人開発は小さく始め、フィードバックを取り入れる**：完成後はコードレビューやブログ投稿でアウトプットを完成させましょう。  

これらの実践を通じて、駆け出しエンジニアでも「学習 → 実装 → 振り返り」の循環を回せるようになり、業務でも自信を持って新しい技術に挑戦できるようになります。
