## SELECT文の基本構文

SELECT文はデータを取得するための基本的なクエリです。最もシンプルな形は次のようになります。

```sql
SELECT 列名1, 列名2, ...
FROM テーブル名;
```

例として、employeesテーブルから全ての従業員の名前と部署を取得するクエリを見てみましょう。

```sql
SELECT name, department
FROM employees;
```

すべての列を取得したい場合はアスタリスクを使用しますが、パフォーマンスの観点から特定の列を指定することを推奨します。

```sql
SELECT *
FROM employees;
```

## WHERE句による条件絞り込み

WHERE句を使用することで、取得するデータに対して条件を設定できます。比較演算子や論理演算子を組み合わせて複雑な条件も表現可能です。

```sql
SELECT name, salary, department
FROM employees
WHERE salary > 500000
  AND department = 'エンジニアリング';
```

文字列の検索にはLIKE演算子を使用します。パーセント記号(%)は任意の文字列にマッチし、アンダースコア(_)は任意の一文字にマッチします。

```sql
SELECT name, email
FROM employees
WHERE name LIKE '田%';
```

日付の範囲検索も頻繁に使用されます。

```sql
SELECT *
FROM orders
WHERE order_date BETWEEN '2024-01-01' AND '2024-12-31';
```

## JOINによるテーブル結合

実務では複数のテーブルを結合してデータを取得することがほとんどです。JOINにはいくつかの種類があります。

### INNER JOIN（内部結合）

INNER JOINは両方のテーブルに存在するレコードのみを返します。

```sql
SELECT e.name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;
```

### LEFT JOIN（左外部結合）

LEFT JOINは左側のテーブルを基準にし、右側のテーブルにマッチするレコードがない場合でも左側のデータを返します。

```sql
SELECT e.name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;
```

### RIGHT JOIN（右外部結合）

RIGHT JOINは右側のテーブルを基準にします。

```sql
SELECT e.name, d.department_name
FROM employees e
RIGHT JOIN departments d ON e.department_id = d.id;
```

## 集計関数の活用

実務ではデータの集計が頻繁に必要になります。COUNT, SUM, AVG, MAX, MINなどの集計関数を覚えておきましょう。

```sql
SELECT department, COUNT(*) as employee_count, AVG(salary) as avg_salary
FROM employees
GROUP BY department;
```

HAVING句を使用することで、集計結果に対して条件を設定できます。

```sql
SELECT department, COUNT(*) as employee_count
FROM employees
GROUP BY department
HAVING COUNT(*) >= 5;
```

並び順の指定にはORDER BYを使用します。

```sql
SELECT department, AVG(salary) as avg_salary
FROM employees
GROUP BY department
ORDER BY avg_salary DESC;
```

## サブクエリの実践的活用

サブクエリはクエリの中に別のクエリを埋め込むことができる機能です。特に実務では次のパターンがよく使用されます。

### 相関サブクエリ

あるテーブルの各レコードに対して条件を適用する場合に有効です。

```sql
SELECT e.name, e.salary
FROM employees e
WHERE e.salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department = e.department
);
```

### EXISTS演算子

サブクエリが存在するかどうかを判定するために使用します。

```sql
SELECT name
FROM departments d
WHERE EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.department_id = d.id
);
```

### サブクエリを使用したINSERT

サブクエリの結果を別のテーブルに挿入するパターンです。

```sql
INSERT INTO department_stats (department_id, employee_count)
SELECT department_id, COUNT(*)
FROM employees
GROUP BY department_id;
```

## 実務でよく使うクエリパターン

### 最新レコードの取得

特定の条件に対して最新のレコードを取得するパターンです。

```sql
SELECT o.*
FROM orders o
WHERE o.order_date = (
    SELECT MAX(order_date)
    FROM orders o2
    WHERE o2.customer_id = o.customer_id
);
```

### ランキングの作成

ウィンドウ関数を使用してランキングを作成します。

```sql
SELECT 
    name,
    salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as rank
FROM employees;
```

### 重複データの除外

DISTINCTを使用して重複を除外します。

```sql
SELECT DISTINCT department
FROM employees;
```

## まとめ

本記事では、SQLの基本的な構文から実務で頻繁に使用するクエリパターンまでを解説しました。SELECT、WHERE、JOINの基本を理解した上で、集計関数やサブクエリを適切に組み合わせることで、より複雑なデータ処理も可能になります。これらのパターンを日常の業務で意識的に使用することで、SQLスキルを着実に向上させることができるでしょう。
