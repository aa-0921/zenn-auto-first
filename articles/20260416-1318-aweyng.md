## Git の基本概念

Git を理解する上で押さえておくべき基本的な概念を紹介します。

### リポジトリ

リポジトリとは、ファイルの変更履歴を管理する場所です。Git を使うと、プロジェクトの各ファイルの変更履歴を追跡し、いつでも特定のバージョンに戻すことができます。

### コミット

コミットとは、ファイルの変更をリポジトリに保存する操作です。コミットを行う際には、変更内容を説明するメッセージを添える必要があります。コミットメッセージは、後で変更内容を理解するために重要です。

### ブランチ

ブランチは、開発の流れを分岐させるための機能です。例えば、メインの開発ブランチ（通常は main や master）から機能開発用のブランチを分岐させ、開発が完了したらメインブランチにマージする、という流れが一般的です。

## 現場でよく使う Git コマンド

### リポジトリのクローン

リモートリポジトリをローカルにコピーするには `clone` コマンドを使用します。

```bash
git clone https://github.com/user/repository.git
```

このコマンドを実行すると、指定したリポジトリが現在のディレクトリにダウンロードされます。

### 変更の取得

リモートリポジトリの最新の変更をローカルに取得するには `pull` コマンドを使用します。

```bash
git pull origin main
```

このコマンドは、リモートリポジトリ（origin）の main ブランチの最新変更を取得し、現在のブランチにマージします。

### ブランチ操作

新しいブランチを作成するには `branch` コマンドを使用します。

```bash
git branch feature/new-feature
```

作成したブランチに切り替えるには `checkout` または `switch` コマンドを使用します。

```bash
# 古い方法
git checkout feature/new-feature

# 新しい方法（Git 2.23以降）
git switch feature/new-feature
```

### 変更のコミット

ファイルの変更をコミットするには、まず `add` コマンドでステージングエリアに追加し、その後 `commit` コマンドでコミットします。

```bash
git add file1.js file2.css
git commit -m "Add new features and fix bugs"
```

`add` コマンドの後に `.` を指定すると、変更されたすべてのファイルをステージングエリアに追加できます。

```bash
git add .
```

### 変更のプッシュ

ローカルでのコミットをリモートリポジトリに反映させるには `push` コマンドを使用します。

```bash
git push origin feature/new-feature
```

このコマンドは、現在のブランチ（feature/new-feature）のコミットをリモートリポジトリ（origin）にプッシュします。

## コンフリクトの対処方法

複数人で開発していると、同じファイルを編集してコンフリクト（競合）が発生することがあります。コンフリクトが発生した場合の対処方法を解説します。

### コンフリクトの検出

コンフリクトが発生した場合、`pull` や `merge` コマンドは失敗し、以下のようなメッセージが表示されます。

```
error: could not apply xxxxxxx... Commit message
hint: Resolve all conflicts manually, mark them as resolved with
hint: "git add/rm <conflicted files>", then run "git commit".
```

### コンフリクトの解決

1. コンフリクトが発生したファイルを開き、Git が挿入したマーカー（`<<<<<<<`, `=======`, `>>>>>>>`）を確認します。
2. どちらの変更を残すか、または両方を組み合わせて修正します。
3. 変更が完了したら、ファイルをステージングエリアに追加します。

```bash
git add conflicted-file.js
```

4. コミットしてコンフリクトを解決を完了します。

```bash
git commit -m "Resolve conflict in conflicted-file.js"
```

## まとめ

本記事では、Git の基本概念と現場でよく使うコマンドを解説しました。特に重要なのは、`clone` → `pull` → `branch` → `commit` → `push` という基本的なワークフローです。また、コンフリクトが発生した場合の対処方法も理解しておきましょう。

Git は初めは複雑に感じるかもしれませんが、日常的に使ううちに自然と身についてきます。自分の開発環境で実際にコマンドを叩いて試してみることが、最も効果的な学習方法です。コードを安全に管理し、チームでの開発を円滑に進めるために、Git の基本的な操作をマスターしましょう。
