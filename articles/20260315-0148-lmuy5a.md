## はじめに
現代の Web アプリケーションでは、API 呼び出しやファイル読み込み、タイマーなど、非同期処理が頻繁に必要となります。従来はコールバック関数をネストさせて実装する「コールバック地獄」が発生しやすく、可読性や保守性が低下していました。この記事では、Promise と async/await を使ってコールバック地獄から脱却する方法、そしてエラーハンドリングの基本を具体的なコード例を交えて解説します。対象は JavaScript の基礎を理解している駆け出し〜中級エンジニアです。

## 背景・前提
- **コールバック地獄とは**  
  複数の非同期処理を順番に実行しようとすると、コールバックがインデントを深くし、ネストが深くなる現象です。例として、3 回の API 呼び出しを順に行うコードを見てみましょう。

```javascript
// コールバック地獄の例getUser(userId, (user) => {
  getOrders(user.id, (orders) => {
    getOrderDetails(orders[0].id, (details) => {
      console.log(details);
    }, handleError);
  }, handleError);
}, handleError);
```

- **なぜ Promise が登場したのか**  
  Promise は「最終的に成功か失敗かの結果を表すオブジェクト」で、`.then()` メソッドをチェーンできるため、ネストを平坦にできます。さらに、`catch()` でエラーを一元的にハンドリングできます。

- **async/await の役割**  
  ES2017 で導入された構文で、Promise をまるで同期コードのように書けるようになります。`async` 関数内では `await` 演算子で Promise の解決を待ち、エラーは `try/catch` で捕捉できます。

## Promise の基本
### Promise の作成
`new Promise(executor)` でインスタンスを生成します。`executor` は `resolve` と `reject` の二つの関数を受け取ります。

```javascript
function fetchData(url) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', url);
    xhr.onload = () => {
      if (xhr.status === 200) {
        resolve(JSON.parse(xhr.responseText));
      } else {
        reject(new Error(`Request failed: ${xhr.status}`));
      }
    };
    xhr.onerror = () => reject(new Error('Network error'));
    xhr.send();
  });
}
```

### チェーンによる連続処理
`.then()` は前の Promise の結果を受け取り、新しい Promise を返します。これによってフラットな記述が可能です。

```javascript
fetchData('/api/user/123')
  .then(user => {
    console.log('User:', user);
    return fetchData(`/api/orders/${user.id}`);
  })
  .then(orders => {
    console.log('Orders:', orders);
    return fetchData(`/api/order/${orders[0].id}`);
  })
  .then(details => {
    console.log('Order details:', details);
  })
  .catch(err => {
    console.error('Something went wrong:', err);
  });
```

### Promise.all と Promise.race
複数の非同期処理を並行して実行し、全て完了するまで待つか、最初に完了したものだけを取得したい場合に便利です。

```javascript// 全て完了するまで待つ
Promise.all([
  fetchData('/api/user/123'),
  fetchData('/api/settings')
]).then(([user, settings]) => {
  console.log('User and settings ready');
});

// 最初に完了したものだけを取得Promise.race([
  fetchData('/api/fast'),
  fetchData('/api/slow')
]).then(result => {
  console.log('Fastest response:', result);
});
```

## async/await の使い方
### 基本構文
`async` キーワードを関数に付けると、その関数は常に Promise を返します。内部で `await` を使うと、Promise が解決されるまで処理を一時停止します。

```javascript
async function fetchUserAndOrders() {
  try {
    const user = await fetchData('/api/user/123');
    console.log('User:', user);
    const orders = await fetchData(`/api/orders/${user.id}`);
    console.log('Orders:', orders);
    const details = await fetchData(`/api/order/${orders[0].id}`);
    console.log('Order details:', details);
    return details;
  } catch (err) {
    console.error('Error occurred:', err);
    throw err; // 必要に応じて再スロー
  }
}

fetchUserAndOrders().then(() => console.log('All done'));
```

### 並列実行と await の組み合わせ`await` は直列に待つため、並列に実行したい場合は `Promise.all` と組み合わせます。

```javascript
async function fetchAll() {
  try {
    const [user, settings] = await Promise.all([
      fetchData('/api/user/123'),
      fetchData('/api/settings')
    ]);
    console.log('User:', user);
    console.log('Settings:', settings);
  } catch (err) {
    console.error('Fetch failed:', err);
  }
}
```

### トップレベル await（ES2022 以降）
モジュールレベルで直接 `await` を使えるようになり、初期化ロジックをシンプルに書けます。

```javascript
// init.js
const config = await fetchData('/api/config');
export { config };
```

## エラーハンドリングの基本
### Promise のエラーハンドリング
- `.catch()` はチェーンのどこかで発生したエラーを捕捉します。チェーンの中間で `.catch()` を入れると、そこから先のチェーンはエラーが発生しても続きません（エラーがハンドリングされたとみなされるため）。
- エラーを再スローしたい場合は、`.catch()` 内で `throw err;` とします。

```javascript
fetchData('/api/user/123')
  .then(user => {
    if (!user.active) throw new Error('User inactive');
    return fetchData(`/api/orders/${user.id}`);
  })
  .catch(err => {
    console.warn('軽微なエラー:', err);
    // デフォルト値で復帰
    return fetchData('/api/guest/orders');
  })
  .then(orders => {
    console.log('Orders:', orders);
  });
```

### async/await のエラーハンドリング
`try / catch` ブロックで捕捉します。複数の `await` がある場合は、いずれかでエラーが発生するとすぐに `catch` に飛びます。

```javascript
async function safeFetch() {
  try {
    const user = await fetchData('/api/user/123');
    const orders = await fetchData(`/api/orders/${user.id}`);
    return { user, orders };
  } catch (err) {
    // ログ収集やユーザー通知など、ここで一元的に処理
    console.error('Operation failed:', err);
    throw err; // 呼び出し側にも伝える場合
  }
}
```

### エラーハンドリングのベストプラクティス
1. **エラーの種類を区別する**  
   ネットワークエラー、バリデーションエラー、予期しない例外など、`instanceof` や独自のエラークラスで分類すると、適切な復旧処理が書きやすくなります。
2. **ユーザーへのフィードバック**  
   UI 層ではエラーメッセージを表示し、開発者向けにはコンソールにスタックトレースを残すなど、目的別に出し分けます。
3. **リトライロジック**  
   一時的なネットワークエラーに対しては、指数バックオフを用いたリトライ関数をラッパーとして用意すると安定性が向上します。

```javascript
async function fetchWithRetry(url, retries = 3, delay = 500) {
  for (let i = 0; i < retries; i++) {
    try {
      return await fetchData(url);
    } catch (err) {
      if (i === retries - 1) throw err;
      await new Promise(res => setTimeout(res, delay * 2 ** i));
    }
  }
}
```

## まとめ- コールバック地獄はネストが深くなり可読性を損ねるため、Promise でチェーン化、`async/await` で同期風に書くことで解消できます。
- Promise は `.then()`／`.catch()` でフラットにエラーを伝播させ、`async/await` は `try / catch` で一元的にハンドリングできます。
- エラーハンドリングでは、エラーの種類を区別し、ユーザーへのフィードバックと開発者向けログを分けることが重要です。リトライやフォールバックロジックを組み込むことで、実務でも堅牢な非同期処理を実現できます。
- これらの概念を踏まえて、実際のプロジェクトでは API ラッパーやユーティリティ関数として共通化し、チーム全体で一貫した非同期処理の書き方を確立すると保守性が向上します。
