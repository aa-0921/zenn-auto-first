## はじめに

Visual Studio Code（以下 VS Code）は、軽量さと拡張性の高さから多くのエンジニアに選ばれています。

本記事では、駆け出し〜中級エンジニアが「すぐに」仕事に活かせるよう、以下の3点に絞って解説します。

1. **開発効率が劇的に上がるおすすめ拡張機能**  
2. **作業を高速化するキーバインドと設定例**  
3. **チームや別デバイスで設定・拡張機能を共有する方法**

実際に手を動かしながら設定できるよう、コードスニペットや設定ファイル例も掲載しています。ぜひ、この記事を読んだらすぐに `settings.json` と `keybindings.json` を書き換えてみてください。

## 1. 開発効率が上がるおすすめ拡張機能

### 1.1 基本的なコード補完系

| 拡張機能 | 主な機能 | インストールコマンド |
|----------|----------|-----------------------|
| **ESLint** | JavaScript/TypeScript の静的解析・自動修正 | `ext install dbaeumer.vscode-eslint` |
| **Prettier - Code formatter** | コード整形（保存時自動） | `ext install esbenp.prettier-vscode` |
| **TabNine** | AI 補完（ローカルモデル） | `ext install TabNine.tabnine-vscode` |

> **ポイント**  
> - `ESLint` と `Prettier` を併用すると、**コード品質**と**統一感**が同時に担保できます。  
> - `TabNine` は大型プロジェクトで変数名を思い出す手間を削減します。

### 1.2 プロジェクト構造・ナビゲーション系

| 拡張機能 | 主な機能 | インストールコマンド |
|----------|----------|-----------------------|
| **Project Manager** | プロジェクトの切り替えをサイドバーに一覧化 | `ext install alefragnani.project-manager` |
| **Path Intellisense** | ファイルパスの自動補完 | `ext install christian-kohler.path-intellisense` |
| **Bookmarks** | 任意行にブックマークを付けて素早くジャンプ | `ext install alefragnani.Bookmarks` |

> **実践例**  
> ```json
> // settings.json に追加
> "projectManager.git.baseFolders": [
>   "~/workspaces",
>   "~/personal"
> ],
> "path-intellisense.mappings": {
>   "@components": "${workspaceRoot}/src/components"
> }
> ```

### 1.3 デバッグ・テスト系

| 拡張機能 | 主な機能 | インストールコマンド |
|----------|----------|-----------------------|
| **Debugger for Chrome** | Chrome デバッガー（フロントエンド） | `ext install msjsdiag.debugger-for-chrome` |
| **Jest** | Jest テストの実行・結果表示 | `ext install Orta.vscode-jest` |
| **REST Client** | `.http` ファイルで API 呼び出しテスト | `ext install humao.rest-client` |

> **Tips**  
> `REST Client` は Postman の代わりに軽量で使えるので、API 開発の初期段階で重宝します。

### 1.4 UI/UX 改善系

| 拡張機能 | 主な機能 | インストールコマンド |
|----------|----------|-----------------------|
| **Bracket Pair Colorizer 2** | 対応括弧を色分け | `ext install CoenraadS.bracket-pair-colorizer-2` |
| **Indent Rainbow** | インデントレベルをカラー表示 | `ext install oderwat.indent-rainbow` |
| **GitLens — Git supercharged** | Git 履歴・ブレーム情報をインライン表示 | `ext install eamodio.gitlens` |

> **効果**  
> コードの構造が視覚的に把握しやすくなるため、**バグの早期発見**につながります。

## 2. キーバインドと設定で作業を高速化

### 2.1 よく使うショートカットのカスタマイズ例

VS Code のデフォルトキーバインドは多くの開発者にとって「慣れ」ですが、**自分の操作感に合わせる**ことで手の移動が減ります。  
以下は JavaScript/TypeScript 開発者が好むカスタム例です。

```json
// keybindings.json
[
  {
    "key": "ctrl+shift+f",
    "command": "workbench.action.findInFiles",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+e",
    "command": "workbench.action.quickOpen",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+alt+o",
    "command": "editor.action.organizeImports",
    "when": "editorTextFocus && editorLangId == 'typescript'"
  },
  {
    "key": "ctrl+shift+r",
    "command": "workbench.action.reloadWindow"
  },
  {
    "key": "ctrl+alt+b",
    "command": "bookmarks.toggle",
    "when": "editorTextFocus"
  }
]
```

- `Ctrl+Alt+O` で **インポートの自動整理**（`organizeImports`）  
- `Ctrl+Alt+B` で **Bookmarks** のトグル  
- `Ctrl+Shift+R` はウィンドウリロード。拡張機能の設定変更後に便利です。

> **設定の反映**  
> `F1` → 「Preferences: Open Keyboard Shortcuts (JSON)」で上記 JSON を貼り付けます。

### 2.2 設定ファイルのベストプラクティス

`settings.json` は **プロジェクト単位** と **ユーザー単位** に分けて管理すると便利です。  
プロジェクトルートに `.vscode/settings.json` を置くと、チーム全員が同じ設定を共有できます。

#### 2.2.1 ユーザー全体の設定例

```json
// ~/.config/Code/User/settings.json (Mac/Linux) or %APPDATA%/Code/User/settings.json (Windows)
{
  "editor.tabSize": 2,
  "editor.formatOnSave": true,
  "files.autoSave": "onFocusChange",
  "editor.minimap.enabled": false,
  "eslint.validate": ["javascript", "typescript"],
  "prettier.singleQuote": true,
  "prettier.trailingComma": "all",
  "git.enableSmartCommit": true,
  "extensions.ignoreRecommendations": false,
  "workbench.startupEditor": "newUntitledFile",
  "workbench.colorTheme": "One Dark Pro",
  "workbench.iconTheme": "vscode-icons"
}
```

- `formatOnSave` と `eslint.validate` の組み合わせで **保存時自動整形 & Lint** が実現します。  
- `editor.minimap.enabled` をオフにすると、エディタが広く使えて **視認性が向上**。

#### 2.2.2 プロジェクト固有の設定例

```json
// .vscode/settings.json
{
  "typescript.tsdk": "node_modules/typescript/lib",
  "eslint.workingDirectories": [{ "mode": "auto" }],
  "jest.enableInlineErrorMessages": true,
  "rest-client.environmentVariables": {
    "$shared": {
      "BASE_URL": "https://api.example.com"
    },
    "development": {
      "API_TOKEN": "dev-token-xxxx"
    },
    "production": {
      "API_TOKEN": "prod-token-yyyy"
    }
  }
}
```

- `typescript.tsdk` でローカルの TypeScript バージョンを使用し、**バージョン不整合**を防ぎます。  
- `rest-client` の環境変数は `.env` を意識せずに `.http` ファイルで切り替え可能です。

### 2.3「設定のインポート/エクスポート」自動化

複数の PC やチームメンバーに同じ環境を配布したい場合、**Settings Sync** 拡張機能が便利です。

1. 拡張機能をインストール  
   `ext install Shan.code-settings-sync`
2. `Shift+Alt+U` で **Upload**（GitHub Gist に保存）  
3. 新しいマシンで `Shift+Alt+D` で **Download**  

> **注意点**  
> - Gist に公開するかプライベートかはプロジェクトのポリシーに合わせて設定してください。  
> - `settings.json`・`keybindings.json`・拡張機能リスト（`extensions.json`）がすべて同期されます。

## 3. 設定・拡張機能の共有方法

### 3.1 `.vscode/extensions.json` で推奨拡張機能を宣言

プロジェクトルートに `extensions.json` を置くと、VS Code が **「推奨拡張機能」** の通知を出します。

```json
// .vscode/extensions.json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "tabnine.tabnine-vscode",
    "eamodio.gitlens",
    "humao.rest-client"
  ],
  "unwantedRecommendations": [
    "some.unwanted.extension"
  ]
}
```

- 新しいメンバーがリポジトリをクローンしただけで、**推奨拡張のインストール** が促されます。  

### 3.2 `devcontainer` で環境をコード化

Docker と VS Code Remote Container を併用すれば、**IDE の設定まで含めた開発環境** をコードで管理できます。

#### 3.2.1 `devcontainer.json` の例

```json
// .devcontainer/devcontainer.json
{
  "name": "Node.js + TypeScript",
  "image": "mcr.microsoft.com/vscode/devcontainers/typescript-node:0-14",
  "extensions": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "TabNine.tabnine-vscode",
    "eamodio.gitlens",
    "humao.rest-client"
  ],
  "settings": {
    "editor.formatOnSave": true,
    "eslint.validate": ["javascript", "typescript"]
  },
  "postCreateCommand": "npm install"
}
```

- コンテナ起動時に自動で拡張機能と設定が適用されるので、**環境差分が減ります**。  

### 3.3 CI で拡張機能のバージョンを固定

CI パイプラインで `code --list-extensions` の出力を `extensions.txt` に保存し、ビルド時に同一バージョンをインストールできます。

```bash
# 開発マシンで生成
code --list-extensions > extensions.txt

# CI でインストール
cat extensions.txt | xargs -L 1 code --install-extension
```

- これにより、ローカルと CI の **拡張機能バージョン不一致** が防げます。

## まとめ

- **拡張機能**は「コード補完」「ナビゲーション」「デバッグ」「UI改善」の4カテゴリに分けて選定すると、導入コストが低く効果が高い。  
- **キーバインド**は頻繁に使う操作（インポート整理、ブックマーク、検索）を自分だけのショートカットに置き換えるだけで、1日数分の時間削減が期待できる。  
- **設定・拡張機能の共有**は `settings.json`・`keybindings.json`・`extensions.json` をリポジトリにコミットし、`Settings Sync` や `devcontainer` と組み合わせると、チーム全体で統一された開発環境を簡単に構築できる。  

まずは **「おすすめ拡張 5 つ」** と **「キーバインド 5 件」** を試し、`settings.json` に上記ベース設定を貼り付けてみましょう。  
慣れたらプロジェクトごとの `.vscode` ディレクトリを整備し、**チーム全員で同じ開発体験**を共有してください。

> **次のステップ**  
> 1. 本記事の設定をローカルに反映 → `Settings Sync` でバックアップ  
> 2. プロジェクトに `.vscode/extensions.json` と `.vscode/settings.json` を追加  
> 3. `devcontainer` を導入して、環境のコード化を体験  

開発効率が上がると、コードを書く時間が増えるだけでなく、**コードの品質やチームの生産性**も自然に向上します。ぜひ実践して、快適な VS Code ライフを手に入れましょう！
