## 前提知識と課題

### AI ツールを導入する理由

* ** boilerplate コードの生成**: プロジェクトのひな型やユーティリティ関数を即座に作成できます。
* ** 学習の加速**: 新しいライブラリの使い方を AI に解説してもらうことで、学習コストを削減できます。
* ** コードレビューの効率化**: AI によるコードの提案を活用して、より迅速なフィードバックを得られます。

### 落とし穴

* **プロンプトの過信**: 曖昧な指示で作成したコードはバグを含んでいる場合があり、そのまま使うと後の修正コストが増加します。
* **主体性の喪失**: AI に作業を任せきりにすると、開発者としてのスキルが退化してしまう可能性があります。
* **依存関係の混乱**: 生成されたコードの出典やライセンスを明確にしない場合、プロジェクトの健全性が損なわれることがあります。

## Claude Code を使った実践フロー

以下は、私が日常的に実践しているシンプルなワークフローです。

### 1️⃣ 問題の明確化とプロンプトの設計

Claude Code に任せる前に、**明確で具体的な要件** を定義することが重要です。以下のチェックリストを参考に、必要な部分を記入します。

* **目的** – 何を実現したいのか？
* **入力** – どのようなデータや設定が必要か？
* **出力** – どのようなファイルや動作が期待されるか？
* **制約** – パフォーマンスやスタイル、依存関係に関する制限は何か？

```markdown
# プロンプト例: Next.js のページ用 SEO メタタグを生成する

役割: Next.js の開発者アシスタント

タスク: ページタイトル（最大 60 文字）、メタ説明（最大 160 文字）、Open Graph 画像の URL を生成する。

入力:
- ページタイトル: "TypeScript でビルドする"
- カテゴリ: "開発者向け"
- ドメイン: "https://example.com"

出力:
- title: "TypeScript でビルドする - 開発者向け"
- description: "TypeScript を使ったプロジェクトのビルド方法を解説する開発者向け記事です。"
- ogImageUrl: "https://example.com/og/typescript-build.png"
```

### 2️⃣ コード生成とレビュー

明確なプロンプトを作成したら、Claude Code に実行させます。生成されたコードは、**まずスタブとして扱う** ことを忘れないでください。

```javascript
// 生成されたサンプルコード
export const generateSEO = ({ title, category, domain }) => {
  const seo = {
    title: `${title} - ${category}`,
    description: `${title} についての開発者向け記事です。`,
    ogImageUrl: `${domain}/og/${title.toLowerCase().replace(' ', '-')}.png`,
  };
  return seo;
};
```

次に、以下の観点から **手動レビュー** を行います。

* **コードの正しさ** – ロジックは意図通りに動作するか？
* **命名規則** – プロジェクトのスタイルガイドに準拠しているか？
* **エッジケース** – 入力が欠けている場合や特殊な文字の場合も考慮されているか？

問題が見つかった場合は、修正を加えてから再度 AI にフィードバックすることで、より洗練された出力が得られます。

### 3️⃣ 自分でコードを書く部分 vs AI に任せる部分

| 自分で書く | AI に任せる |
|----------|------------|
| **コアアルゴリズム** – ビジネスロジックやパフォーマンスが重要な部分。 | **Boilerplate** – ルーターの設定、ユーティリティ関数のひな型、CI/CD パイプラインの骨格。 |
| **エッジケースの処理** – 異常時の動作は慎重に設計する必要がある。 | **ドキュメントの草案** – API の概要やフォルダの説明を AI に作成してもらう。 |
| **テストケースの設計** – コードの動作を保証する。 | **コードの要約** – PR やコミットメッセージの説明文を生成する。 |

この区別を明確にすることで、AI の生産性を活かしつつ、エンジニアとしてのスキルを維持できます。

## バランスを取るためのヒント

1. **生成されたコードを「下書き」として扱う**
   一度生成したコードはすぐにコピーせず、まず自分のエディタで確認します。必要に応じて修正を加え、修正点はメモしておくと、次回のプロンプト作成に役立ちます。

2. **プロンプトの調整を習慣化する**
   生成されたコードが期待通りでなかった場合、プロンプトを修正して再実行します。このプロセスを繰り返すことで、AI との「対話」スキルが向上します。

3. **バージョン管理で出典を明確にする**
   AI が生成したファイルには、先頭に簡単なコメントを追加しましょう（例: `# Generated with Claude Code – 2025-...`）ことで、チーム内でのトレーサビリティが確保されます。

4. **定期的な「コード書き出し」セッションを実施する**
   週に数回、意図的に AI を使わずにコードを書く時間を設けます。これにより、スキルレベルの低下を防ぎ、新しい技術の理解を深めることができます。

5. **チーム内で使い方を共有する**
   AI ツールの活用方法はチームによって異なります。自分なりの「ベストプラクティス」をメモし、チーム内で知見を共有することで、より健全な開発文化を築くことができます。

## まとめ

AI ツールは開発効率を飛躍的に向上させますが、それらを盲目的に信頼するのは危険です。**明確なプロンプト設計**、**生成されたコードの慎重なレビュー**、そして**自分でコードを書く部分と AI に任せる部分の明確な区別**が、開発者としての成長を維持しながら AI を活用する鍵となります。このバランスを実践することで、より生産的で質の高い開発プロセスを構築できるでしょう。
