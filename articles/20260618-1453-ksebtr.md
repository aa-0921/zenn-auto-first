## はじめに

## REST API とは何か

REST（Representational State Transfer）とは、Web システムの設計原則の一つです。簡単に言うと、「HTTP プロトコルの特性を最大限に活用して、リソース（データ）を操作する設計手法」のことです。

REST API の最大の特徴は、**「URL（エンドポイント）」**と**「HTTP メソッド」**を組み合わせて、何を行いたいかを明確に表現することにあります。

### リソースの考え方
REST では、操作対象となるデータを「リソース」として捉えます。例えば、「ユーザー情報」というリソースを操作したい場合、URL は `/users` のようになります。

- `GET /users` → ユーザー一覧を取得する
- `GET /users/123` → ID 123 のユーザー詳細を取得する
- `POST /users` → 新しいユーザーを作成する

このように、URL に動詞（`/getUsers` や `/createUser`）を含めず、名詞（`/users`）で表現するのが RESTful な設計の基本です。

## HTTP メソッドとステータスコード

API 通信では、「何をしたいか」を伝える **HTTP メソッド** と、「結果どうなったか」を伝える **ステータスコード** の組み合わせでやり取りが行われます。

### よく使われる HTTP メソッド

| メソッド | 役割 | 例 | 特徴 |
| :--- | :--- | :--- | :--- |
| **GET** | 取得 | `/users` | サーバーからデータを取得する。副作用がない（データを書き換えない）。 |
| **POST** | 作成 | `/users` | 新しいリソースを作成する。リクエストボディにデータを乗せて送信する。 |
| **PUT** | 更新（置換） | `/users/123` | リソースを丸ごと更新する。 |
| **PATCH** | 更新（一部） | `/users/123` | リソースの一部だけを書き換える。 |
| **DELETE** | 削除 | `/users/123` | リソースを削除する。 |

### 理解しておくべきステータスコード

サーバーからのレスポンスに含まれる 3 桁の数字（ステータスコード）は、通信の結果を意味します。

- **2xx (成功)**
    - `200 OK`: リクエストが成功し、期待したレスポンスが返ってきた。
    - `201 Created`: リソースの作成に成功した（主に POST 後に返る）。
- **4xx (クライアントエラー)**
    - `400 Bad Request`: リクエストの内容が不正（バリデーションエラーなど）。
    - `401 Unauthorized`: 認証が必要なリクエストに認証情報がない。
    - `403 Forbidden`: 権限がないためアクセスが禁止されている。
    - `404 Not Found`: 指定した URL のリソースが見つからない。
- **5xx (サーバーエラー)**
    - `500 Internal Server Error`: サーバー側で予期せぬエラーが発生した。

## JSON によるデータのやり取り

REST API では、データの形式として **JSON (JavaScript Object Notation)** が標準的に使われます。JSON は軽量で、JavaScript のオブジェクト形式に似ているため、フロントエンドで扱いやすいのがメリットです。

### リクエストボディの例
POST や PUT でデータを送信する場合、以下のような JSON 形式でデータを送ります。

```json
{
  "name": "田中 太郎",
  "email": "tanaka@example.com",
  "age": 25
}
```

### レスポンスボディの例
サーバーからは、処理結果とともに以下のような JSON が返ってきます。

```json
{
  "id": 123,
  "name": "田中 太郎",
  "email": "tanaka@example.com",
  "createdAt": "2023-10-01T10:00:00Z"
}
```

## フロントエンドから API を叩く実装例

実際に JavaScript を使って API 通信を行う方法を解説します。現代のフロントエンド開発では、標準の `fetch` API か、ライブラリの `axios` を使うのが一般的です。

### 1. fetch API を使う方法
`fetch` はブラウザ標準の機能であるため、ライブラリのインストールなしで利用できます。

```javascript
async function getUserData() {
  try {
    const response = await fetch('https://api.example.com/users/123');
    
    // response.ok はステータスコードが 200-299 の時に true になる
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    const data = await response.json(); // レスポンスを JSON として解析
    console.log('ユーザーデータ:', data);
  } catch (error) {
    console.error('通信エラーが発生しました:', error);
  }
}

// データの作成（POST）の例
async function createUser(userData) {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(userData), // JSオブジェクトをJSON文字列に変換
  });
  const result = await response.json();
  return result;
}
```

### 2. axios を使う方法
`axios` は、`fetch` よりも高機能で、コードを簡潔に書ける人気のライブラリです。

```bash
# インストール
npm install axios
```

```javascript
import axios from 'axios';

async function getUserData() {
  try {
    // axios はデフォルトで JSON として解析してくれるため .json() が不要
    const response = await axios.get('https://api.example.com/users/123');
    console.log('ユーザーデータ:', response.data);
  } catch (error) {
    // 4xx, 5xx エラーが自動的に catch ブロックに飛ぶため、エラーハンドリングが楽
    console.error('エラーが発生しました:', error.response?.status);
  }
}

async function createUser(userData) {
  try {
    // 第2引数にデータを渡すだけで自動的に JSON 形式で送信される
    const response = await axios.post('https://api.example.com/users', userData);
    return response.data;
  } catch (error) {
    console.error(error);
  }
}
```

### fetch と axios の違いは？
- **fetch**: 標準機能なので軽量。ただし、HTTP エラー（404 や 500）でも `catch` に飛ばないため、`response.ok` でチェックする必要がある。
- **axios**: インストールが必要だが、タイムアウト設定、インターセプター（リクエスト・レスポンスの共通処理）、自動的な JSON 変換など、実務で便利な機能が豊富。

## まとめ

REST API の基本は、**「適切な URL」** に対して **「適切な HTTP メソッド」** を使い、**「JSON 形式」** でデータのやり取りを行うことです。

- **GET**: 取得 / **POST**: 作成 / **PUT・PATCH**: 更新 / **DELETE**: 削除
- ステータスコードを見て、成功したか（2xx）、クライアント側のミスか（4xx）、サーバー側のミスか（5xx）を判断する。
- 実装には `fetch` または `axios` を使い、非同期処理（`async/await`）で適切にハンドリングする。

これらの基本を抑えておけば、どのような API 仕様書を読んでもスムーズに実装に入れるはずです。まずは簡単な公開 API（JSONPlaceholder など）を使って、実際に GET や POST を試してみることをおすすめします。
