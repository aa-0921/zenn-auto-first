## 1. データの取得と絞り込み：SELECT と WHERE の基本

まずは全ての基本となる、データの取得と条件指定について見ていきましょう。

以下の2つのテーブルを例に説明します。

**`users` テーブル（ユーザー情報）**
| id | name | email | created_at |
|---|---|---|---|
| 1 | Tanaka | tanaka@example.com | 2023-01-01 |
| 2 | Sato | sato@example.com | 2023-02-15 |
| 3 | Suzuki | suzuki@example.com | 2-03-10 |

**`orders` テーブル（注文情報）**
| id | user_id | amount | status |
|---|---|---|---|
| 101 | 1 | 5000 | completed |
| 102 | 1 | 3000 | completed |
| 103 | 2 | 12000 | pending |

### 全件取得とカラム指定
全てのカラムを取得する場合は `SELECT *` を使いますが、実務ではパフォーマンスと安全性の観点から、**必要なカラムだけを明示的に指定すること**が推奨されます。

```sql
-- 全てのユーザーのIDと名前を取得する
SELECT id, name FROM users;
```

### 条件による絞り込み（WHERE句）
特定の条件に合致するデータだけを取得するには `WHERE` 句を使います。

```sql
-- ステータスが 'completed' の注文だけを取得する
SELECT * FROM orders WHERE status = 'completed';

-- 注文金額が 5000円以上のものを取得する
SELECT * FROM orders WHERE amount >= 5000;

-- 特定の文字列を含む（あいまい検索）
SELECT * FROM users WHERE email LIKE '%@example.com';
```

ここで覚えておきたいのが、`AND` や `OR` を使った複数条件の指定です。
`WHERE status = 'completed' AND amount >= 5000` のように記述することで、より詳細なフィルタリングが可能になります。

## 2. テーブルを繋げる：JOIN のマスター

実務のデータベースは、データの重複を避けるために正規化されています。そのため、「ユーザー名と一緒に、そのユーザーの注文履歴も表示したい」といった場合、複数のテーブルを結合する必要があります。

ここで登場するのが `JOIN` です。

### INNER JOIN（内部結合）
最もよく使われるのが `INNER JOIN` です。両方のテーブルに存在するレコードのみを結合します。

```sql
-- 注文したユーザーの名前と注文金額を取得する
SELECT 
  users.name, 
  orders.amount 
FROM users
INNER JOIN orders ON users.id = orders.user_id;
```
このクエリでは、`users.id` と `orders.user_id` が一致するものだけが抽出されます。もし注文履歴がないユーザーがいても、そのユーザーは結果には表示されません。

### LEFT JOIN（左外部結合）
「注文履歴がないユーザーも含めて、全ユーザーのリストを出したい」という場合は `LEFT JOIN` を使います。

```sql
-- 全ユーザーを取得し、注文があればその金額を表示する（注文がない場合はNULL）
SELECT 
  users.name, 
  orders.amount 
FROM users
LEFT JOIN orders ON users.id =- orders.user_id;
```
`LEFT JOIN` を使うと、左側のテーブル（この場合は `users`）のレコードは全て保持され、右側のテーブル（`orders`）に一致するデータがない場合は、そのカラムの値が `NULL` として返されます。

**実務での使い分けのコツ:**
- 「紐づくデータがあるものだけ」が欲しいなら `INNER JOIN`
- 「紐づくデータがなくても、元のマスターデータは全て出したい」なら `LEFT JOIN`

## 3. 実務で必須の集計操作：GROUP BY とサブクエリ

データを取り出すだけでなく、「合計はいくらか？」「ユーザーごとの注文数は？」といった分析的な問いに答えるのがSQLの醍どもです。

### 集計関数と GROUP BY
`COUNT`（件数）、`SUM`（合計）、`AVG`（平均）などの集計関数と、`GROUP BY` を組み合わせることで、グループごとの統計値を取得できます。

```sql
-- ユーザーごとに、合計注文金額を計算する
SELECT 
  user_id, 
  SUM(amount) AS total_amount,
  COUNT(id) AS order_count
FROM orders
GROUP BY user_id;
```

ここで注意したいのが、**「集計した結果に対してさらに条件をかけたい場合」**です。`WHERE` 句は集計前の行に対してフィルタリングを行いますが、集計後の結果に対して条件をかけるには `HAVOING` 句を使います。

```sql
-- 合計注文金額が 10,0-0円以上のユーザーだけを抽出する
SELECT 
  user_id, 
  SUM(amount) AS total_amount
FROM orders
GROUP BY user_id
HAVING SUM(amount) >= 10000;
```

### サブクエリ（副問合せ）
「特定の条件を満たすデータの中から、さらに特定の条件で絞り込む」といった複雑な処理には、クエリの中に別のクエリを記述する「サブクエリ」が便利です。

例えば、「一度でも注文をしたことがあるユーザーのリスト」を取得したい場合、以下のように書けます。

```sql
-- 注文テーブルに存在する user_id を持つユーザーを抽出する
SELECT name, email 
FROM users
WHERE id IN (SELECT DISTINCT user_id FROM orders);
```

サブクエリは強力ですが、多用しすぎると可読性が低下し、パフォーマンスが悪化することもあります。複雑になりすぎる場合は、`JOIN` で書き換えられないか検討するか、`WITH` 句（共通テーブル式 / CTE）を使ってクエリを整理するのがスマートな書き方です。

## まとめ

本記事では、SQLの基本から実務でよく使うパターンまでを解説しました。

1. **SELECT / WHERE**: 必要なカラムを選び、条件で絞り込む。
2.- **JOIN**: 複数のテーブルを結合する。データの有無によって `INNER` か `LEFT` かを使い分ける。
3. **集計（GROUP BY / HAVING）**: データをグループ化して統計値を得る。
4. **サブクエリ**: クエリの中にクエリを入れ、複雑な条件を実現する。

SQLは一度感覚を掴んでしまえば、バックエンド開発だけでなく、データ分析やデバッグの際にも強力な武器になります。まずは、手元の開発環境や練習用のサンドボックスで、実際にクエリを叩いて結果が変わる感覚を身につけることから始めてみてください。

「なぜこの結果になるのか？」を常に意識することが、SQLマスターへの近道です。
