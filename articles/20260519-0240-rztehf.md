### Gitの役割と初期設定  
Gitはバージョン管理システムで、複数人の開発者がコードを共有・協働するためのツールです。  

```bash
# Gitのバージョン確認
git --version
# ユーザー名とメールアドレスの設定（ Lochalty 重要）
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

### リポジトリ操作の流れ  
#### 1. ローカルリポジトリ初期化  
```bash
# ディレクトリ内でリポジトリ作成
mkdir my-project
cd my-project
git init
```

#### 2. リモートリポジトリクローン  
```bash
# GitHubなどのリモートリポジトリをクローン
git clone https://github.com/example/repo.git
cd repo
```

#### 3. ファイル追加とコミット  
```bash
# ファイルの変更を追跡
git add .
# ステージングエリアの確認
git status
# コミットメッセージを書く
git commit -m "Initial commit: Add README"
```

### コラボレーションワークフロー  
#### 4. 変更の取得（Pull）  
```bash
# リモートリポジトリの最新変更を取得
git pull origin main
```

#### 5. 新機能開発（Branch）  
```bash
# 新しいブランチ作成と切り替え
git checkout -b feature/login
# ブランチ一覧確認
git branch
```

### 実務で頻出なコマンド  
#### 6. ローカル変更のコミット  
```bash
git add .
git commit -m "Fix bug in login flow"
```

#### 7. リモートへのプッシュ  
```bash
git push origin feature/login
```

### 衝突（Conflict）の解決方法  
1. **衝突発生時のチェック**  
```bash
# 変更の差分を確認
git diff
```
2. **冲突ファイルの編集**  
```bash
# 衝突マーカーを手動修正
git status
# ファイル内の <<< HEAD、=======、>>>>>>>> を削除
git add resolved-file.txt
```
3. **コミットとプッシュ**  
```bash
git commit -m "Resolve merge conflict"
git push origin main
```

### 現場で役立つコマンドまとめ  
| 目的 | コマンド |  
|------|--------|  
| ローカルリポジトリ初期化 | `git init` |  
| リモートリポジトリクローン | `git clone URL` |  
| ファイル追加 | `git add .` |  
| コミット | `git commit -m "メッセージ"` |  
| プッシュ | `git push origin branch-name` |  
| プル | `git pull origin branch-name` |  

### まとめ  
- **基本フロー**：`clone → add → commit → push/pull`  
- **衝突時は必ず `git status` でファイルを確認！**  
- **ブランチ管理は `git checkout -b` で新規作成**  

このワークフローを習得することで、現場の開発プロジェクトでスムーズに協働できます。実務では「Pull Request」や「Rebase」などの上級技術も重要ですが、まずはこれらの基本を確実に押さえておきましょう。
