## はじめに

Visual Studio Code（以下 VS Code）は、拡張機能（Extension）と設定ファイルだけで自分好みの開発環境にカスタマイズできる点が大きな魅力です。  
しかし、何も入れずにデフォルトのままで使い続けていると、意外と作業が非効率になりがち。この記事では、**駆け出しエンジニアがすぐにでも導入すべき拡張機能**、**覚えておきたいキーバインド**、そして**チームや別マシン間で設定を共有するベストプラクティス**を具体例とコードスニペットを交えて解説します。

> **この記事のゴール**  
> 1. 本文で紹介する拡張機能をインストールすれば、コード補完・デバッグ・Git 操作がスムーズになる  
> 2. キーバインドをカスタマイズすれば、マウス操作を減らし作業時間を 10〜20% 短縮できる  
> 3. `settings.json` と `extensions.json` をリポジトリで管理すれば、環境構築が数分で完了する

## 背景・前提

- **対象読者**: VS Code を日常的に使い始めたばかりのエンジニア、または「設定が散らかっている」状態から脱却したい中級者  
- **前提知識**: 基本的な VS Code の操作（エディタの開閉、ターミナルの起動）ができること  
- **環境**: VS Code 1.85 以上、Node.js がインストールされたローカル環境（他言語でも同様に適用可能）  

この記事では、**拡張機能は「必須」10 個**、**キーバインドは「覚えておくべき」5 個**、**設定共有の手順は 3 ステップ**に絞って解説します。実際に手を動かしながら進めてみてください。

## 1. 開発効率が上がる必須拡張機能 10選

| # | 拡張機能 | 主な機能 | インストールコマンド |
|---|----------|----------|----------------------|
| 1 | **ESLint** | コーディング規約の自動検証・自動修正 | `code --install-extension dbaeumer.vscode-eslint` |
| 2 | **Prettier - Code formatter** | コード整形をワンクリックで統一 | `code --install-extension esbenp.prettier-vscode` |
| 3 | **GitLens — Git supercharged** | コミット履歴・ブレーム情報をインライン表示 | `code --install-extension eamodio.gitlens` |
| 4 | **Bracket Pair Colorizer 2** | 対応する括弧を色分けして視認性向上 | `code --install-extension CoenraadS.bracket-pair-colorizer-2` |
| 5 | **Path Intellisense** | ファイルパス補完を即時提示 | `code --install-extension christian-kohler.path-intellisense` |
| 6 | **Live Server** | HTML/JS の変更を即時リロード | `code --install-extension ritwickdey.LiveServer` |
| 7 | **Docker** | Dockerfile/Compose のシンタックスハイライトとデバッグ | `code --install-extension ms-azuretools.vscode-docker` |
| 8 | **REST Client** | VS Code 内で HTTP リクエストをテスト | `code --install-extension humao.rest-client` |
| 9 | **TODO Highlight** | `TODO:`・`FIXME:` をハイライト | `code --install-extension wayou.vscode-todo-highlight` |
|10| **Error Lens** | Lint エラー・警告をエディタ上部に強調表示 | `code --install-extension usernamehw.errorlens` |

### 1‑1. 拡張機能の一括インストールスクリプト

```bash
# extensions.sh
extensions=(
  dbaeumer.vscode-eslint
  esbenp.prettier-vscode
  eamodio.gitlens
  CoenraadS.bracket-pair-colorizer-2
  christian-kohler.path-intellisense
  ritwickdey.LiveServer
  ms-azuretools.vscode-docker
  humao.rest-client
  wayou.vscode-todo-highlight
  usernamehw.errorlens
)

for ext in "${extensions[@]}"; do
  code --install-extension "$ext"
done
```

このスクリプトをプロジェクトの `scripts/` に置き、`bash scripts/extensions.sh` で一括導入できます。新人が入社した際のセットアップ手順としても便利です。

## 2. 覚えておきたいキーバインド 5選

VS Code のデフォルトキーバインドは多くの操作に割り当てられていますが、**頻出操作は自分の手に馴染むように再割り当て**すると大きな効果があります。以下は macOS キーバインドを例にしていますが、`Ctrl` → `Cmd` 変換すれば Windows/Linux でも同様です。

| キー | 機能 | 設定方法 |
|------|------|----------|
| `Cmd + Shift + P` → `Cmd + B` | **コマンドパレット** → **サイドバーの表示/非表示** | `⌘B` に `workbench.action.toggleSidebarVisibility` を割り当て |
| `Cmd + D` → `Cmd + L` | **次の同一文字列選択** → **行全体選択** | `⌘L` に `editor.action.selectLine` |
| `Cmd + /` → `Cmd + K Cmd + C` | **コメントトグル** → **行コメントの追加** | `⌘K ⌘C` に `editor.action.commentLine` |
| `Ctrl + Shift + \` → `Cmd + \` | **ターミナル切替** → **ターミナルの表示/非表示** | `⌘\` に `workbench.action.terminal.toggleTerminal` |
| `Cmd + Shift + O` → `Cmd + P` | **シンボル検索** → **クイックオープン** | `⌘P` に `workbench.action.quickOpen` |

### 2‑1. `keybindings.json` に書く例

```json
// keybindings.json
[
  {
    "key": "cmd+b",
    "command": "workbench.action.toggleSidebarVisibility"
  },
  {
    "key": "cmd+l",
    "command": "editor.action.selectLine",
    "when": "editorTextFocus"
  },
  {
    "key": "cmd+k cmd+c",
    "command": "editor.action.commentLine",
    "when": "editorTextFocus && !editorReadonly"
  },
  {
    "key": "cmd+\\",
    "command": "workbench.action.terminal.toggleTerminal"
  },
  {
    "key": "cmd+p",
    "command": "workbench.action.quickOpen"
  }
]
```

> **Tips**  
> - `when` 条件は「エディタがフォーカス中」や「読み取り専用でない」など、コンテキストに応じて絞り込めます。不要な衝突を防ぐために条件を付けると快適です。

## 3. 設定・拡張機能の共有方法

### 3‑1. `settings.json` と `extensions.json` をリポジトリで管理

VS Code では **ワークスペース設定** (`.vscode/settings.json`) と **推奨拡張機能リスト** (`.vscode/extensions.json`) をプロジェクトにコミットできます。これにより、クローンしただけで同じ開発環境が再現します。

#### 3‑1‑1. `settings.json` の例

```json
// .vscode/settings.json
{
  // エディタ全体の統一感
  "editor.tabSize": 2,
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },

  // Prettier と ESLint の競合回避
  "prettier.requireConfig": true,
  "eslint.validate": ["javascript", "typescript"],

  // GitLens の表示設定
  "gitlens.hovers.enabled": true,
  "gitlens.codeLens.enabled": true,

  // ファイル除外（node_modules, dist など）
  "files.exclude": {
    "**/node_modules": true,
    "**/dist": true
  },

  // Live Server のポート固定
  "liveServer.settings.port": 5500
}
```

#### 3‑1‑2. `extensions.json` の例

```json
// .vscode/extensions.json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "eamodio.gitlens",
    "CoenraadS.bracket-pair-colorizer-2",
    "christian-kohler.path-intellisense",
    "ritwickdey.LiveServer",
    "ms-azuretools.vscode-docker",
    "humao.rest-client",
    "wayou.vscode-todo-highlight",
    "usernamehw.errorlens"
  ],
  "unwantedRecommendations": [
    "some.extension.you.dont.want"
  ]
}
```

### 3‑2. 設定の同期機能（Settings Sync）を活用する

個人のマシン間で同一設定を使いたい場合は、VS Code の **Settings Sync**（`Ctrl+Shift+P` → `Settings Sync: Turn On`）を有効化します。GitHub アカウントで認証すれば、以下の情報が自動でクラウドに保存されます。

- `settings.json`
- `keybindings.json`
- `snippets/`（自作スニペット）
- `extensions.json`（インストール済み拡張のリスト）

#### 3‑2‑1. 手動バックアップスクリプト（オプション）

```bash
#!/usr/bin/env bash
# backup-vscode.sh
CONFIG_DIR="$HOME/Library/Application Support/Code/User"
DEST_DIR="./vscode-backup"

mkdir -p "$DEST_DIR"
cp -r "$CONFIG_DIR"/* "$DEST_DIR/"

echo "VS Code 設定を $DEST_DIR にバックアップしました"
```

このスクリプトを CI の `post-install` ステップで走らせれば、**CI/CD 環境でも同一設定でテストが走ります**。

### 3‑3. チームでの導入フロー

1. **リポジトリに `.vscode/` を追加**  
   - `settings.json` と `extensions.json` をコミット
2. **README に「VS Code の推奨設定」セクションを追記**  
   - `code .` でプロジェクトを開くと自動で拡張機能インストールの案内が出ます
3. **新メンバーは `Settings Sync` を有効化し、リポジトリの設定を取得**  
   - `Ctrl+Shift+P` → `Preferences: Open Settings (JSON)` → 必要に応じて `settings.json` をマージ

これだけで、**新人がローカル環境を作る時間が 5 分以内**に短縮できます。
