### 1. 同期と非同期の違い
JavaScript はシングルスレッドの実行環境ですが、I/O 操作はブロックしない非同期で処理されます。同期コードは実行順序が保証されますが、ネットワークリクエストやファイル読み込みなどでは待機が発生しパフォーマンスが低下します。非同期はイベントループが複数のタスクを並列に処理できるため、ユーザー体験の向上に寄与します。

### 2. Promise の概念
Promise は「約束」を表すオブジェクトで、非同期処理の結果または失敗を表します。`new Promise((resolve, reject) => {...})` で生成し、`resolve` が成功、`reject` が失敗を示します。`then` で結果を受け取り、`catch` でエラーを捕捉できます。

## Promise の基本と then の使い方
```js
// 簡単な Promise の生成例
function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      // 成功時
      resolve('データ取得完了');
    }, 1000);
  });
}

// then で結果を取得
fetchData()
  .then(data => {
    console.log(data); // => データ取得完了
    return fetchData(); // 再帰的にチェーン可能
  })
  .then(data2 => {
    console.log(data2); // 再びデータ取得完了
  })
  .catch(err => {
    console.error('エラー:', err);
  });
```
then は複数回チェーンでき、各ステップで新しい Promise を返すため、途中で失敗した場合は `catch` が捕捉されます。エラーハンドリングは最後に `catch` を置くか、各 `then` に `catch` を付けるかで柔軟に対応できます。
