## 背景：AIツールは「非常に近いが、間違いがある」存在だという認識

AIツールの利便性はとても高く、ある程度のコードを高速で生成してくれます。しかし気づくと「自分が書いていないコード」のまま、すぐに本番環境に導入していることに気づきます。そんな状況を避けるためには、まず「AIツールは完璧ではない」という前提を持つことが大切です。

筆者の場合、最初は「Claude CodeにAPI設計を任せっき」と大胆なことをしました。結果、フロントエンドからBackendの型定義まで自動生成され、チームメンバーも「これでいいや」と受け入れたのです。すると数日後、実際に実装しようとしたとき、「なぜかユーザー情報が2重になる構造」「認証エラーのハンドリングが不十分」などの問題が見つかりました。AIは知識を持っている一方で、プロダクト固有のビジネスロジックを完全に理解しているわけではありません。

## 1. AIへのリクエスト（プロンプト）の作り方のコツ

AIツールを活用する最大のコツは、「リクエストを具体的にする」ことです。曖昧な指示だと、AIも曖昧な回答をしてきます。

### 良い例：
```
次の要件でユーザー登録機能を実装してください。
- メールアドレスとパスワードの必須入力
- パスワードは8文字以上かつ英字と数字を含むこと
- 既存のユーザーがいたらエラーメッセージを表示
- 成功時はJSONで{"status": "success"}を返す
- 使用するフレームワークはExpress.js
- 既存のコードベースに追加する形でお願いします
```

### 悪い例：
```
ユーザー登録機能作って
```

良い例では「技術スタック」「入出力仕様」「errorの扱い方」「既存コードとの統合方法」まで指定しています。これによりAIはコンテキストを正しく理解し、より適したコードを生成できます。

また、AIへのリクエストには「制約条件」を明示することも忘れません。例えば「既存のコードベースに追加する形で」と書くことで、既存のパターンや命名規則に応じたコードを生成してもらえます。

## 2. AIの仕事と自分の仕事をはっきりさせる：役割の定義

AIツールの役割を「高速な下書き作成者」「疑rospectのコードレビュー」「リファクタリングのサポート」に限定することで、開発サイクルの効率化を図ります。

### AIに任せる主なタスク：
- ざっくりしたロジックのスケッチ作成
- 冗長な Boilerplate コードの生成
- 既存コードのリファクタリング提案
- テストコードの雛形作成

### 私が必ず手を入れる部分：
- ビジネスロジックの核心部分
- 外部APIとのインターフェース
- セキュリティ関連の処理
- パフォーマンスに関する最適化

具体的な例を見てみましょう。次のようなコードをAIが生成したとします。

```javascript
// AI生成コード
function calculateTotal(items) {
  let total = 0;
  for (let i = 0; i < items.length; i++) {
    total += items[i].price * items[i].quantity;
  }
  return total;
}
```

これを私が確認・改良するのです。「税率込み」「割引適用」「通貨単位の付加」など、プロダクト固有の要件を追加します。

```javascript
// 私が改良したコード
function calculateTotal(items, discountRate = 0, taxRate = 0.1) {
  const subtotal = items.reduce((sum, item) => {
    return sum + (item.price * item.quantity);
  }, 0);
  
  const discounted = subtotal * (1 - discountRate);
  const totalWithTax = Math.round(discounted * (1 + taxRate));
  
  return {
    subtotal,
    discountAmount: subtotal * discountRate,
    total: totalWithTax,
    currency: 'JPY'
  };
}
```

このように、AIに「ある程度の完成度」まで作ってもらい、自分は「必要な改良を最小の手間で行う」スタイルです。

## 3. コードレビューの時間を活用する：AI出力の検証サイクル

AI生成コードをそのままマージするのではなく、必ず「自分の目」と「テスト」をかけます。この検証フェーズを急がずに取り込むことで、品質を保つことができます。

### 確認するポイント：
1. **エラーハンドリング**：想定外の入力やAPIエラーはきちんと処理されているか？
2. **パフォーマンス**：ループやネストが深すぎないか？
3. **セキュリティ**：SQLインジェクションやXSS、認証情報の漏洩などはないか？
4. **可読性**：変数名や関数名がチームのスタイルに合っているか？

具体的なチェックフローとして、私は次のようなワークフローを採っています。

1. AI生成コードを一目で確認（5分）
2. 自分のスタイルガイドに合わせて命名を修正（10分）
3. 単純なユニットテストを書く（15分）
4. レビューしてもらったメンバーに確認してもらう（10分）

これにより、「AIの素早さ」と「人間の確実さ」を掛け合わせることができます。

## まとめ：AI時代の開発現場で生き残るための3つの原則

いかがでしょうか？ AIツールを開発に活用する際、私が心がけているのは次の3つの原則です。

1. **AIへのリクエストは具体的かつ制約を含める**
2. **AIの出力には必ず人間の手を加える**
3. **コードレビューの時間を惜しまない**

これらを守ることで、AIの「非常に良い助手」になる一方で、「怠惰を誘発される」リスクを回避できます。最後に書いておくても、AIツールは開発の「手助け」であり、「代替」ではありません。技術の進化は素晴らしいですが、その先には「理解」と「責任」がついてきます。ぜひ皆さんも、AIを活用する際にはこれらのポイントを意識してみてください。
