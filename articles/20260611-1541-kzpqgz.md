## 背景・前提
- **対象読者**: VS Codeを初めて使う人、もしくは拡張機能に不慣れな人
- **前提知識**: 基本的なコマンドパレットや設定ファイル（`settings.json`）の編集方法を知っていること
- **環境**: Windows 10/11 / macOS Catalina以降 / Linux (Ubuntu 20.04以降) で確認済み

VS Codeは「拡張機能を入れれば入れるほど便利になる」というイメージですが、逆に設定が多すぎると逆に迷子になりがちです。そこで「必要最低限の拡張機能 + シンプルな設定」を実現するためのフレームワークを紹介します。

## 本文

### 1. 基本的に入れておくべき拡張機能
| 拡張機能 | 主な機能 | 具体的な使い方例 |
|----------|----------|------------------|
| **Prettier - Code formatter** | コードフォーマット統一 | `Shift+Alt+F` で自動整形 |
| **ESLint** | JavaScript/TypeScript の linting | 右下の赤いバーをクリックしてエラーを修正 |
| **GitLens — Git supercharged** | Git情報の可視化 | コミットログをサイドバーで確認 |
| **Live Server** | ローカル開発サーバー | `Alt+L, A` で起動 |
| **Bracket Pair Colorizer 2** | 括弧の視認性向上 | 自動で色付け |
| **Todo Tree** | TODOコメントの一覧表示 | `Ctrl+Shift+P` → `Todo Tree: Show` |
| **Remote - SSH** | SSH経由でリモート開発 | `Remote-SSH: Connect to Host...` |
| **REST Client** | APIテスト | `Send Request` をクリック |
| **Path Intellisense** | パス補完 | ファイルパス入力時に候補表示 |
| **Settings Sync** | 設定・拡張機能の同期 | `Ctrl+Shift+P` → `Settings Sync: Turn On` |

#### コード例: Prettier と ESLint の連携
```json
// .vscode/settings.json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "eslint.validate": ["javascript", "typescript"],
  "eslint.alwaysShowStatus": true
}
```
この設定により、保存時に自動整形と lint が同時に走ります。

### 2. キーバインドのカスタマイズ
VS Codeはデフォルトのキーバインドで十分ですが、作業フローに合わせて微調整するとさらに高速化できます。  
#### 例: `Ctrl+Alt+L` でファイル全体を整形
```json
// keybindings.json
[
  {
    "key": "ctrl+alt+l",
    "command": "editor.action.formatDocument",
    "when": "editorHasDocumentFormattingProvider && editorTextFocus && !editorReadonly"
  }
]
```
#### 例: `Ctrl+Shift+O` で関数・クラス一覧へジャンプ
```json
{
  "key": "ctrl+shift+o",
  "command": "workbench.action.gotoSymbol",
  "when": "editorTextFocus && !editorReadonly"
}
```

### 3. 設定の共有方法
チーム開発では「環境の差異」が大きなボトルネックになるケースがあります。以下の手順で設定を共有しましょう。

#### 3-1. `settings.json` を Git にコミット
`.vscode/` ディレクトリをプロジェクトルートに配置し、以下を `.gitignore` から除外します。
```gitignore
# .gitignore
!/.vscode/*
```

#### 3-2. 拡張機能リストを `extensions.json` に記述
`.vscode/extensions.json` に推奨拡張機能をリストアップします。
```json
{
  "recommendations": [
    "esbenp.prettier-vscode",
    "dbaeumer.vscode-eslint",
    "eamodio.gitlens",
    "ritwickdey.LiveServer",
    "CoenraadS.bracket-pair-colorizer-2",
    "wayou.vscode-todo-highlight",
    "ms-vscode-remote.remote-ssh",
    "humao.rest-client",
    "christian-kohler.path-intellisense",
    "Shan.code-settings-sync"
  ]
}
```
これをクローンした際に `Ctrl+Shift+P` → `Extensions: Install Recommended Extensions` で一括インストール可能です。

#### 3-3. Settings Sync で個人設定を同期
`Settings Sync` を使えば、個人の拡張機能や設定を GitHub Gist を経由してクラウドに保存できます。設定画面から `Turn On` を選び、GitHub アカウントで認証すると、他のマシンでも同じ設定が復元されます。

### 4. 便利なショートカットを覚えておくと助かる
| 機能 | キー | 説明 |
|------|------|------|
| コマンドパレット | `Ctrl+Shift+P` | すべてのコマンドを検索 |
| ターミナル表示 | `Ctrl+` | 統合ターミナルを開閉 |
| コメントアウト | `Ctrl+/` | 行コメント |
| ブレークポイント設定 | `F9` | デバッグブレークポイント |
| 変更履歴確認 | `Alt+Shift+G` | Git Diff |

## まとめ
- **拡張機能は「必須」と「便利」を分けて選ぶ**：Proの作業を高速化するものだけを入れ、不要なものは削除。
- **設定はプロジェクト単位で管理**：`.vscode/settings.json` と `extensions.json` を Git にコミットしてチーム全員で統一。
- **キーバインドは自分の作業フローに合わせてカスタマイズ**：頻繁に使う操作はショートカットに割り当てる。
- **Settings Sync で個人設定をクラウド化**：複数マシンで同一環境を即座に構築。

これらを実践すれば、VS Code が「軽くて簡単」から「プロフェッショナルな開発環境」へと変貌します。ぜひ今日から試してみてください！
