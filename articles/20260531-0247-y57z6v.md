## Dockerの基本：イメージとコンテナの違い  
Dockerは「アプリケーションを隔離した環境で動かす」仕組みを提供します。まずは、**イメージ**と**コンテナ**の違いを理解しましょう。  

### イメージとは何か？  
イメージは、**アプリケーションとその依存関係を記述したテンプレート**です。例えば、`nginx:latest`というイメージは、Nginxサーバーを動かすための設定がすでに含まれています。Docker Hubなどで公開されたイメージを使用することで、ローカルに必要な環境を素早く構築できます。  

### コンテナとは何か？  
コンテナは、**イメージを元に実際に実行されている状態**です。例えば、`docker run`コマンドでイメージを起動すると、そのイメージをベースにプロセスが動き始めます。コンテナは一時的なもので、変更が加えられても元のイメージには影響しません。  

### イメージとコンテナの関係性  
- **イメージ**は「設計図」、コンテナは「実際の建物」。  
- イメージは共有可能で、複数のコンテナが同じイメージをベースに起動できます。  
- コンテナは一時的なもので、終了するとデータや設定が失われます（ただし、`docker commit`で変更をイメージに反映することも可能）。  

### コマンド例：イメージの確認  
```bash
# イメージ一覧を表示
docker images

# イメージを削除
docker rmi <イメージ名>
```  

## Docker Compose：複数サービスを一括で管理する方法  
開発環境では、データベースやキャッシュサーバーなど、複数のコンテナが連携するケースが多いです。このような場合、**docker-compose**を使うと、複数のコンテナを一括で管理できます。  

### docker-compose.ymlの基本構造  
`docker-compose.yml`は、コンテナの設定を記述するYAMLファイルです。以下は、MySQLとアプリケーションサーバーを起動する例です。  

```yaml
version: '3'
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: example
      MYSQL_DATABASE: mydb
    volumes:
      - db_data:/var/lib/mysql
    ports:
      - "3306:3306"

  app:
    build: .
    ports:
      - "8000:8000"
    depends_on:
      - db

volumes:
  db_data:
```

### コマンド例：Composeで環境を構築  
```bash
# 環境を構築
docker-compose up -d

# コンテナの状態を確認
docker-compose ps

# コンテナを停止
docker-compose down
```

### Composeのメリット  
- **複数コンテナの依存関係を明示**できるため、開発環境の再現性が高まります。  
- **設定ファイルの管理が簡単**で、チーム間での環境の整備が容易になります。  
- **ローカル開発での依存関係の統一**が可能です。  

## ローカル開発でDockerを使うメリット  
Dockerは「開発環境」に特化したメリットが豊富です。以下に具体的なポイントを紹介します。  

### 1. 環境の再現性を保つ  
「自分のPCで動くが、チームのPCでは動かない」という問題を解消します。Dockerは環境を「イメージ」として保存できるため、誰でも同じ環境で開発を進められます。  

### 2. 依存関係の管理を効率化  
データベースやAPIサーバーなど、複数のコンポーネントが必要なプロジェクトでは、`docker-compose`で全てのコンテナを一括で起動できます。これにより、設定ミスのリスクが減少します。  

### 3. ローカルでのテストが簡単になる  
Dockerコンテナを使うことで、実際の運用環境に近い形で開発を進められます。例えば、MySQLのバージョンやサーバー設定を変更せずに、ローカルでテストできます。  

### 4. セキュリティと隔離性  
コンテナはホストマシンから隔離されるため、サーバーの設定ミスやセキュリティリスクを最小限に抑えることができます。  

## まとめ  
Dockerは、開発環境の構築や運用環境の管理に役立つ強力なツールです。イメージとコンテナの違いを理解し、`docker-compose`を活用することで、複数コンテナの依存関係を簡単に管理できます。ローカル開発では、環境の再現性や依存関係の統一が可能になり、開発効率が大幅に向上します。
