## 前提条件と背景

VS Code単体でも十分に便利ですが、適切な拡張機能と設定を活用すれば、以下のようなメリットがあります：

- コード記述速度の向上
- エラー発見の容易さ
- チーム内での設定統一
- 作業環境のポータビリティ

特に新しいエンジニアは、どの拡張機能を入れるべきか悩むことが多いでしょう。本記事で紹介する内容を参考に、効率的な開発環境を構篲してみましょう。

## 必須の拡張機能紹介

### 1. Prettier - コードフォーマッター

Prettierはコードを自動的にフォーマットするツールです。一貫したコードスタイルを維持できるため、チーム開発での相互理解が深まります。

```json
// settings.jsonでの設定例
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "prettier.tabWidth": 2,
  "prettier.useTabs": false
}
```

### 2. ESLint - JavaScript静的解析

ESLintはコーディング規約違反や潜在的なバグを検出します。特にJavaScript/TypeScript開発では欠かせません。

```json
// .eslintrc.jsonの例
{
  "extends": ["eslint:recommended"],
  "rules": {
    "no-unused-vars": "warn",
    "no-console": "warn"
  }
}
```

### 3. GitLens - Git履歴の可視化

GitLensを使えば、コードの変更履歴をGUIで簿表示したり、特定の行がいつ変更されたかを素早く確認できます。

### 4. Bracket Pair Colorizer - かっこの視覚化

入れ子になったかっこの対応関係を色で視覚化します。特に複雑なJSONやネスト構造のコードを読む際に役立ちます。

### 5. Auto Rename Tag - HTML/XMLタグの自動リネーム

HTMLやXMLで開始タグと終了タグのペアを入力すると、一方を変更した時にもう一方を自動でリネームします。

## 便利なキーバインド集

効率的なワークフローを実現するためのおすすめショートカット：

| 機能 | キーバインド | 説明 |
|------|-------------|------|
| コマンドパレット | `Ctrl+Shift+P` | あらゆるコマンドにアクセス |
| ファイル切り替え | `Ctrl+P` | クイックオープン |
| 行の移動 | `Alt+↑/↓` | 選択中の行を上下に移動 |
| 端末表示/隠す | `` Ctrl+` `` | 統合ターミナルの表示切替 |
| ブランチ切り替え | `Ctrl+Shift+G` | Gitコントロールパネル |

```json
// カスタムキーバインドの例 (keybindings.json)
[
  {
    "key": "ctrl+shift+h",
    "command": "editor.action.replaceAll",
    "when": "editorTextFocus"
  }
]
```

## 設定の共有方法

チームで開発環境を統一するためには、設定の共有が不可欠です。VS Codeでは主に2つの方法で設定を共有できます。

### 1. Settings Syncの活用

Microsoftが提供するSettings Syncを使用すると、GitHubやMicrosoftアカウントと同期させることができます。

```bash
# コマンドパレットで "Settings: Turn On Settings Sync" を実行
# 初期設定ではMicrosoftアカウントがデフォルトですが、GitHubとの連携も可能
```

### 2. JSON設定ファイルのバージョン管理

`.vscode/settings.json`と`.vscode/extensions.json`をプロジェクトルートに配置することで、チームメンバーのVS Codeが自動的に必要な拡張機能を推奨します。

```json
// .vscode/settings.json
{
  "python.linting.enabled": true,
  "python.linting.pylintEnabled": true,
  "editor.rulers": [80, 100]
}
```

```json
// .vscode/extensions.json
{
  "recommendations": [
    "ms-python.python",
    "ms-python.pylint"
  ],
  "unwantedRecommendations": []
}
```
