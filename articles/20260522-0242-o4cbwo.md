## 1. 型とは何か

TypeScriptは「型」を中心にした静的型付け言語です。コンパイラはコード中の変数や関数がどんな値を持つのかを解析し、実行前にエラーを検出します。初心者にとっては「型を付ける」こと自体が新しい概念です。従来のJavaScriptでは型を意識しなくても動作しましたが、TypeScriptでは `number`, `string`, `boolean` などの基本型からユーザー定義型まで、あらゆる値に型を付与します。この違いは、コードの可読性向上やリファクタリングのしやすさに大きく寄与しますが、一方で型定義の書き方に慣れるまで混乱しやすいポイントも多くあります。

## 2. 基本的な型定義の書き方

変数に型を明示的に指定する最もシンプルな例が以下です。

```ts
let count: number = 0;
let name: string = "Zenn";
let isActive: boolean = true;
```

関数に型を付けるときは、引数と戻り値の両方に型を記述します。

```ts
function add(a: number, b: number): number {
  return a + b;
}
```

上記のように明示的に型を書くことを「**型アノテーション**」と言います。型推論が働くケースも多く、コンパイラが自動で型を決定できることがありますが、初心者は「どこに型を書くべきか」が曖昧になりがちです。特に、配列やオブジェクトといった複雑な構造体の型定義は、次のセクションで詳しく見ていきましょう。

## 3. any の危険性と安全に回避する方法

`any` は「型が不明な値」を表す特別な型です。一時的に型情報を失う必要があるときや、徐々にTypeScript化する過程で便利に使えますが、過剰に使うと型チェックの意味を失ってしまいます。代表的なパターンと対策を以下に示します。

### 3.1. `any` が混入しやすい場面

```ts
let data: any = fetch('/api/users');
```

上記のように非同期通信の応答や、外部ライブラリから取得したデータを `any` で受け取ると、コンパイラは以降の操作で型チェックをしません。その結果、意図しないプロパティへのアクセスや型違いの計算が実行されてもエラーに気づきにくくなります。
