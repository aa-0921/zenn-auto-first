## Docker の基本概念：イメージとコンテナの違い

Docker を理解する上で重要なのが「イメージ」と「コンテナ」の違いです。これらを理解せずに Docker を使うと、混乱の元になります。

### イメージとは

イメージは、アプリケーションとその実行環境をまとめたものです。たとえば、Ubuntu をベースにした Node.js アプリケーションの実行環境を含むイメージを作れます。イメージは不変であり、一度作成されると変更されません。

### コンテナとは

コンテナは、イメージから作成され実行されるインスタンスです。コンテナはイメージの上に載って動作し、実行中に変更を加えることができます。しかし、その変更はコンテナ内に閉じられ、元のイメージには影響を与えません。

### 例: Docker Hub からイメージを取得してコンテナを起動

```bash
# nginx のイメージを取得
docker pull nginx

# イメージからコンテナを起動
docker run -d -p 8080:80 --name my-nginx nginx
```

このコマンドで nginx のコンテナが起動し、ブラウザで http://localhost:8080 にアクセスすると「Welcome to nginx!」と表示されます。

## docker-compose で開発環境を構築する

docker-compose は複数のコンテナを一括で管理するためのツールです。たとえば、Web アプリケーションとデータベースを同時に起動したい場合に便利です。

### docker-compose.yml の基本構造

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - .:/app
    depends_on:
      - db

  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: myapp
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

### Dockerfile の例

```dockerfile
FROM node:16

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
```

### docker-compose の基本コマンド

```bash
# コンテナを起動
docker-compose up -d

# コンテナを停止
docker-compose down

# ログを確認
docker-compose logs -f

# 特定のサービスだけ起動
docker-compose up -d web
```

## ローカル開発で Docker を使うメリット

Docker を使った開発環境には大きなメリットがあります。

### 環境の一貫性

Docker を使えば、チーム全員が同じ環境を共有できます。これにより、「私の環境では動く」という問題を防ぐことができます。

### 依存関係の管理

データベースやキャッシュサーバーなどの外部サービスも Docker で管理できるため、ローカルにインストールする必要がありません。

### 簡単なセットアップ

新しいチームメンバーが参加した際、リポジトリをクローンして `docker-compose up` を実行するだけで開発環境が整います。

## まとめ

Docker は開発環境の構築と管理を劇的に簡単にするツールです。イメージとコンテナの違いを理解し、docker-compose を活用することで、一貫した開発環境を簡単に構築できます。ぜひ今回学んだ方法を実践で活用してみてください。
