## 2. 必須拡張機能
以下に、 daily development で最低限入れておきたい拡張機能をジャンル別にまとめました。各拡張機能のインストール手順と、主な機能概要を簡潔に説明します。

### 2.1 コード整形・ linting
- **Prettier - Code formatter**  
  ```json
  // .vscode/settings.json に追記
  {
    "editor.formatOnSave": true,
    "prettier.requireConfig": true
  }
  ```
  ファイル保存時に自動でコード整形を行い、チーム全体のコーディング規約を統一します。

- **ESLint**  
  ```json
  // .eslintrc.js があれば自動で適用
  {
    "eslint.validate": ["javascript", "javascriptreact", "typescript", "typescriptreact"]
  }
  ```

### 2.2 デバッグ・デプロイ
- **GitLens — Git supercharged**  
  リポジトリ履歴や変更点をビジュアルに表示し、コミットの背景を即座に確認できます。

- **Remote - SSH**    リモートサーバーへの SSH 接続を直接 VS Code から管理。サーバー側の拡張機能をローカルと同様に利用可能です。

### 2.3 UI/UX 向上
- **Bracket Pair Colorizer 2**  
  対応する括弧を色分けして見やすくします。ネストが深くなる JSON や XML で特に便利です。

- **Error Lens**  
  エラーメッセージをコード上に直接表示し、警告を目視でキャッチしやすくします。

## 3. 便利な設定とキーバインド
拡張機能だけでなく、設定ファイルを共有することでチーム全体の環境を統一できます。ここでは、代表的な設定例とキーバインドのカスタマイズ方法を紹介します。

### 3.1 settings.json のサンプル
```json
{
  // エディタの基本設定
  "editor.tabSize": 2,
  "editor.insertSpaces": true,
  "editor.renderWhitespace": "all",
  "editor.minimap.enabled": false,

  // ファイル自動保存とフォーマット
  "files.autoSave": "afterDelay",
  "editor.formatOnSave": true,

  // 文字コードと改行コード
  "files.encoding": "utf8",
  "files.eol": "\n",

  // 重要な拡張機能の無効化（例：不要な Welcome ページ）
  "extensions.welcome.enabled": false
}
```

### 3.2 keybindings.json でのカスタムショートカット
VS Code のキーバインドは `keybindings.json` で編集可能です。以下は頻繁に使う操作をショートカット化した例です。

```json
[
  {
    "key": "ctrl+shift+p",
    "command": "workbench.action.quickOpen",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+alt+n",
    "command": "workbench.action.newFile",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+alt+o",
    "command": "editor.action.openNext",
    "when": "editorTextFocus"
  }
]
```

### 3.3 設定の共有方法
1. **リポジトリに `.vscode/` ディレクトリを作成**  
   `settings.json` と `keybindings.json` を配置し、`.gitignore` に `*.code-workspace` などを除外します。

2. **チーム内で共有**  
   `git clone` 後に自動で設定が適用されるよう、`postCreateCommand` を利用した Workspace の自動適用が推奨です。

3. **個人向けの上書き**  
   `User Settings` で `settings.json` を開き、特定のキーだけ上書きすれば、リポジトリのデフォルト設定を崩さずにカスタマイズできます。

## 4. まとめ
VS Code の拡張機能と設定を適切に組み合わせることで、コードの可読性・保守性・デバッグ効率が格段に向上します。本記事で紹介した拡張機能は「必須」ではなく「推奨」ですが、まずは上記の設定を導入し、徐々に自分流にカスタマイズしていくのがベストです。ぜひこの記事を参考に、開発環境を最適化し、日々のコーディングをよりスムーズにしてください。
