## 1. 開発効率を劇的に上げるおすすめ拡張機能

拡張機能を入れすぎるとエディタが重くなるため、「何でも入れる」のではなく「目的を持って入れる」ことが重要です。ここでは、汎用性が高く、多くの現場で標準的に導入されているものを厳選して紹介します。

### コード品質とフォーマットを自動化する
手動でインデントを調整したり、セミコロンの有無を気にしたりするのは時間の無駄です。これらはツールに任せましょう。

- **Prettier - Code formatter**:
  業界標準のコードフォーマッターです。保存時に自動でコードを整形するように設定することで、チーム内でのコードスタイルを統一し、レビュー時の「書き方の指摘」をゼロにできます。
- **ESLint**:
  JavaScript/TypeScript の静的解析ツールです。バグになりそうな記述や、使われていない変数などをリアルタイムで警告してくれます。

### 開発体験（DX）を向上させる
- **Error Lens**:
  通常、エラーや警告は波線で表示され、マウスホバーしないと内容が見えませんが、Error Lens を入れると**行の末尾にエラー内容が直接表示**されます。これにより、気づいた瞬間に修正できるため、デバッグ効率が大幅に上がります。
- **GitLens — Git supercharged**:
  「この行は誰がいつ、何の目的で変更したのか」をエディタ上で即座に確認できます。実務でのコードリーディングにおいて、Git の履歴を辿る回数を劇的に減らせます。
- **Path Intellisense**:
  ファイルパスの補完機能です。長いディレクトリ構造の中でファイルパスを手入力してタイポし、実行時に「File not found」で悩む時間をなくせます。

### 視認性と操作性を高める
- **Material Icon Theme**:
  ファイル形式に応じてアイコンを分かりやすく表示します。フォルダ構成を視覚的に把握しやすくなるため、ファイル探しに迷う時間が減ります。
- **Rainbow Brackets / Bracket Pair Colorization (標準機能)**:
  ※現在は VS Code の標準機能として搭載されています。`editor.bracketPairColorization.enabled: true` を設定することで、対応する括弧に色がつき、ネストが深いコードでも構造を把握しやすくなります。

## 2. 迷ったらこれを設定！おすすめの `settings.json`

VS Code の設定は GUI でも変更可能ですが、`settings.json` で管理することで、設定内容をテキストとして保存・共有できます。以下に、実務で推奨される設定例を挙げます。

### 設定の開き方
`Cmd + Shift + P` (Mac) または `Ctrl + Shift + P` (Win) を押し、`Open User Settings (JSON)` と入力して Enter を押してください。

### 推奨設定例
```json
{
  // --- エディタの基本設定 ---
  "editor.tabSize": 2, // インデントを2スペースに設定（多くのプロジェクトの標準）
  "editor.fontSize": 14,
  "editor.wordWrap": "on", // 長い行を自動で折り返す
  "editor.cursorBlinking": "smooth", // カーソルの点滅を滑らかにする（視覚的なストレス軽減）
  "editor.formatOnSave": true, // 保存時に自動でフォーマットを適用（Prettierと連携）

  // --- 検索・ファイル操作 ---
  "files.autoSave": "onFocusChange", // ウィンドウからフォーカスが外れた時に自動保存
  "explorer.confirmDelete": false, // ファイル削除時の確認ダイアログを非表示にする

  // --- 視覚的な補助 ---
  "editor.bracketPairColorization.enabled": true, // 括弧のペアに色をつける
  "editor.guides.bracketPairs": "active", // 現在のスコープのガイドラインを強調
  "editor.minimap.enabled": false, // ミニマップが不要な場合はオフにして作業領域を広げる

  // --- 言語別の設定 (例: TypeScript/JavaScript) ---
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

特に `editor.formatOnSave: true` は必須級の設定です。保存するたびにコードが綺麗になる快感は、開発のモチベーションにも繋がります。

## 3. 操作スピードを加速させるキーバインドとショートカット

マウスに手を伸ばす回数を減らすことが、コーディング速度を上げる最短ルートです。以下のショートカットは、指に覚え込ませることをおすすめします。

### 必須の基本操作
- **コマンドパレット**: `Cmd + Shift + P` / `Ctrl + Shift + P`
  - 全ての機能への入り口です。設定変更、拡張機能の実行など、ここから検索して実行します。
- **クイックオープン**: `Cmd + P` / `Ctrl + P`
  - ファイル名を入力して瞬時にファイルを開けます。サイドバーからファイルを探す必要はありません。
- **シンボル検索**: `Cmd + Shift + O` / `Ctrl + Shift + O`
  - ファイル内の関数名や変数名へジャンプできます。長いファイルでの移動に必須です。

### 編集効率を上げる操作
- **行の移動**: `Alt + ↑ / ↓`
  - 選択した行をそのまま上下に移動させます。切り取り＆貼り付けの手間が省けます。
- **行のコピー**: `Shift + Alt + ↓ / ↑`
  - 現在の行をそのまま下にコピーします。
- **マルチカーソル**: `Cmd + D` / `Ctrl + D`
  - 同じ単語を順番に選択し、一括で置換できます。正規表現を使わずに簡易的な一括変更ができるため非常に便利です。

## 4. チームで設定を共有する方法

個人で設定を最適化できたら、次はチームでの共有です。「人によってインデントが違う」「フォーマッターが違う」といった状況は、Git の差分（diff）に不要な変更が混じり、レビューの妨げになります。

### `.vscode/settings.json` の活用
プロジェクトのルートディレクトリに `.vscode` フォルダを作成し、その中に `settings.json` を配置します。ここにプロジェクト固有の設定を記述し、Git にコミットすることで、チーム全員が同じエディタ設定で開発できます。

### `.vscode/extensions.json` で推奨拡張機能を提示
チームメンバーに「この拡張機能を入れてください」と口頭で伝えるのではなく、`extensions.json` に推奨リストを記載しましょう。

```json
{
  "recommendations": [
    "esbenp.prettier-vscode",
    "dbaeumer.vscode-eslint",
    "eamodio.gitlens"
  ]
}
```
これを設定しておくと、プロジェクトを開いた際に VS Code が「このリポジトリで推奨されている拡張機能があります。インストールしますか？」と通知してくれるため、環境構築の手間を大幅に削減できます。

## まとめ

VS Code のカスタマイズに正解はありませんが、「不便だと感じた瞬間に設定を探す」という習慣をつけることが大切です。

1. **拡張機能**で機能を追加し、自動化できるものは全て自動化する。
2. **`settings.json`** で自分に合った視認性を確保する。
3. **ショートカット**を習得し、キーボードから手を離さない時間を増やす。
4. **共有設定**を用いて、チーム全体の開発体験を底上げする。

まずは本記事で紹介した設定を導入し、自分にとって最適な「最強の武器」に育ててみてください。
