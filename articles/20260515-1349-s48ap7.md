## REST APIとは何か？

REST（Representational State Transfer）APIは、クライアントとサーバーがデータをやり取りするための標準的なプロトコルです。たとえば、ウェブアプリケーションでユーザー登録や商品一覧を取得する際、フロントエンド（ブラウザやアプリ）がバックエンド（サーバー）と通信する際に使われます。REST APIの特徴は「リソース指向」と「無状態（stateless）」です。リソースはURL（例: `/users/1`）で表され、各リクエストは前後の状態に依存しない設計になっています。

## HTTPメソッドとステータスコードの基礎

### HTTPメソッドの種類
HTTPメソッドはリクエストの種類を表します。以下に主な4つを解説します。

#### GETメソッド（データ取得）
サーバーからデータを取得する際に使用します。例: `/users`にGETを送ると、全ユーザーのリストを取得できます。
```javascript
// fetchを使ったGETリクエスト
const response = await fetch('https://jsonplaceholder.typicode.com/users');
const data = await response.json();
console.log(data); // JSON形式のユーザーデータ配列が表示
```

#### POSTメソッド（データ送信）
サーバーにデータを送信する際に使います。例: 新規ユーザー登録の場合、ユーザー情報をJSON形式で送信します。
```javascript
// axiosを使ったPOSTリクエスト
const axios = require('axios');
const user = { name: 'John', email: 'john@example.com' };
const response = await axios.post('https://jsonplaceholder.typicode.com/users', user);
console.log(response.data); // 新規登録されたユーザーIDが返却
```

#### PUT/PATCHメソッド（データ更新）
PUTはリソースを完全更新、PATCHは部分更新に使います。例: ユーザーのメールアドレスを変更する場合。
```javascript
// PUTリクエスト例
const updatedUser = { id: 1, email: 'new_email@example.com' };
await axios.put('https://jsonplaceholder.typicode.com/users/1', updatedUser);
```

#### DELETEメソッド（データ削除）
リソースを削除する際に使用します。例: ユーザーID1のデータを削除。
```javascript
await axios.delete('https://jsonplaceholder.typicode.com/users/1');
```

### ステータスコードの意味
サーバーから返されるステータスコードは、リクエストの結果を示します。よく使うものを以下にまとめます。

| ステータスコード | 意味 |
|------------------|------|
| 200 OK           | 成功（通常のレスポンス） |
| 201 Created      | 新規リソース作成成功 |
| 400 Bad Request  | 要求の形式に問題 |
| 404 Not Found    | リソースが見つからない |
| 500 Internal Server Error | サーバーエラー |

## JSONのやり取り方法
REST APIでは、データのフォーマットとしてJSONが一般的です。JavaScript側では、`JSON.parse()`で受信データをオブジェクト化、`JSON.stringify()`で送信データを文字列化します。

```javascript
// JSONの変換例
const jsonString = '{"name":"Alice","age":30}';
const obj = JSON.parse(jsonString); // { name: "Alice", age: 30 }
const newJson = JSON.stringify(obj); // "{"name":"Alice","age":30}"
```

## fetchとaxiosの使い分け
### fetch API（ブラウザ内蔵）
- 利点: インストール不要、ブラウザで直接使える
- 例: GETリクエスト
```javascript
async function getData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) throw new Error('ネットワークエラー');
    return await response.json();
  } catch (error) {
    console.error(error);
  }
}
```

### axios（外部ライブラリ）
- 利点: エラーハンドリングが直感的、JSONデータの自動パース
- 例: POSTリクエスト
```javascript
import axios from 'axios';

async function postData() {
  try {
    const response = await axios.post('https://api.example.com/data', { key: 'value' });
    console.log(response.data);
  } catch (error) {
    console.error('リクエスト失敗:', error.response?.data || error.message);
  }
}
```

## まとめ
REST APIを理解するには、HTTPメソッドとステータスコードの意味を知ることが必須です。フロントエンドからのリクエストは`fetch`や`axios`で簡単に実装でき、JSON形式でデータをやり取りすることが標準です。実際の開発では、公式ドキュメントを参照しながら試行錯誤することが効果的です。特にエラーハンドリング（ステータスコード4xx/5xxの対応）に注意が必要です。
