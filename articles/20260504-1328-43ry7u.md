## 02 背景と前提条件
### 2‑1. REST の位置づけ
REST（Representational State Transfer）は、URL で資源を表現し、HTTP メソッドで状態遷移を行う設計思想です。クライアントとサーバーが **Stateless**（状態依存しない）でやり取りすることで、スケーラビリティやキャッシュが容易になり、異なるシステム間の連携がシンプルになります。

### 2‑2. API 叩きが必要な理由
フロントエンドは UI を担当し、バックエンドはデータや業務ロジックを提供することが主流です。API がなければ、フロントエンドはデータを直接保持することになり、保守性が低下します。したがって、API の設計と呼び出し方法を理解しておくことは、どのエンジニアにとっても必須のスキルです。
