# VS Code で開発効率を上げる！おすすめ拡張機能と設定集

## はじめに

Visual Studio Code（以下 VS Code）は、軽量かつ拡張性の高いエディタとして、多くの開発者に愛用されています。特に駆け出しエンジニアにとっては、適切な拡張機能と設定を導入することで、開発効率が劇的に向上します。

本記事では、実務で役立つ拡張機能の選び方から、効率的なキーバインド設定、設定ファイルの共有方法まで、幅広く解説します。これらの知識を身につけることで、あなたの開発ライフがより快適になること間違いなしです。

## 開発効率を上げる必須拡張機能

### 言語別必須拡張機能

#### JavaScript/TypeScript開発者向け

```json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "bradlc.vscode-tailwindcss",
    "ms-vscode.vscode-typescript-next",
    "ms-vscode.vscode-json"
  ]
}
```

- **ESLint**: コードの品質チェックと自動修正
- **Prettier**: コードフォーマッターで統一感のあるコードを実現
- **Tailwind CSS**: Tailwind CSSの IntelliSense を提供

#### Python開発者向け

```json
{
  "recommendations": [
    "ms-python.python",
    "ms-python.vscode-pylance",
    "ms-python.black-formatter",
    "ms-python.flake8",
    "njpwerner.autodocstring"
  ]
}
```

- **Python**: 公式拡張機能で基本的な機能を提供
- **Pylance**: 高度な言語サポートと型チェック
- **Black Formatter**: Python公式のコードフォーマッター

#### フロントエンド開発者向け

```json
{
  "recommendations": [
    "mrmlnc.vscode-scss",
    "bradlc.vscode-tailwindcss",
    "ms-vscode.vscode-typescript-next",

}
```

- **SCSS IntelliSense**: SCSSの変数やmixinの補完
- **Live Sass Compiler**: Sassファイルをリアルタイムでコンパイル

### 設定の共有方法

拡張機能の設定は `settings.json` で管理します。設定を共有する際は、以下のようにまとめると便利です。

```json
// .vscode/settings.json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

## 効率的なキーバインド設定

### 基本的なショートカット

VS Codeには多くのショートカットが用意されていますが、特に覚えておきたいものを紹介します。

```json
// keybindings.json
[
  {
    "key": "ctrl+shift+p",
    "command": "editor.action.triggerSuggest",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+d",
    "command": "editor.action.deleteLines",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+shift+k",
    "command": "editor.action.copyLinesDownAction",
    "when": "editorTextFocus"
  }
]
```

### カスタムキーバインドの設定

よく使うコマンドを自分好みのショートカットに割り当てることで、作業効率が大幅に向上します。

```json
// カスタムキーバインド例
[
  {
    "key": "ctrl+shift+r",
    "command": "editor.action.formatDocument",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+shift+f",
    "command": "workbench.action.terminal.toggleTerminal"
  }
]
```

### ショートカットの確認と変更方法

1. **ショートカットの確認**: `Ctrl+K Ctrl+S` でキーボードショートカットを表示
2. **ショートカットの変更**: 歯車アイコンから「キーバインドの変更」を選択
3. **JSONでの編集**: 詳細な設定は `keybindings.json` を直接編集

## 設定ファイルの共有と管理

### 設定ファイルの構成

VS Codeの設定は複数のファイルに分かれています。

```bash
.vscode/
├── extensions.json    # 拡張機能の推奨
├── settings.json      # エディタ設定
└── tasks.json         # タスク設定
```

### チームでの設定共有

チームで統一した開発環境を整えるために、以下の方法が有効です。

1. **プロジェクトルートに.vscodeフォルダを作成**
2. **settings.jsonとextensions.jsonを配置**
3. **Gitで管理して共有**

```json
// .vscode/extensions.json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "ms-vscode.vscode-typescript-next"
  ]
}
```

### 設定のエクスポートとインポート

個人の設定を移行する際は、以下の手順で行います。

1. **設定のエクスポート**: `Settings Sync` 拡張機能を使用
2. **手動での移行**: 設定ファイルをコピーして移行

```bash
# 設定ファイルのバックアップ例
cp ~/Library/Application\ Support/Code/User/settings.json ./vscode_settings_backup.json
```
