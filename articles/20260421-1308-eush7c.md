## 1. はじめに
Web アプリケーションやモバイルアプリでデータを取得・更新する際に最も一般的に使われるのが REST API です。REST（Representational State Transfer）は、HTTP を基盤とした軽量なアーキテクチャで、リソースを URI で表現し、HTTP メソッドで操作を行います。本記事では、初心者がすぐに実務で活かせるように、HTTP メソッドとステータスコード、JSON のやり取り、そしてフロントエンドから `fetch` や `axios` で API を呼び出すまでの流れを具体例とともに解説します。

## 2. HTTP メソッドとステータスコードの基礎
REST API では主に 4 つの HTTP メソッドが使われます。

| メソッド | 用途 | 典型的なステータスコード |
|----------|------|--------------------------|
| GET      | リソースの取得 | 200 OK（取得成功）、404 Not Found（存在しない） |
| POST     | 新規リソースの作成 | 201 Created（作成成功）、400 Bad Request（リクエストエラー） |
| PUT/PATCH| リソースの更新 | 200 OK（更新成功）、400 Bad Request（バリデーション失敗） |
| DELETE   | リソースの削除 | 204 No Content（削除成功）、404 Not Found（対象なし） |

ステータスコードはクライアントに応答の結果を伝える重要な情報です。コード 2xx は成功、3xx はリダイレクト、4xx はクライアントエラー、5xx はサーバーエラーを示します。実装時は必ず適切なコードを返すように設計し、フロント側でもエラーハンドリングを漏れなく行う必要があります。

## 3. JSON でのデータ交換
REST API のリクエストボディやレスポンスは基本的に JSON 形式でやり取りされます。JSON は軽量でJavaScript と相性が良いため、フロントエンドから直接送受信しやすいです。リクエストヘッダーで `Content-Type: application/json`、レスポンスヘッダーで `Content-Type: application/json; charset=utf-8` を設定するのが一般的です。

```json
{
  "id": 1,
  "title": "サンプル投稿",
  "content": "こんにちは！",
  "userId": 10
}
```

サーバー側は受け取った JSON をパースし、バリデーションや業務ロジックを実行した後、適切なステータスコードと JSON レスポンスを返します。

## 4. フロントエンドで API を呼び出す例
### 4-1. fetch を使うケース
```javascript
// GET リクエストの例
fetch('https://api.example.com/posts/1', {
  method: 'GET',
  headers: {
    'Accept': 'application/json'
  }
})
.then(response => {
  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }
  return response.json(); // JSON パース
})
.then(data => console.log('取得結果:', data))
.catch(err => console.error('エラー:', err));
```

### 4-2. axios を使うケース
```javascript
import axios from 'axios';

// POST リクエストの例axios.post('https://api.example.com/posts', {
  title: '新規投稿',
  content: 'axios で送信',
  userId: 2
}, {
  headers: { 'Content-Type': 'application/json' }
})
.then(res => {
  console.log('作成成功:', res.status); // 201
  console.log('作成されたオブジェクト:', res.data);
})
.catch(err => {
  if (err.response) {
    // サーバーが 4xx/5xx を返した場合
    console.error('サーバーエラー:', err.response.status);
  } else {
    console.error('リクエストエラー:', err.message);
  }
});
```

上記の例では、`fetch` はブラウザに最初から用意された API で、プロミスベースで非同期処理を行います。`axios` は Promise をラップし、エラーハンドリングやタイムアウト設定が豊富なため、実務ではよく利用されます。

## 5. エラーハンドリングと注意点
1. **ステータスコードのチェック**  
   `fetch` では `response.ok`（200〜299）で成功とみなすか判定します。それ以外の場合は `response.json()` でエラーメッセージを取得し、適切に処理します。

2. **CORS の設定**  
   フロントとバックが別ドメインの場合、サーバー側で `Access-Control-Allow-Origin` ヘッダーを設定しないとブラウザにリクエストがブロックされます。

3. **トークン認証**  
   API が認証を必要とする場合は、`Authorization: Bearer <token>` ヘッダーを付与します。トークンの有効期限やリフレッシュ戦略を事前に設計してください。

4. **レートリミット**  
   大量アクセスが予想される場合は、サーバー側でレートリミットを設け、クライアント側でもリトライロジックを実装すると良いでしょう。

## 6. まとめ
- REST API は HTTP メソッドとステータスコードで操作を表現し、JSON でデータ交換を行うのが基本です。  
- フロントエンドでは `fetch` や `axios` で API を呼び出し、レスポンスを JSON にパースして UI に反映させます。  - エラーハンドリング、CORS、認証、レートリミットなど、実務で必要な注意点をしっかり把握して実装すれば、初心者でも安全に API を活用できます。  

このように、REST API の基本概念から実装例までをステップバイステップで理解すれば、フロントエンドエンジニアとしてのスキルが確実に向上します。
