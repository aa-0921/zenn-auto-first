Git の基本と現場でよく使うコマンド

## Git の基本と実務で役立つコマンド

Git はバージョン管理システムの基礎であり、開発プロセスを円滑にするための不可欠なツールです。初心者が押さえておくべきポイントは、コマンドの流れと実務での応用です。この記事では、clone / pull / branch / commit / push の基本流れから、コンフリクト時の対処までを解説します。

### 1. Git のインストールと初期設定

Git をインストールし、ユーザ名とメールアドレスを設定します。これが、コミット時に記録される基本情報です。

```bash
# インストール（例: macOS Homebrew）
brew install git

# 初期設定
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

### 2. リポジトリの取得と更新

現場では、既存のリポジトリをローカル環境に取得したり、サーバー上の変更を取り込む必要があります。

```bash
# リポジトリのクローン
git clone https://github.com/user/repo.git

# 最新の変更をプル
git pull origin main
```

`git pull` は `git fetch` と `git merge` の組み合わせです。特にブランチ名が指定されていない場合は、デフォルトブランチ（通常 `main` または `master`）に対する更新が適用されます。

### 3. ブランチの管理と切り替え

ブランチは機能開発や修正を独立させたり、リリース管理を容易にします。

```bash
# ブランチの作成
git branch new-feature

# 作成したブランチに切り替え
git checkout new-feature

# 作成と切り替えを同時に行う
git checkout -b new-feature

# ブランチの一覧と現在のブランチを表示
git branch -a
```

ブランチを切り替える際は、現在の変更がコミットされていないと切り替えができないことに注意してください。作業中の変更を一時的に退避させるには `git stash` コマンドを使います。

### 4. 変更の追跡とコミット

ローカルで作成したファイルや変更を追跡し、コミットしてリポジトリに登録します。

```bash
# 変更をステージングエリアに追加
git add file.txt

# コミット
git commit -m "Fix typo in README"

# 変更をステージングエリアから外す
git reset file.txt
```

コミットメッセージは、変更の内容を明確に伝えることが重要です。例えば「Fix login bug」よりも「Fix NullPointerException in UserLoginService.java when password is null」が適切です。

### 5. 変更の共有と更新

ローカルでの作業をサーバー上のリポジトリに反映させたり、他の開発者の変更を取り込むための `push` と `pull` コマンドを理解します。

```bash
# ローカルのコミットをリモートブランチに送信
git push origin new-feature

# リモートブランチから最新の変更をプル
git pull origin main
```

`git push` 時にリモートブランチが存在しない場合、エラーが発生します。この場合、`git push --set-upstream origin new-feature` でリモートブランチを作成し、同時にプッシュできます。

### 6. コンフリクトの解決

複数の開発者が同じファイルの同じ部分を変更した場合、コンフリクトが発生します。この時の対処法を理解することが重要です。

```bash
# コンフリクトが発生したファイルの一覧を表示
git status

# コンフリクトを解決
git mergetool

# コンフリクトを手動で解決
# ファイルに <<<<<<<, ======, >>>>>>> のマークが追加される
# 手動で変更後、git add と git commit を実行
```

コンフリクトの解決は、編集ソフト（VS Code、Sublime Text など）のマージツールを使うと効率的です。特に、ファイルの変更箇所を視覚的に比較できるツールが推奨されます。

### 7. 履歴の確認と復元

変更履歴を確認したり、過去の状態に復元するコマンドを知っておきます。

```bash
# コミット履歴の表示
git log

# 過去のコミットに戻る（HEAD を移動）
git checkout 9e3e2a7

# 過去の状態に復元（ファイルの変更を元に戻す）
git checkout -- file.txt

# 特定のファイルの履歴を表示
git log -- file.txt
```

`git revert` コマンドを使うと、過去のコミットを元に戻すことができます。このコマンドは、コミットを破壊しない復元方法です。

### 8. テーマブランチとリベース

機能開発を進める際のテーマブランチ（topic branch）の作成と、リベースによる変更の統合方法を説明します。

```bash
# リベースによる統合
git checkout main
git pull origin main
git checkout new-feature
git rebase main
```

リベースは、履歴を線形に保つことができますが、既存のリポジトリへのプッシュは、`git push -f` で強制プッシュする必要があります。この操作は慎重に行う必要があります。

### 9. 設定と拡張

Git の設定をカスタマイズし、拡張機能を使って作業効率を向上させる方法を紹介します。

```bash
# 設定の編集
git config --global core.editor "code --wait"

# 拡張機能のインストール
brew install git-extras
```

`git-extras` や `hub` などの拡張機能を使うと、`git log` の表示形式や `git create` コマンドなどの便利なコマンドを利用できます。

### 10. ベストプラクティス

最後に、Git を使う上でのベストプラクティスをまとめます。

- 小さな単位でコミットする
- コミットメッセージは英語でも日本語でも、一貫性をもって書く
- リモートブランチを常に最新に保つ
- コンフリクトは早めに解決する
- テストを通った後にコミットする

Git は、開発者が効率的にコードを管理するための強力なツールです。この記事で紹介したコマンドとポイントを実践すれば、Git を使った開発作業をよりスムーズに行えるようになります。
