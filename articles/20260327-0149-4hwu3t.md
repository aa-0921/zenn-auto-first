# JavaScript 非同期処理の基礎を学ぼう：Promise と async/await でコールバック地獄から脱出する

JavaScript は、ブラウザや Node.js などの環境で動作するシングルスレッドの言語です。そのため、同期処理（直列実行）のコードが長くなった場合、他の処理が待たされる「ブロッキング」問題が発生します。非同期処理は、これを解決するための重要な概念です。この記事では、JavaScript の非同期処理の基礎を学び、コールバック地獄からの脱出方法を解説します。

## 非同期処理の問題点と Promise の登場

JavaScript でファイル読み込みや API 呼び出しなどの処理を同期的に実行すると、その処理が完了するまで次のコードが実行されません。例えば、以下のコードでは、`console.log('Done')` は、`readFile` が完了するまで実行されません。

```javascript
const fs = require('fs');
const data = fs.readFileSync('file.txt'); // 同期読み込み
console.log(data); // ファイル内容
console.log('Done'); // 実行される
```

しかし、実際のアプリケーションでは、ファイル読み込みやネットワークアクセスなどの非同期処理が必要です。非同期処理では、コールバック関数を引数として渡して、処理が完了したタイミングで実行されます。

```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) throw err;
  console.log(data);
});
console.log('Done'); // 非同期処理が完了する前に実行される
```

このコードでは、`console.log('Done')` が先に実行され、ファイル読み込みが完了してから `console.log(data)` が実行されます。しかし、複数の非同期処理を連続して実行する場合、コールバック地獄（Callback Hell）という問題が発生します。

```javascript
fs.readFile('file1.txt', (err1, data1) => {
  if (err1) throw err1;
  fs.readFile('file2.txt', (err2, data2) => {
    if (err2) throw err2;
    fs.readFile('file3.txt', (err3, data3) => {
      if (err3) throw err3;
      // 処理
    });
  });
});
```

コールバック地獄では、コードの可読性やメンテナンス性が大幅に低下します。この問題を解決するために、Promise が登場しました。

## Promise の基本と then によるチェーン

Promise は、非同期処理の結果を抽象化するオブジェクトです。Promise には、3つの状態があります：Pending（待機中）、Fulfilled（成功）、Rejected（失敗）。Promise は、`.then()` メソッドを使って、成功時の処理と失敗時の処理を定義できます。

```javascript
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Hello, Promise!');
  }, 1000);
});

promise.then((result) => {
  console.log(result); // 'Hello, Promise!'
}).catch((error) => {
  console.error(error);
});
```

`.then()` は、Promise が Fulfilled 状態になった場合に実行される処理を定義します。`.catch()` は、Promise が Rejected 状態になった場合に実行される処理を定義します。複数の非同期処理を連続して実行するには、`.then()` をチェーン化します。

```javascript
const fetchUser = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve({ id: 1, name: 'Alice' });
    }, 500);
  });
};

fetchUser()
  .then(user => {
    console.log(user.name); // 'Alice'
    return user.id; // 次の then に値を渡す
  })
  .then(id => {
    console.log(id); // 1
    return id * 2;
  })
  .then(result => {
    console.log(result); // 2
  });
```

Promise のチェーンは、コールバック地獄よりも読みやすくなります。しかし、Promise の構文は少し冗長で、エラーハンドリングが煩雑な場合があります。

## async/await による非同期処理の簡略化

async/await は、Promise ベースの非同期処理を、同期的なコードのように記述できる構文です。async キーワードは関数の前に付けると、その関数は Promise を返します。await キーワードは、Promise が解決されるまで待機し、解決した値を変数に代入します。

```javascript
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('エラー:', error);
  }
}

fetchData().then(data => {
  console.log(data);
});
```

async/await では、エラーハンドリングが try/catch で簡潔に行えます。また、Promise チェーンよりもコードが読みやすくなります。しかし、async/await は Promise の上に構築されているため、Promise の概念を理解することが重要です。

## then と async/await の使い分け

then と async/await は、Promise ベースの非同期処理を記述する方法です。then は、Promise オブジェクトに対して直接呼び出すことができ、async/await は関数の内部で使う構文です。使い分けは、次のとおりです：

- **then を使う場合**：Promise オブジェクトを直接操作する必要がある場合（例：複数の Promise をマージする）。
- **async/await を使う場合**：関数の中で非同期処理を記述したい場合（例：API 呼び出しの処理）。

```javascript
// then を使った例
Promise.all([fetchData1(), fetchData2()])
  .then(results => {
    console.log(results);
  });

// async/await を使った例
async function fetchAllData() {
  const [data1, data2] = await Promise.all([fetchData1(), fetchData2()]);
  console.log(data1, data2);
}
```

## 実践的なコード例：API 呼び出しとデータ処理

次のコードでは、async/await を使って、API からデータを取得し、そのデータを加工して表示する処理を示します。

```javascript
// データ加工関数
const processData = (data) => {
  return data.map(item => ({ ...item, processed: true }));
};

// API 呼び出し関数
const fetchData = async (url) => {
  try {
    const response = await fetch(url);
    if (!response.ok) throw new Error('HTTP エラー');
    return await response.json();
  } catch (error) {
    console.error('API エラー:', error);
    throw error; // 再スロー
  }
};

// メイン処理
async function main() {
  try {
    const url = 'https://api.example.com/data';
    const rawData = await fetchData(url);
    const processedData = await processData(rawData);
    console.log(processedData);
  } catch (error) {
    console.error('メイン処理エラー:', error);
  }
}

main();
```

このコードでは、API 呼び出しのエラーハンドリング、データ加工の処理、そしてメイン処理のエラーハンドリングを一貫して行っています。async/await と try/catch を使うことで、非同期処理のエラーハンドリングを同期的なコードのように記述できます。

## まとめ

JavaScript の非同期処理は、同期処理よりも複雑ですが、Promise と async/await を理解することで、コールバック地獄からの脱出が可能です。Promise は、非同期処理の結果を抽象化し、then と catch で成功時と失敗時の処理を定義できます。async/await は、Promise ベースの非同期処理を、同期的なコードのように記述できる構文です。then と async/await は、使い分けが重要で、実務で役立つ知見を得るために、これらの概念を理解し、コード例を実践する必要があります。
