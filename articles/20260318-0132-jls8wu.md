# VS Code で入れておきたい拡張機能と設定

## はじめに

VS Code は軽量かつ高機能なエディタとして、多くの開発者に愛用されています。しかし、そのまま使うだけでは開発効率はあまり上がりません。適切な拡張機能と設定を導入することで、開発スピードと快適性が劇的に向上します。

この記事では、駆け出しエンジニアが押さえておくべき VS Code の拡張機能と設定について、実践的な観点から解説します。具体的な拡張機能の紹介、設定ファイルのカスタマイズ方法、キーバインドの最適化、設定の共有方法まで網羅的に説明します。

## 必須の拡張機能

### 基本的な拡張機能

まずはどの開発者にも共通しておすすめしたい拡張機能から紹介します。

**1. GitLens**
Git の履歴を視覚化し、コードの変更履歴を簡単に追跡できます。誰がいつどのような変更を加えたかが一目でわかります。

```json
{
  "gitlens.historyExplorer.enabled": true,
  "gitlens.views.repositories.location": "gitlens",
  "gitlens.views.compare.location": "gitlens",
  "gitlens.views.fileHistory.location": "gitlens",
  "gitlens.views.lineHistory.location": "gitlens"
}
```

**2. Prettier**
コードフォーマッターとして必須の拡張機能。一貫したコードスタイルを保ちます。

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}
```

**3. Bracket Pair Colorizer 2**
対応する括弧を色分けして表示し、ネストの深いコードも視覚的にわかりやすくします。

### 言語別の拡張機能

**JavaScript/TypeScript**
- ESLint: コードの静的解析と問題の指摘
- TypeScript Importer: 自動インポート補完
- Auto Rename Tag: HTML/XMLタグの自動リネーム

**Python**
- Python: 公式拡張機能
- Pylance: 型チェックとインテリセンス
- Jupyter: Jupyter Notebookのサポート

**PHP**
- PHP Intelephense: コード補完と解析
- PHP CS Fixer: PHPコードスタイリング

## 設定ファイルのカスタマイズ

### settings.json の設定

VS Code の設定は `settings.json` ファイルで管理します。ユーザ設定はグローバルに、ワークスペース設定はプロジェクトごとに適用できます。

```json
{
  "editor.tabSize": 2,
  "editor.insertSpaces": true,
  "editor.detectIndentation": false,
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "explorer.confirmDelete": false,
  "explorer.confirmDragAndDrop": false,
  "terminal.integrated.shell.windows": "C:\\WINDOWS\\System32\\cmd.exe",
  "workbench.colorTheme": "Monokai",
  "workbench.iconTheme": "material-icon-theme",
  "window.zoomLevel": 0,
  "emmet.includeLanguages": {
    "javascript": "javascriptreact"
  },
  "emmet.triggerExpansionOnTab": true,
  "files.associations": {
    "*.tpl": "html"
  }
}
```

### 重要な設定項目

**インデント関連**
- `editor.tabSize`: タブの幅
- `editor.insertSpaces`: タブをスペースに変換するか
- `editor.detectIndentation`: ファイルからインデントを自動検出

**自動保存**
- `files.autoSave`: 保存タイミング（off, afterDelay, onFocusChange, onWindowChange）
- `files.autoSaveDelay`: 自動保存までの遅延時間（ミリ秒）

**エディタ表示**
- `editor.minimap.enabled`: ミニマップの表示/非表示
- `editor.wordWrap`: 折り返し設定

## キーバインドの最適化

### 基本的なショートカット

VS Code のデフォルトショートカットは非常に豊富ですが、自分の手に馴染むようにカスタマイズすることをおすすめします。

**よく使うショートカット**
- `Ctrl + Shift + P`: コマンドパレットを開く
- `Ctrl + ,`: 設定を開く
- `Ctrl + B`: サイドバーの表示/非表示
- `Ctrl + Shift + N`: 新しいウィンドウを開く
- `Ctrl + Shift + T`: 閉じたタブを復元

### カスタムキーバインド

`keybindings.json` で独自のキーバインドを設定できます。

```json
{
  "key": "ctrl+shift+r",
  "command": "editor.action.rename",
  "when": "editorTextFocus"
},
{
  "key": "ctrl+d",
  "command": "editor.action.deleteLines",
  "when": "editorTextFocus"
},
{
  "key": "ctrl+shift+alt+up",
  "command": "editor.action.copyLinesUpAction",
  "when": "editorTextFocus"
},
{
  "key": "ctrl+shift+alt+down",
  "command": "editor.action.copyLinesDownAction",
  "when": "editorTextFocus"
}
```

### プラグインでの拡張

拡張機能によって追加されるコマンドもキーバインドに割り当てられます。例えば GitLens のコマンドを割り当てることで、Git 操作を高速化できます。

## 設定の共有と管理

### 設定のエクスポート/インポート

設定を複数の環境で共有する方法を紹介します。

**1. 設定のエクスポート**
- 設定エディタから JSON をコピー
- 拡張機能マネージャから拡張機能の一覧をエクスポート

**2. Settings Sync 拡張機能**
- GitHub アカウントで設定を同期
- 複数の端末間で設定を共有可能

```json
// 拡張機能のリストをエクスポート
{
  "extensions": [
    "ms-vscode.vscode-typescript-next",
    "esbenp.prettier-vscode",
    "ms-vscode.vscode-typescript-next",
    "coenraads.bracket-pair-colorizer-2",
    "ms-azuretools.vscode-docker"
  ]
}
```

### バージョン管理での管理

設定ファイルを Git で管理することも有効です。

```bash
# 設定ファイルの追加
git add .vscode/settings.json

# 拡張機能リストの管理
code --list-extensions > extensions.txt
```

## 実践的な設定例

### フロントエンド開発向け設定

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "emmet.includeLanguages": {
    "javascript": "javascriptreact",
    "typescript": "typescriptreact"
  },
  "files.associations": {
    "*.jsx": "javascriptreact",
    "*.tsx": "typescriptreact"
  },
  "workbench.colorCustomizations": {
    "editor.selectionBackground": "#264f78",
    "editor.wordHighlightBackground": "#3a3d41",
    "editor.wordHighlightStrongBackground": "#264f78"
  }
}
```

### バックエンド開発向け設定

```json
{
  "editor.formatOnSave": false,
  "editor.defaultFormatter": "vscode.php",
  "php.suggest.basic": false,
  "php.validate.enable": true,
  "files.exclude": {
    "**/.git": true,
    "**/.svn": true,
    "**/.hg": true,
    "**/CVS": true,
    "**/.DS_Store": true,
    "**/node_modules": true,
    "**/vendor": true
  },
  "search.exclude": {
    "**/node_modules": true,
    "**/bower_components": true,
    "**/dist": true
  }
}
```

## トラブルシューティング

### よくある問題と解決策

**1. 拡張機能が正常に動作しない**
- VS Code を再起動する
- 拡張機能を無効化して再度有効化する
- 拡張機能をアンインストールして再インストールする

**2. 設定が反映されない**
- 設定を変更したら VS Code を再起動する
- ユーザ設定とワークスペース設定の優先順位を確認する
- 設定ファイルの文法エラーをチェックする

**3. パフォーマンスが低下する**
- 使用していない拡張機能を無効化する
- メモリ設定を最適化する
- ワークスペースを整理する

## まとめ

VS Code を最大限に活用するためには、適切な拡張機能と設定が不可欠です。この記事で紹介した内容をベースに、自分の開発スタイルに合わせてカスタマイズしていきましょう。

**重要なポイントのおさらい**
- 必須の拡張機能を導入する
- 設定ファイルを自分の好みにカスタマイズする
- キーバインドを最適化する
- 設定を共有・管理する仕組みを作る

VS Code の学習コストは最初は高く感じるかもしれませんが、一度最適な環境を構築すれば、その恩恵は計り知れません。ぜひこの記事を参考に、自分だけの最強の開発環境を構築してください。
