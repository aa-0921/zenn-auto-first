## 背景・前提
- Rails 7.x（APIモードでも動作確認済み）  
- データベースは PostgreSQL  
- デバッグツールは **Rails console**, **byebug**, **bullet gem**, **rails routes** などを想定  
- 前提として、基本的な MVC の流れや ActiveRecord の使い方は把握しているものとします。

### 1. ルーティングエラー
#### 発生シナリオ
```bash
$ rails routes | grep users
```
```
users GET    /users(.:format)          users#index
```
あるいは、`POST /users` を呼び出した際に `No route matches [POST] "/users"` というエラーが返るケースがあります。

#### 原因チェックリスト
| チェック項目 | 実施コマンド | 期待結果 |
|--------------|--------------|----------|
| ルーティング定義 | `rails routes | grep users` | `users#create` が存在 |
| HTTPメソッド | `curl -X POST http://localhost:3000/users` | 302 または 201 |
| コントローラアクション | `rails console` → `UserController.action(:create)` | `Method` が定義済み |
| パラメータ形式 | `params.permit(:name, :email)` | エラー無く受け取れる |

#### デバッグ手順
1. **routesを確認**  
   ```bash
   rails routes | grep users
   ```
2. **リクエストを再現**  
   ```bash
   curl -v -X POST http://localhost:3000/users -d '{"name":"foo","email":"bar@example.com"}'
   ```
3. **コンソールでアクションが呼ばれるか確認**  
   ```ruby
   UserController.action(:create).call(ActionDispatch::Request.new({}))
   ```
4. **ログレベルを上げる**  
   `config/environments/development.rb`  
   ```ruby
   config.log_level = :debug
   ```

#### 実際の修正例
```ruby
# config/routes.rb
Rails.application.routes.draw do
  resources :users, only: [:index, :create, :show]
end

# app/controllers/users_controller.rb
class UsersController < ApplicationController
  def create
    @user = User.new(user_params)
    if @user.save
      render json: @user, status: :created
    else
      render json: { errors: @user.errors.full_messages }, status: :unprocessable_entity
    end
  end

  private

  def user_params
    params.require(:user).permit(:name, :email)
  end
end
```
> **ポイント**  
> `params.require(:user)` が抜けていると `ActionController::ParameterMissing` になる。

### 2. マイグレーション失敗
#### 発生シナリオ
```bash
$ rails db:migrate
```
```
== 20230920000000 CreateUsers: migrating
   -> 0.0000s
== 20230920000000 CreateUsers: migrated (0.0000s)
```
その後、アプリを起動すると `PG::UndefinedTable: ERROR: relation "users" does not exist` が出る。

#### 原因チェックリスト
| チェック項目 | 実施コマンド | 期待結果 |
|--------------|--------------|----------|
| マイグレーションファイル | `cat db/migrate/20230920000000_create_users.rb` | 正しいテーブル名・カラム定義 |
| ステータス | `rails db:migrate:status` | `up` |
| データベース接続 | `rails console` → `ActiveRecord::Base.connection.tables` | `users` が存在 |
| 版管理 | `git diff HEAD~1 HEAD` | 変更点が確認できる |

#### デバッグ手順
1. **migration ファイルを確認**  
   ```ruby
   create_table :users do |t|
     t.string :name, null: false
     t.timestamps
   end
   ```
2. **`schema.rb` を確認**  
   ```bash
   cat db/schema.rb
   ```
3. **手動でテーブル作成**  
   ```bash
   rails db:rollback STEP=1
   rails db:migrate
   ```
4. **エラー内容を読む**  
   `pg_dump` のエラーメッセージは「relation 'users' does not exist」とあるなら、テーブルが作られていないことを示す。

#### 実際の修正例
```ruby
# db/migrate/20230920000000_create_users.rb
class CreateUsers < ActiveRecord::Migration[7.0]
  def change
    create_table :users do |t|
      t.string :name, null: false
      t.string :email, null: false, index: { unique: true }
      t.timestamps
    end
  end
end

# 変更後に再マイグレーション
rails db:migrate:reset
```
> **ポイント**  
> `null: false` を付け忘れるとデフォルトで `NULL` が許容されるので、後で `NOT NULL` 制約が無いことにより `save!` が失敗し、エラーになる。

### 3. N+1問題
#### 発生シナリオ
```ruby
# app/controllers/posts_controller.rb
def index
  @posts = Post.all
  @posts.each do |post|
    puts post.user.name
  end
end
```
Post が 100 件あると 101 回の SQL が発行される（1 回は `posts` を取得、N 回は `users` を取得）。

#### 原因チェックリスト
| チェック項目 | 実施コマンド | 期待結果 |
|--------------|--------------|----------|
| SQL 実行回数 | `rails console` → `ActiveRecord::Base.logger = Logger.new(STDOUT)` | 1 回のみ |
| eager loading | `includes` / `preload` / `eager_load` | N+1 が解消 |
| gem bullet | `bundle exec rails server` | N+1 警告が出る |

#### デバッグ手順
1. **バルクログを確認**  
   ```bash
   rails console
   ActiveRecord::Base.logger = Logger.new(STDOUT)
   Post.includes(:user).all.each { |p| p.user.name }
   ```
2. **Bullet gem を導入**  
   ```ruby
   # Gemfile
   gem 'bullet', group: :development
   ```
   ```ruby
   # config/environments/development.rb
   config.after_initialize do
     Bullet.enable = true
     Bullet.alert = true
     Bullet.bullet_logger = true
   end
   ```
3. **ルーティングで N+1 を検知**  
   ```bash
   rails server
   # ブラウザでアクセスするとアラートが出る
   ```

#### 実際の修正例
```ruby
# before
@posts = Post.all

# after
@posts = Post.includes(:user).all
```
> **ポイント**  
> `includes` は `LEFT OUTER JOIN` を使って一括取得するので、クエリ回数を減らせる。

## まとめ
- **ルーティングエラー**は `rails routes` で定義確認、パラメータ形式をチェック。  
- **マイグレーション失敗**は `schema.rb` と `migration` ファイルを比較し、`null: false` などの制約を忘れない。  
- **N+1問題**は `includes` を使い、Bullet gem で可視化。  

実務でエラーが発生した際は、まず「どの層で失敗しているか」を特定し、上記チェックリストに沿って原因を絞り込むことが重要です。これらのテクニックを習得すれば、開発のスピードと品質を大幅に向上できます。
