## Gitを使う前に押さえておきたい背景と前提
解説に入る前に、Git の基本的な前提をおさらいします。Git には「リモートリポジトリ」と「ローカルリポジトリ」の2種類が存在します。

- **リモートリポジトリ**: GitHub や GitLab などのサーバー上にあり、チームメンバー全員で共有する場所
- **ローカルリポジトリ**: 自分の PC 上にあり、自由に実験や編集ができる場所

チーム開発では、基本的に「リモートから最新コードを取ってきて（pull）→ ローカルで作業（branch/commit）→ リモートに反映（push）」というサイクルを回します。このサイクルさえ理解できれば、Git の全体像の 8 割を把握したも同然です。

## 現場で必須の基本コマンド操作フロー
それでは、実際の開発現場を想定した一連のフローを見ていきましょう。

### 1. リポジトリを取得する：git clone
まずはチームのリモートリポジトリを自分の PC にコピーします。これを「クローン」と呼びます。

```bash
# リモートリポジトリのURLを指定して取得
git clone https://github.com/your-team/sample-project.git

# 取得したディレクトリに移動
cd sample-project
```

### 2. 最新状態を取り込む：git pull
クローン直後や、他のメンバーが更新を反映した後は、必ず最新状態を取り込みます。

```bash
# 現在のブランチの最新変更をローカルに反映
git pull origin main
```

### 3. 作業ブランチを切る：git branch と git switch
メインブランチ（main や master）を直接書き換えるのはチーム開発の禁忌です。必ず作業用のブランチを作成し、そこでコードを書きます。

```bash
# 新しいブランチを作成して切り替える（Git 2.23以降推奨）
git switch -c feature/login-button

# ブランチを作成したことを確認
git branch
```

### 4. 変更を記録する：git add と git commit
ファイルを編集したら、変更内容をローカルリポジトリの履歴として記録（コミット）します。

```bash
# 変更したファイルをステージングエリアに追加
git add src/components/Button.tsx

# 変更内容をコミット（分かりやすいメッセージを添える）
git commit -m "ログインボタンのスタイルを修正"
```

### 5. 共有する：git push
ローカルで行ったコミットをリモートリポジトリに送信し、チームメンバーと共有します。

```bash
# 自身の作業ブランチをリモートに push
git push origin feature/login-button
```

これで、後は GitHub 上でプルリクエスト（マージリクエスト）を作成すればレビュー工程に進めます。

## コンフリクトが起きたときの対処法
複数人で同じファイルの近い場所を編集すると、`git pull` や `git merge` の際に「コンフリクト（競合）」が発生します。コンフリクトはエラーではなく「人の手で解決してください」という Git からの通知です。

### コンフリクトの発生原因と表示の読み方
例えば、同じ `app.js` の 10 行目を自分と同僚が別々に書き換えたとします。`git pull` した際、以下のような表示が出ます。

```text
Auto-merging app.js
CONFLICT (content): Merge conflict in app.js
Automatic merge failed; fix conflicts and then commit the result.
```

該当ファイルを開くと、以下のようなマーカーが挿入されています。

```javascript
<<<<<<< HEAD
console.log("自分が書いた処理");
=======
console.log("同僚が書いた処理");
>>>>>>> feature/other-branch
```

- `<<<<<<< HEAD` から `=======` までが「自分の変更」
- `=======` から `>>>>>>>` までが「相手の変更」

### 実際の解消手順とコマンド
1. エディタでファイルを開き、マーカーごと不要なコードを削除し、正しい状態に手動で編集します。
2. マーカーを完全に消したら保存し、ステージングとコミットを行います。

```bash
# コンフリクトを解消したファイルを追加
git add app.js

# マージコミットとして記録
git commit -m
