## HTTPメソッドとステータスコードの理解

REST APIでは、HTTPプロトコルに基づいてリクエストとレスポンスが送受信されます。HTTPメソッド（GET, POST, PUT, DELETEなど）はリクエストの目的を表し、ステータスコード（200, 404, 500など）はリクエストの結果を示します。

### 主なHTTPメソッド
- **GET**: リソースを取得する（例: `/users`からユーザーデータを取得）
- **POST**: リソースを作成する（例: 新規ユーザー登録）
- **PUT/PATCH**: リソースを更新する（全体を更新するPUT vs 部分更新するPATCH）
- **DELETE**: リソースを削除する

### よく使うステータスコード
- **200 OK**: 成功
- **201 Created**: リソースが新規作成された
- **400 Bad Request**: リクエスト形式が不正
- **404 Not Found**: リソースが見つからない
- **500 Internal Server Error**: サーバー内部エラー

## JSONのやり取り方法

APIは通常、JSON形式でデータをやり取りします。JSONはキーと値のペアで構造化されたデータ形式で、JavaScriptとの連携が容易です。

### 例: JSONリクエストデータ
```json
{
  "name": "山田太郎",
  "email": "taro@example.com"
}
```

### 例: JSONレスポンスデータ
```json
{
  "id": 1,
  "name": "山田太郎",
  "createdAt": "2023-01-01T00:00:00Z"
}
```

## fetchとaxiosでAPIを叩く

フロントエンドからAPIを呼び出す際は、JavaScriptの`fetch`APIまたはライブラリ`axios`を使います。両者は類似機能を提供しますが、`axios`はPromiseベースのAPIが充実しています。

### 1. fetchを使ったGETリクエスト
```javascript
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    if (!response.ok) throw new Error('ネットワークエラー');
    const data = await response.json();
    console.log(data); // {id: 1, name: '山田太郎'}
  } catch (error) {
    console.error(error);
  }
}
```

### 2. fetchを使ったPOSTリクエスト
```javascript
async function createUser(userData) {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(userData)
  });
  const data = await response.json();
  return data;
}
```

### 3. axiosを使ったGETリクエスト
```javascript
import axios from 'axios';

async function getUsersWithAxios() {
  try {
    const response = await axios.get('https://api.example.com/users');
    console.log(response.data);
  } catch (error) {
    if (error.response) {
      // サーバーが4xx/5xxエラーを返した場合
      console.error('サーバーエラー:', error.response.status);
    } else {
      console.error('ネットワークエラー');
    }
  }
}
```

### 4. axiosを使ったPOSTリクエスト
```javascript
async function createUserWithAxios(userData) {
  try {
    const response = await axios.post('https://api.example.com/users', userData);
    return response.data;
  } catch (error) {
    console.error('エラー:', error.message);
  }
}
```

## まとめ

REST APIを効率的に利用するためには、HTTPメソッドとステータスコードの理解、JSONのデータ構造の把握が重要です。フロントエンドからの呼び出しでは、`fetch`や`axios`を使い、非同期処理を`async/await`で管理するのが一般的です。本記事で解説した流れを踏まえ、実際のプロジェクトでAPIを実装してみると、エラー対応やデータ検証の実装も意識する必要があります。さらに、APIドキュメントの確認やCORS設定の理解も、実務で役立つスキルです。
