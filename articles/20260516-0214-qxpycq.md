## REST APIの基本概念とHTTPメソッド

REST APIは、リソースをURIで表し、HTTPのメソッドを使ってそのリソースに対して行う操作を定義するアーキテクチャスタイルです。最もよく知られるHTTPメソッドは次の4つです。

### GETメソッド

GETは、リソースの取得を意図しています。例えば、ユーザー一覧を取得する場合:

```javascript
// GETリクエストの例
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data));
```

### POSTメソッド

POSTは、リソースの作成に使用します。新しいユーザーを作成する例:

```javascript
// POSTリクエストの例
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: '山田太郎',
    email: 'yamada@example.com'
  })
})
.then(response => response.json())
.then(data => console.log('作成されたユーザー:', data));
```

### PUTメソッド

PUTは、リソースの更新に使用します。既存のユーザー情報を更新する例:

```javascript
// PUTリクエストの例
fetch('https://api.example.com/users/123', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: '山田更新太郎',
    email: 'updated@example.com'
  })
})
.then(response => response.json())
.then(data => console.log('更新後のユーザー:', data));
```

### DELETEメソッド

DELETEは、リソースの削除に使用します。

```javascript
// DELETEリクエストの例
fetch('https://api.example.com/users/123', {
  method: 'DELETE'
})
.then(response => {
  if (response.ok) {
    console.log('削除に成功しました');
  }
});
```

## HTTPステータスコードとJSONのやり取り

APIとの通信では、リクエストが成功したか失敗したかを示すステータスコードが返されます。よく使われるステータスコードをいくつか紹介します。

### 成功を示すステータスコード

- **200 OK**: リクエストが成功したことを示します
- **201 Created**: リソースが正常に作成されたことを示します
- **204 No Content**: リクエストが成功し、返すコンテンツがないことを示します

### クライアントエラーを示すステータスコード

- **400 Bad Request**: リクエストが不正であることを示します
- **401 Unauthorized**: 認証が必要であることを示します
- **404 Not Found**: 要求したリソースが見つからないことを示します

### サーバーエラーを示すステータスコード

- **500 Internal Server Error**: サーバー内部でエラーが occurredしたことを示します

APIからの応答は、基本的にJSON形式で返されます。フロントエンドでJSONを扱う際は、常に適切にパースすることが重要です。

```javascript
// JSONのパースとエラーハンドリング
fetch('https://api.example.com/users')
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return response.json();
  })
  .then(data => {
    console.log('取得したデータ:', data);
  })
  .catch(error => {
    console.error('エラーが発生しました:', error);
  });
```

## fetchとaxiosを使ったAPI呼び出しの比較

フロントエンドでAPIを呼び出す際、主に使用されるのは`fetch` APIと`axios`です。両方の使い方を比較してみましょう。

### fetch APIの基本的な使い方

`fetch`はブラウザネイティブのAPIで、追加のライブラリなしで使用できます。

```javascript
// GETリクエスト
async function fetchUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    if (!response.ok) {
      throw new Error('ネットワークレスポンスが正しくありませんでした');
    }
    const users = await response.json();
    return users;
  } catch (error) {
    console.error('フェッチエラー:', error);
  }
}

// POSTリクエスト
async function createUser(userData) {
  try {
    const response = await fetch('https://api.example.com/users', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData)
    });
    return await response.json();
  } catch (error) {
    console.error('ユーザー作成エラー:', error);
  }
}
```

### axiosの使い方

`axios`は人気のあるHTTPクライアントライブラリで、より多くの機能と使いやすいAPIを提供します。

```bash
npm install axios
```

```javascript
import axios from 'axios';

// GETリクエスト
async function fetchUsers() {
  try {
    const response = await axios.get('https://api.example.com/users');
    return response.data;
  } catch (error) {
    console.error('axiosエラー:', error);
  }
}

// POSTリクエスト
async function createUser(userData) {
  try {
    const response = await axios.post('https://api.example.com/users', userData);
    return response.data;
  } catch (error) {
    console.error('ユーザー作成エラー:', error);
  }
}

// デフォルトヘッダーの設定
axios.defaults.baseURL = 'https://api.example.com';
axios.defaults.headers.common['Authorization'] = 'Bearer your-token';
axios.defaults.headers.post['Content-Type'] = 'application/json';

// インターセプターでリクエスト/レスポンスの加工
axios.interceptors.request.use(config => {
  // リクエスト送信前の処理
  console.log('リクエスト送信:', config.url);
  return config;
});

axios.interceptors.response.use(response => {
  // レスポンス受信後の処理
  console.log('レスポンス受信:', response.status);
  return response;
});
```

### fetch vs axiosのメリット・デメリット

| 項目 | fetch | axios |
|------|-------|-------|
| 追加依存関係 | 不要 | 必要 |
| ブラウザ対応 | モダンブラウザ | 全ブラウザ |
| エラーハンドリング | 手動で実装 | 自動でキャッチ |
| タイムアウト | 制御不可 | 簡単に設定可 |
| インターセプター | なし | あり |

## まとめ

この記事では、REST APIの基本からフロントエンドでの呼び出し方法までを網羅的に解説しました。HTTPメソッド（GET、POST、PUT、DELETE）を正しく使い、ステータスコードを適切に解釈すること、そしてJSONデータのやり取りをスムーズに行うこと—これらはフロントエンド開発者にとって必須のスキルです。

fetchとaxiosの両方を理解し、状況に応じて使い分けることが重要です。fetchは lightweight でシンプルながらも、 axios は機能-rich で開発がしやすいという特徴があります。

実践での開発では、API通信のロジックを汚さないように、ラッパー関数を作成したり、状態管理システムと組み合わせたりするなど、良い設計を心がけましょう。API呼び出しのベストプラクティスを確固たものだけではなく、チーム内での共有も忘れずに行いましょう。
