# JavaScriptの非同期処理のマスター

### 背景と前提

現代のWeb開発では、JavaScriptが動的なインタラクションを支える重要な役割を果たしています。特に、ユーザーからの入力や外部APIとの連携が頻繁に行われる環境では、非同期処理が不可欠です。しかし、JavaScriptの非同期処理は「コールバック地獄」と呼ばれる問題を引き起こすこともあります。

**Promise / async/await** の登場で、この難題に挑戦する方法が明確になってきました。本記事では、コールバック地獄の回避策、`then`と`async/await`の使い分け、実際のエラーハンドリングを中心に解説します。

---

## コールバック地獄の危険性

JavaScriptの非同期処理でよく使うのはコールバック関数です。例えば、`fetch()`や`setTimeout()`で非同期操作を行うとき、関数を呼び出す前に結果を処理する必要があります。この繰り返しは「コールバック地獄」と呼ばれ、コードの可読性が低下し、メンテナンスが難しくなります。

**例：コールバック地獄**
```javascript
fetch(url)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));
```

このコードは読みやすいですが、複数の非同期処理が重なると、理解が難しくなります。

---

## Promiseとasync/awaitの利点

**Promise**は、非同期処理を扱うための標準的な方法です。`then`メソッドを使って結果を扱うことで、読みやすさを保つことができます。

**async/await**は、Promiseを簡潔に記述できる新しい構文です。これにより、非同期コードを直感的に書くことができます。

```javascript
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('エラーが発生しました:', error);
  }
}
```

このコードは「async/await」の力を発揮し、非同期処理の流れが自然になります。

---

## 実際のエラーハンドリングのポイント

エラーハンドリングは、非同期処理を扱う上で最も重要なスキルの一つです。`try/catch`ブロックを使ってエラーを捕捉し、適切に処理しましょう。

**例：エラーハンドリングの基本**
```javascript
async function loadContent() {
  try {
    const response = await fetch('https://api.example.com');
    if (!response.ok) throw new Error('Network response was not ok');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('APIエラー:', error);
  }
}
```

この例では、ネットワークエラーやJSON解析エラーなど、さまざまな状況を処理しています。

---

## まとめ

JavaScriptの非同期処理は、現代開発の基盤です。**Promise**と**async/await**をマスターすることで、コードの可読性と保守性が大幅に向上します。特に、**エラー処理**の習慣をしっかりと身につけることが重要です。実務でこれらの技術を活用することで、より効率的なアプリケーションが実現します。
