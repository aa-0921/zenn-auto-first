## 2. 開発効率が上がる必須拡張機能 5選

| 拡張機能 | 主な機能 | 推奨理由 |
|---|---|---|
| **Prettier - Code formatter** | コード自動整形 | フォーマットの統一でレビューが楽になる |
| **ESLint** | 静的解析・自動修正 | バグの早期検出とコーディング規約の徹底 |
| **GitLens — Git supercharged** | Git 履歴・ブレーム表示 | ブレームが一目で分かり、過去の変更に素早く遡れる |
| **Bracket Pair Colorizer 2** | 対応括弧を色分け | ネストが深いコードでも視認性が向上 |
| **Path Intellisense** | ファイルパス補完 | import 文の手入力を防ぎ、タイピングミスを削減 |

### 2.1 拡張機能のインストール例

VS Code のコマンドパレット (`Ctrl+Shift+P`) で **Extensions: Install Extensions** を選び、以下のように一括インストールできます。

```bash
code --install-extension esbenp.prettier-vscode
code --install-extension dbaeumer.vscode-eslint
code --install-extension eamodio.gitlens
code --install-extension CoenraadS.bracket-pair-colorizer-2
code --install-extension christian-kohler.path-intellisense
```

> **Tip**: `code` コマンドは VS Code の **Shell Command: Install 'code' command in PATH** を有効にすると利用できます。

## 3. 作業時間を短縮するおすすめキーバインド

拡張機能だけでなく、**キーバインドのカスタマイズ**は開発スピードに直結します。以下は「よく使う」操作を 1 手で実行できるようにした例です。

| 操作 | デフォルト | 推奨カスタムキー | 設定例 (`keybindings.json`) |
|---|---|---|---|
| ファイル保存 + フォーマット | `Ctrl+S` (保存) | `Ctrl+Alt+S` | `{ "key": "ctrl+alt+s", "command": "editor.action.formatDocument", "when": "editorTextFocus" }` |
| 行コメントトグル | `Ctrl+/` | `Ctrl+Shift+/` | `{ "key": "ctrl+shift+/", "command": "editor.action.commentLine", "when": "editorTextFocus" }` |
| Git ブランチ切替 | なし | `Ctrl+Alt+B` | `{ "key": "ctrl+alt+b", "command": "git.checkout", "when": "gitOpenRepositoryCount > 0" }` |
| ターミナルのトグル | ``Ctrl+` `` | `Ctrl+Alt+`` ` | `{ "key": "ctrl+alt+`", "command": "workbench.action.terminal.toggleTerminal" }` |
| ファイル検索 (全体) | `Ctrl+P` | `Ctrl+Shift+P` | `{ "key": "ctrl+shift+p", "command": "workbench.action.quickOpen" }` |

### 3.1 `keybindings.json` の編集手順

1. `Ctrl+Shift+P` → **Preferences: Open Keyboard Shortcuts (JSON)** を選択  
2. 上記 JSON オブジェクトを配列の要素として貼り付ける  
3. 保存すれば即座に有効化

> **注意**: キーバインドは他の拡張が同じ組み合わせを使用している場合、競合が起きることがあります。競合したら `when` 条件を調整してください。

## 4. 設定・拡張リストをチームや別環境に共有する方法

### 4.1 Settings Sync (公式) を使う

VS Code 1.48 以降、**Settings Sync** が標準機能として提供されています。GitHub アカウントや Microsoft アカウントで認証すれば、以下が自動で同期されます。

- `settings.json`（エディタ設定全般）  
- `keybindings.json`（キーバインド）  
- `extensions.json`（インストール済み拡張のリスト）  
- `snippets`（ユーザースニペット）  

#### 有効化手順

1. `Ctrl+Shift+P` → **Settings Sync: Turn On**  
2. GitHub / Microsoft アカウントでサインイン  
3. 「Upload Settings Now」を選択して現在の環境をクラウドにアップロード  

以降、別マシンで **Turn On** すれば自動的に復元されます。

### 4.2 手動で共有する場合（`.vscode` ディレクトリ活用）

プロジェクトルートに `.vscode` フォルダを作り、以下のファイルをコミットします。

```
.vscode/
├─ settings.json
├─ extensions.json
└─ keybindings.json
```

#### 例: `extensions.json`

```json
{
  "recommendations": [
    "esbenp.prettier-vscode",
    "dbaeumer.vscode-eslint",
    "eamodio.gitlens",
    "CoenraadS.bracket-pair-colorizer-2",
    "christian-kohler.path-intellisense"
  ]
}
```

`settings.json` にはチーム共通の設定（例: `editor.tabSize`, `files.trimTrailingWhitespace`）を書き、`keybindings.json` には上記で紹介したカスタムキーを入れます。  
**VS Code は自動的にこのフォルダを検出し、拡張のインストールや設定の適用を促します**。

### 4.3 さらに便利なツール：`dotfiles` リポジトリ

個人レベルで複数マシンを管理したい場合、GitHub に `dotfiles` リポジトリを作り、以下のように構成するとシンプルです。

```
dotfiles/
├─ .vscode/
│  ├─ settings.json
│  └─ keybindings.json
└─ install.sh
```

`install.sh` の例:

```bash
#!/usr/bin/env bash
# dotfiles のインストールスクリプト

# VS Code 設定ディレクトリ取得 (OS 毎に変わる)
if [[ "$OSTYPE" == "darwin"* ]]; then
  VSCODE_DIR="$HOME/Library/Application Support/Code/User"
else
  VSCODE_DIR="$HOME/.config/Code/User"
fi

# 設定ファイルをシンボリックリンクで配置
ln -sf $(pwd)/.vscode/settings.json $VSCODE_DIR/settings.json
ln -sf $(pwd)/.vscode/keybindings.json $VSCODE_DIR/keybindings.json

# 推奨拡張を自動インストール
cat $(pwd)/.vscode/extensions.json | \
  jq -r '.recommendations[]' | \
  xargs -L1 code --install-extension

echo "VS Code 設定のインストールが完了しました"
```

このスクリプトを `curl` で取得して実行するだけで、**全マシンで同一環境が構築**できます。

## 5. まとめ

- **拡張機能**は「コード整形」「静的解析」「Git 情報可視化」など、実務で頻繁に使う機能を中心に選定する。  
- **キーバインド**は自分の作業フローに合わせて最小限の手数に削減し、`keybindings.json` で一元管理する。  
- **設定共有**は公式の Settings Sync が最も手軽。チームでの統一が必要な場合は `.vscode` ディレクトリをリポジトリにコミットし、個人のマシン間同期は `dotfiles` リポジトリ＋シェルスクリプトで自動化できる。  

これらをすぐにでも導入すれば、**コードを書く時間が増え、環境構築にかかる時間が激減**します。  
ぜひ本記事の手順を試し、開発効率の向上を実感してください。
