## はじめに：年齢は言い訳じゃない、でも現実はある

「30代でエンジニアなんて、もう遅いよ」——そう言われたことがあります。実際、私も30代半ばで未経験からITエンジニアへ転職しました。転職活動中は、自己学習の遅さや若手との経験差に悩むこともありました。しかし、今では「あの時、一歩を踏み出してよかった」と心から思っています。この記事では、20代後半・30代からエンジニアを目指すあなたへ、私の等身大の体験談を基に、学習の進め方、転職のコツ、現場で感じた手応えを具体的に解説します。

## 背景・前提：なぜ30代からでもエンジニアを目指すのか？

現代のIT業界では、多様なバックグラウンドを持つ人材が求められています。私の場合、前職は営業職で、システムの仕組みに興味を持ったのがきっかけです。30代でのキャリアチェンジは、若手と比べて「学習時間が取れない」「技術トレンドについていけない」という不安がありました。しかし、逆に言えば、前職で培った問題解決能力やコミュニケーションスキルは、エンジニアとしての強みになり得ると気づきました。重要なのは、年齢ではなく「何を学び、どうアピールするか」です。

## 本文：実践的なステップとコード例

### 学習の進め方：プロジェクト駆動学習で Portfolio を築く

30代ともなると、まとまった学習時間を確保するのは難しいものです。そこで私は、「プロジェクトベース学習」を徹底しました。例えば、Webアプリケーションの作成に挑戦し、GitHubでコードを管理。具体的には、PythonのFlaskを使った簡単なTODOアプリを開発しました。

```python
# 学習中に書いた Flask アプリの一部
from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)
todos = []

@app.route('/')
def index():
    return render_template('index.html', todos=todos)

@app.route('/add', methods=['POST'])
def add():
    todo = request.form['todo']
    todos.append(todo)
    return redirect(url_for('index'))

if __name__ == '__main__':
    app.run(debug=True)
```

このように、実際に動くものを作ることで、学習のモチベーションが持続しました。さらに、GitHubのリポジトリを公開し、コードレビューをコミュニティから受けることで、実践的なスキルを身につけました。30代の強みを活かし、ビジネス視点で「なぜこの機能が必要か」を説明できるようになったことが、学習効果を高めました。

### 転職活動のコツ：ポートフォリオと「ストーリー」で勝負する

未経験での転職では、ポートフォリオが命です。私は、3つのプロジェクト（TODOアプリ、API連携ツール、データ可視化ダッシュボード）をGitHubにまとめ、各プロジェクトの「課題解決プロセス」をReadmeに詳細に記載しました。また、面接では「前職の営業経験を活かして、クライアントとの要件調整に貢献したい」と、年齢を弱点ではなく「多様な経験」としてアピール。年齢を気にしない企業（ベンチャーや第二新卒歓迎の会社）をターゲットに、IndeedやWantedlyで「キャリアチェンジ歓迎」と明記された求人を重点的に探しました。

### 現場で感じた手応え：即戦力ではないが「成長株」として認められる

入社後は、期待以上に「場数」がものをいう現場でした。例えば、バグ修正時に、デバッグで`print`文を駆使するよりも、ログ解析ツール（Pythonの`logging`モジュール）を使うことを先輩から教わりました。

```python
# 実務で役立ったログ出力の例
import logging
logging.basicConfig(level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s')
logging.info('ユーザー認証処理を開始')
```

このように、基礎的なコードスキルに加え、ツールの適切な使い方を学ぶことで、効率性が向上しました。また、30代の落ち着いた対応が、チーム内のトラブル時にも信頼につながりました。「若手にはない慎重さ」を評価されることもあり、年齢がプラスに働く瞬間を実感。継続学習が必須で、毎週1本は技術ブログを読み、スキルアップに努めています。

## まとめ：「遅すぎた」は自分自身が決めること

30代からエンジニアを目指す過程で、学習速度や若手との比較で落ち込むこともありました。しかし、プロジェクトを通じた実践的学習、ポートフォリオを基にした転職、現場での経験蓄積により、「遅くない」と確信しています。年齢は、多様な視点をもたらす財産です。もし迷っているなら、小さな一歩から始めてみてください。コードを1行書き、GitHubにプッシュし、その経験を語れるようになる——それが、キャリアチェンジの第一歩になります。
