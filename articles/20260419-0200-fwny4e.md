## 1. 路由エラーの原因と対処法
Ruby on Rails ではルーティング定義に小さなミスがあると `No route matches [GET] /hoge` などのエラーが出ます。代表的な原因は次の通りです。

1. **HTTP メソッドの不一致**  
   `resources :posts` で生成されたルートは `GET /posts`（index）、`POST /posts`（create）だけです。`PUT /posts/:id` や `DELETE /posts/:id` を使うと `method not allowed` になるので、`resources` のブロックに `as:` オプションで名前を付け、意図した HTTP メソッドを明示します。

2. **ネストされたルートの省略**  
   ネストしたリソースは `resources :comments, shallow: true` のように明示しないと `comments` の親 URL が自動生成されません。ネストが深い場合は `member` や `collection` を使い、`resources :posts do
  resources :commentsend` の形で explicit に定義します。

3. **パラメータ名の誤り**  
   Strong Parameter で許可していないキーを渡すと `param is missing or the value is invalid` が出ます。このエラーはコントローラで `params.require(:post).permit(:title, :body)` のように明示的に許可リストを設定して回避します。

```ruby
# config/routes.rb
Rails.application.routes.draw do
  resources :posts do
    resources :comments, only: [:index, :create]
  end
  root to: 'posts#index'
end
```

```ruby
# app/controllers/comments_controller.rb
class CommentsController < ApplicationController
  def create
    @post = Post.find(params[:post_id])
    @comment = @post.comments.build(comment_params)
    if @comment.save
      redirect_to post_path(@post), notice: 'コメントを追加しました'
    else
      render :new, alert: 'コメントの保存に失敗しました'
    end
  end  private

  def comment_params
    params.require(:comment).permit(:content)
  end
end
```

このようにルート定義とコントローラ側で意図した HTTP メソッド・パラメータを明示すれば、ほとんどのルーティングエラーは解消します。

## 2. マイグレーションでハマるケース
マイグレーションはデータベーススキーマを変える重要な工程ですが、以下のようなミスが頻発します。

### 2‑1. カラム名の誤字・脱字
`add_column :users, :email_digest, :string` のように意図した列名と実際の列名がずれると、`column does not exist` エラーが出ます。Rails 5 以降では `rake db:migrate` 実行前に `schema.rb` を確認し、命名規則を統一することが推奨です。
