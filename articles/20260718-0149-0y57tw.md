## 2. 代表的な拡張機能とその目的  
以下に、最も頻繁に推奨される拡張機能をカテゴリ別にまとめました。公式 Marketplace から簡単にインストールできます。

| カテゴリ | 拡張機能 | 主な機能 |
|----------|----------|----------|
| 言語サポート | **ESLint** | JavaScript/TypeScript の静的解析 |
| 言語サポート | **Prettier - Code formatter** | コードの自動整形 |
| デバッグ | **Debugger for Chrome** | Chrome デバッグ |
| 完成度向上 | **GitLens** | Git 操作の可視化 |
| UI カスタマイズ | **Material Icon Theme** | アイコンの統一 |
| テスト支援 | **Run on Save** | ファイル保存時に自動実行 |

特に **Prettier** と **ESLint** は、コードの品質を保つ上で必須と言えます。以下に、これらを組み合わせた設定例を示します。

```json
// .vscode/settings.json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    "typescript",
    "typescriptreact"
  ],
  "eslint.run": "onSave"
}
```

この設定により、保存時に自動でコードが整形され、保存前に ESLint がエラーを検出できます。

## 3. キーバインドのカスタマイズと共有  
VS Code はデフォルトのキーバインドでも十分に強力ですが、頻繁に使用する操作をカスタマイズすると作業効率が向上します。例えば、ファイルツリーで **`Ctrl + N`** で新しいファイルを作成したり、**`Alt + ↑ / ↓`** で行を上下に移動できます。

### 3.1 キーバインドの編集手順  
1. `Ctrl + K Ctrl + S` でキーバインド設定画面を開く  
2. `json` タブを選択し、`keybindings.json` を編集  
3. 任意のコマンドにショートカットを割り当てる

```json
// .vscode/keybindings.json
[
  {
    "key": "ctrl+n",
    "command": "workbench.action.files.newFile",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+shift+r",
    "command": "workbench.action.reloadWindow",
    "when": "editorTextFocus"
  }
]
```

上記設定はチーム全体で共有できるため、リポジトリの `.vscode` ディレクトリに置くだけで、メンバー全員が同じキーバインドで作業できます。

### 3.2 共有方法とベストプラクティス  
- **リポジトリに `.vscode` ディレクトリをコミット**：設定ファイル（`settings.json`, `keybindings.json`, `extensions.json`）をバージョン管理に含める  
- **`extensions.json` で必須拡張機能の一覧管理**：新規参入者が自動でインストールできるようにする  
  ```json
  // extensions.json
  {
    "recommendations": [
      "dbaeumer.vscode-eslint",
      "esbenp.prettier-vscode",
      "ms-azuretools.vscode-docker",
      "eamodio.gitlens"
    ]
  }
  ```
- **ドキュメント化**：`README.md` に「開発環境の準備手順」を記載し、新人向けに手順をまとめる

## 4. 設定のベストプラクティスと注意点  
1. **設定は最小限に**：不要な設定は削除し、チーム共有のルールに沿った設定だけを残す  
2. **`.gitignore` でローカル設定を除外**：`settings.json` のローカルユーザー設定は `.gitignore` に追記して管理しない  
3. **定期的な見直し**：半年に一度は拡張機能や設定を見直し、最新のベストプラクティスに合わせる  

## 5. まとめ  
VS Code の生産性を高めるためには、以下の 3 つのポイントを押さえることが重要です。  
1. **必須拡張機能のインストール**（ESLint、Prettier、GitLens など）  
2. **設定ファイルのリポジトリ共有**（`.vscode/settings.json`, `keybindings.json`, `extensions.json`）  
3. **キーバインドのカスタマイズとチームルール化**  

これらを実践すれば、開発環境が統一され、チーム全体のコード品質と作業効率が向上します。ぜひ、最初のプロジェクトから導入してみてください。
