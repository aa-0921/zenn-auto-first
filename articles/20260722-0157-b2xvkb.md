## 1. useEffectの依存配列エラー

### 背景と問題点

`useEffect`の第2引数である依存配列は、コンポーネントの振る舞いに大きな影響をします。依存配列を正しく設定しないと、**無限ループ**や**期待しない実行タイミング**が発生します。

### 具体例：依存配列の見落とし

```jsx
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  // NG: userId が依存配列に入っていない
  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => setUser(data));
  }, []); // ← ここが問題！

  return <div>{user?.name}</div>;
}
```

上記のコードでは、`userId`が変化した場合にデータが再取得されません。ユーザー切り替え時に古いユーザー情報が表示されたままになってしまいます。

### 解決策

依存配列に`userId`を追加します。

```jsx
useEffect(() => {
  fetch(`/api/users/${userId}`)
    .then(res => res.json())
    .then(data => setUser(data));
}, [userId]); // ← 正しく依存させる
```

### 依存配列に注意したい3つのポイント

1. **コンポーネント外部から渡されたpropsは必ず依存配列に入れる**
2. **依存させたくない値は`useRef`や`useCallback`でラップする**
3. **ESLintの`react-hooks/exhaustive-deps`ルールを有効にすると自動チェックできる**

```jsx
// eslint.config.js などでルールを有効化
{
  plugins: ['react-hooks'],
  rules: {
    'react-hooks/rules-of-hooks': 'error',
    'react-hooks/exhaustive-deps': 'warn',
  },
}
```

## 2. 再レンダーの最適化と無限ループ

### 背景と問題点

Reactコンポーネントは状態やpropsが変化すると再レンダーされます。依存配列が不適切だと、**状態の更新が常にトリガーされる**ことにつながり、無限ループが発生します。

### 具体例：setStateの不適切な使用

```jsx
import { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  const [doubled, setDoubled] = useState(0);

  // NG: countが変化るたびにsetDoubledが呼ばれ、再度countが変わる
  useEffect(() => {
    setDoubled(count * 2);
  }, [count]); // ← 依存配列は正しいが、ロジックが問題

  return (
    <div>
      <p>Count: {count}</p>
      <p>Doubled: {doubled}</p>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}
```

上記は無限ループにはならないものの、**不必要な再レンダー**が頻発します。特に`setDoubled`の呼び出しが毎回走るため、パフォーマンスが低下します。

### 解決策：useMemoやuseCallbackを活用

状態の算出は`useMemo`でキャッシュし、コールバック関数は`useCallback`でメモ化します。

```jsx
import { useState, useEffect, useMemo, useCallback } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const doubled = useMemo(() => count * 2, [count]);

  const increment = useCallback(() => {
    setCount(c => c + 1);
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <p>Doubled: {doubled}</p>
      <button onClick={increment}>+1</button>
    </div>
  );
}
```

この書き換えにより、`doubled`は`count`が変化った時だけ再計算され、無駄な再レンダーが削減されます。

## 3. キーの付け忘れとリストレンダリング

### 背景と問題点

Reactのリスト処理では、各要素に一意な`key`プロップを付ける必要があります。キーが不適切だと、**要素の不正確な差し替え**や**状態の乱れ**が発生します。

### 具体例：keyの重複やインデックスの使用

```jsx
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo, index) => (
        // NG: indexをkeyにすると順序変更時に問題が発生
        <li key={index}>
          <input type="checkbox" checked={todo.done} onChange={() => {}} />
          {todo.text}
        </li>
      ))}
    </ul>
  );
}
```

上記の`key={index}`は、リストの順序が変わったり要素が削除されたりすると、Reactが正しく要素を特定できず、**チェックボックスの状態が乱れる**などの不具合が発生します。

### 解決策：一意なkeyを使用する

各要素に一意かつ安定したIDを`key`として使用します。

```jsx
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo) => (
        <li key={todo.id}>
          <input type="checkbox" checked={todo.done} onChange={() => {}} />
          {todo.text}
        </li>
      ))}
    </ul>
  );
}
```

### keyに注意したい3つのポイント

1. **必ず一意な値を使用する（通常はデータのid）**
2. **インデックスは絶対に使用しない（順序が変わると危険）**
3. **keyは「Reactの内部管理用」なので、他の用途には使用しない**

## 4. よくあるエラーメッセージと対処法まとめ

| エラーメッセージ | 原因 | 解決策 |
|----------------|------|--------|
| `Maximum update depth exceeded` | useEffectでsetStateを無限に呼んでいる | 依存配列を確認し、不必要な更新を防ぐ |
| `Each child in a list should have a unique "key"` | リスト要素にkeyがない、または重複 | 一意なkeyを付与する |
| `React Hook useEffect has a missing dependency` | 依存配列が不完全 | ESLintルールで自動補完、依存値を確認 |
| `Warning: Extra prop "x" is being ignored` | DOM要素に不適切なpropsを渡している | DOM以外の要素にはpropsを適切に渡す |

---

## まとめ

Reactでよくあるエラーは、**「依存配列の見落とし」「不必要な再レンダー」「キーの不備」**が根底にあります。これらは単なる書き間違いではなく、アプリケーションの安定性やパフォーマンスに直結する重要なポイントです。

- `useEffect`の依存配列は必ず`eslint-plugin-react-hooks`でチェック
- 再レンダーを防ぐために`useMemo`・`useCallback`を適切に活用
- リストの`key`は一意で安定した値を使用

これらのポイントを意識してコードを書けば、予期しないバグやパフォーマンス問題の多くを回避できます。実務で役立つので、ぜひ自分のプロジェクトに取り入れてみてください。
