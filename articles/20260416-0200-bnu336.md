## 1. REST APIとは
REST（Representational State Transfer）は、Web上のリソースをURIで表現し、HTTPメソッドでその操作を表すアーキテクチャです。CRUD操作をそれぞれ GET, POST, PUT, DELETE に紐付けられるため、シンプルかつ可預かりな設計が可能です。JSONでデータをやり取りすることが一般的で、クライアントはサーバーの状態を保持せず、 Stateless（無状態）な通信を実現できます。

## 2. HTTPメソッドとステータスコード
| メソッド | 用途 | 主なステータスコード |
|----------|------|----------------------|
| GET      | リソース取得 | 200 OK, 404 Not Found |
| POST     | リソース作成 | 201 Created, 400 Bad Request |
| PUT      | リソース更新 | 200 OK, 400 Bad Request |
| DELETE   | リソース削除 | 204 No Content, 404 Not Found |

ステータスコードはレスポンスの結果を示すだけでなく、エラーハンドリングの指針にもなります。フロントエンド側はこのコードを元にエラーメッセージや再試行ロジックを実装します。

## 3. フロントエンドでAPIを呼び出す流れ1. **エンドポイントの決定**  
   例: `https://api.example.com/users`
2. **リクエストオプションの作成**  
   メソッド、ヘッダー（`Content-Type: application/json`）、ボディ（JSON）を設定。
3. **fetch / axios でリクエスト送信**  
   プロミスや async/await を使って非同期処理を記述。
4. **レスポンスの検証**  
   ステータスコードが 2xx 系か確認し、JSON をパース。
5. **状態管理**  
   取得したデータを Redux や Context API などで管理し、コンポーネントに反映。

## 4. 実装例（fetch と axios）

### 4-1. fetch での呼び出し
```javascript
async function fetchUsers() {
  const response = await fetch('https://api.example.com/users', {
    method: 'GET',
    headers: {
      'Content-Type': 'application/json',
    },
  });

  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }

  const data = await response.json();
  return data;
}

// 使用例
fetchUsers()
  .then(users => console.log(users))
  .catch(err => console.error(err));
```

### 4-2. axios での呼び出し
```javascript
import axios from 'axios';

async function fetchUsersWithAxios() {
  try {
    const response = await axios.get('https://api.example.com/users', {
      headers: { 'Content-Type': 'application/json' },
    });
    return response.data; // 2xx 系のステータスが自動で処理される
  } catch (error) {
    // エラーハンドリング
    console.error('API呼び出しに失敗:', error.response?.status);
    throw error;
  }
}

fetchUsersWithAxios()
  .then(users => console.log(users))
  .catch(err => console.error(err));
```

## 5. まとめ
- REST APIは HTTP メソッドとステータスコードでインターフェースを統一し、JSON でデータ交換を行うのが主流です。  
- フロントエンド側では `fetch` や `axios` を使ってリクエストを送信し、ステータスコードとレスポンス体形を適切に処理することが重要です。  
- エラーハンドリングや状態管理を適切に行うことで、ユーザー体験を損なわない堅牢なフロントエンド実装が可能になります。
