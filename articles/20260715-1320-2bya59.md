## ルーティングエラーの対処法
Rails のルーティングは `config/routes.rb` で定義されますが、設定ミスは頻発します。

### よくあるルーティングエラー例
```ruby
# routes.rb（間違った例）
Rails.application.routes.draw do
  get 'users', to: 'users#index'
  # resources :users の代わりに個別にルートを定義してしまった
end
```

上記の設定では `/users/1/edit` のようなルートが存在しないため、`No route matches` エラーが発生します。

### 解決策
`resources :users` を使用して自動的にルートを生成します。
```ruby
# routes.rb（正しい例）
Rails.application.routes.draw do
  resources :users
end
```

また、ルートの確認は `rails routes` コマンドで行います。
```bash
$ rails routes
```

## マイグレーションのトラブルシューティング
マイグレーションはデータベースのスキーマを管理する重要な機能ですが、間違った操作をするとエラーが発生します。

### エラー例：カラムの追加忘れ
```ruby
# マイグレーションファイル
class AddEmailToUsers < ActiveRecord::Migration[7.0]
  def change
    add_column :users, :email, :string
  end
end
```

このマイグレーションを実行後に `User.new(email: "test@example.com")` とすると `unknown attribute` エラーが発生します。原因は `schema.rb` が正しく更新されていない可能性があります。

### 解決策
マイグレーションが正しく適用されているか確認し、必要であれば `rails db:migrate:reset` を実行します。
```bash
$ rails db:migrate
$ rails db:migrate:status
```

## N+1 クエリの問題を解消する
N+1 クエリはパフォーマンスの低下を引き起こす一般的な問題です。

### 問題のあるコード例
```ruby
# app/controllers/users_controller.rb
def index
  @users = User.all
end

# app/views/users/index.html.erb
<% @users.each do |user| %>
  <p><%= user.posts.count %></p>
<% end %>
```

このコードではユーザー数 × 投稿数のクエリが発生し、N+1 問題が発生します。

### 解決策
`includes` を使用して関連データを事前に読み込みます。
```ruby
# app/controllers/users_controller.rb
def index
  @users = User.includes(:posts)
end
```

また、`bullet` gem を導入して問題を検出できます。
```ruby
# Gemfile
gem 'bullet', group: :development
```

## まとめ
- ルーティングは `resources` を活用して自動生成する
- マイグレーションは `rails db:migrate:status` で確認し、必要に応じてリセットする
- N+1 問題は `includes` や `bullet` gem で解消する

これらのコツを活用することで、Rails 開発の効率を大幅に向上させることができます。
