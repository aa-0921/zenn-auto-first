## 背景・前提
-  newly 入社したエンジニアは、業務にすぐ追われるため「とりあえず動かす」ことが優先されがちです。  
-  しかし、コードレビューで指摘されると「自分で書けない」状態になりやすく、成長が止まります。  
-  そんな時、**AI ツールを「補助」** として位置付け、適切に活用できれば、短期間でタスク消化速度とコード品質の向上が期待できます。  
-  ここでは、Claude Code を例に取りますが、同様の手法は他のコード生成 AI でも適用可能です。

## AIツール導入のステップ
### 1. 目的を明確にする
まずは「何をしたいか」を明確にします。  
- 新機能のプロトタイプを作りたい  
- テストコードを自動生成したい  
- エラー処理をテンプレート化したい  
目的がはっきりすれば、プロンプトの質が上がり、無駄な修正を減らすことができます。

### 2. 小さなタスクから始める
最初は 1〜2 行の関数実装や、ユーティリティメソッドの作成から試すと良いです。  
例）`parseJson` 関数を生成させるだけでも、入出力の形を把握できます。  
小さな成功体験が「自分でも書ける」という自信につながります。

### 3. 出力を検証し、自分で修正する
AI が生成したコードは **完全に正しいとは限りません**。  
- 型チェックやエラーハンドリングが抜けているケースが多い  
- 可読性や慣習に合わないことがある  
そこで、**自分のコードスタイルに合わせて手直し**し、コメントで「なぜこの修正をしたか」を残す習慣をつけましょう。  
```ts
// AI が生成した初期実装
function parseJson(str: string): any {
  return JSON.parse(str);
}

// 修正後：エラーハンドリングと型定義を追加
function parseJson<T>(str: string): T | null {
  try {
    return JSON.parse(str);
  } catch (e) {
    console.error('Invalid JSON:', e);
    return null;
  }
}
```

### 4. フィードバックループを回す
毎回生成後に **自動テストや Linter** を走らせ、結果をツールに返すと、次回の出力が精度を増します。  
- 「このテストが失敗したので、戻り値を Option<T> に変更してほしい」  
- 「エラーメッセージをロギングしてほしい」  
このように **具体的なリクエスト** を投げかけると、AI は学習し、徐々に期待通りのコードを提供してくれます。

## 具体的なコード例
以下は、Claude Code に「Node.js で CSV を読み込み、特定列の合計を求める関数を作って」と依頼した例です。

```bash
# Claude Code に指示
> Write a function that reads a CSV file and returns the sum of the "price" column as a number.
```

```js
// 生成された最初のコード例
const fs = require('fs');

function sumPrice(csvPath) {
  const data = fs.readFileSync(csvPath, 'utf8');
  const rows = data.split('\n').slice(1); // ヘッダー除く
  let total = 0;
  rows.forEach(row => {
    const cols = row.split(',');
    total += parseFloat(cols[2]); // 3列目が price
  });
  return total;
}

module.exports = { sumPrice };
```

**修正ポイント**
1. エラー処理を追加  
2. `parseFloat` が数値でない場合の安全策を入れる  
3. CSV パarsing に `csv-parse` などのライブラリを推奨  

```js
const fs = require('fs');
const { parse } = require('csv-parse');

function sumPrice(csvPath) {
  try {
    const data = fs.readFileSync(csvPath, 'utf8');
    const records = parse(data);
    const total = records.reduce((acc, rec) => {
      const price = parseFloat(rec.price);
      return isNaN(price) ? acc : acc + price;
    }, 0);
    return total;
  } catch (err) {
    console.error('Failed to read CSV:', err);
    return 0;
  }
}
```

このように **目的 → 生成 → 検証 → 修正 → 再依頼** のサイクルを回すことで、AI の力を最大限に活かしつつ、自分のコード力も着実に伸ばすことができます。

## まとめ
- **目的を明確に**し、小さなタスクから始めることで、AI の出力が実際の開発にどう適合するかを体感できます。  
- **出力は必ず検証**し、自分のコーディングルールに合わせて修正する習慣をつけると、品質と理解度が向上します。  
- **フィードバックループ**を確立すれば、AI が徐々に自分の期待に沿ったコードを提供してくれるようになります。  
- 最終的には **「AIに任せる」ではなく「AIと協働する」** という姿勢が、初心者から中級者への最短ルートです。  

Claude Code のようなツールは、単なるコード生成機械ではなく、開発プロセス全体を支えるパートナーです。上手く付き合いながら、ぜひ自分らしいコードを書き続けてください。
