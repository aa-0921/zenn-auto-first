## はじめに：なぜ非同期処理が重要なのか

Webアプリケーションでは、API通信、ファイル読み込み、タイマー処理など、時間がかかる操作を扱うことが日常茶飯事です。JavaScriptはシングルスレッドであるため、これらの処理をブロッキングせずに「後で実行する」仕組み、つまり非同期処理が必須になります。しかし、従来のコールバック関数を使った実装では、複数の非同期処理が入れ子になるとコードが読みにくくなる「コールバック地獄」が発生していました。この記事では、Promiseとasync/awaitという現代的な解決策について、具体的なコード例を通じて解説します。

## Promiseの基本： thenチェーンとエラーハンドリング

Promiseは「将来完了するかもしれない操作」をオブジェクトとして表現する仕組みです。以下の例は、2秒後にメッセージを返す非同期関数をPromiseで実装したものです。

```javascript
function waitMessage(msg, seconds) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (msg) resolve(`メッセージ: ${msg}`);
      else reject(new Error('メッセージが空です'));
    }, seconds * 1000);
  });
}

// thenチェーンの使用例
waitMessage('Hello', 1)
  .then(result => {
    console.log(result); // 1秒後に出力: メッセージ: Hello
    return waitMessage('World', 1);
  })
  .then(result => {
    console.log(result); // さらに1秒後に出力: メッセージ: World
  })
  .catch(error => {
    console.error('エラー:', error.message);
  });
```

Promiseの特徴は、`then`で成功時の処理を、`catch`でエラー時の処理をチェーンできる点です。`finally`を使えば、成否に関わらず実行する処理も追加できます。注意点として、`then`は常に新しいPromiseを返すため、戻り値を返さない関数をチェーンすると意図しない動作になることがあります。

## async/awaitの使い方：同期コードのように書く

async/awaitはPromiseを「同期のように」書ける構文です。以下の例は、先ほどのwaitMessage関数をasync/awaitで書き直したものです。

```javascript
async function sequentialMessages() {
  try {
    const result1 = await waitMessage('Hello', 1);
    console.log(result1);
    const result2 = await waitMessage('World', 1);
    console.log(result2);
  } catch (error) {
    console.error('エラー:', error.message);
  }
}

sequentialMessages();
```

`await`はPromiseの解決を待ち、その結果を返します。`async`関数は常にPromiseを返すため、以下のようにも呼び出せます。

```javascript
sequentialMessages()
  .then(() => console.log('完了'))
  .catch(error => console.error('失敗:', error));
```

## 実践的なエラーハンドリングと使い分け

### エラーハンドリングのベストプラクティス

1. **Promiseチェーンの場合**：チェーンの末尾に単一の`catch`を置く
2. **async/awaitの場合**：`try-catch`で囲むか、関数自体を`try-catch`してPromiseを返す

```javascript
// 関数内でエラーをキャッチし、呼び出し元に伝搬
async function safeOperation() {
  try {
    const data = await fetchData();
    return process(data);
  } catch (error) {
    console.error('操作に失敗:', error);
    throw error; // 必要に応じて再スロー
  }
}
```

### Promiseとasync/awaitの使い分け

| シナリオ | 推奨 | 理由 |
|----------|------|------|
| 直列処理が明確な場合 | async/await | コードが直感的でデバッグしやすい |
| 並列処理が必要な場合 | Promise.all | 複数の非同期を同時に開始できる |
| 条件分岐を含む複雑なフロー | ケースバイケース | Promiseチェーンが適する場合も |

### 並列処理の例：Promise.allの活用

```javascript
async function fetchMultipleUsers(userIds) {
  try {
    // 複数のAPI呼び出しを並列実行
    const promises = userIds.map(id => fetchUser(id));
    const users = await Promise.all(promises);
    console.log('全ユーザー取得成功:', users);
    return users;
  } catch (error) {
    console.error('いずれかの取得に失敗:', error);
    // Promise.allは1つでも失敗すると全体が失敗する
    // 失敗しても続行したい場合はPromise.allSettledを使用
  }
}

// Promise.allSettledの例：個別の結果を確認
async function fetchWithFallback(userIds) {
  const results = await Promise.allSettled(
    userIds.map(id => fetchUser(id))
  );
  
  results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      console.log(`ユーザー${userIds[index]}:`, result.value);
    } else {
      console.warn(`ユーザー${userIds[index]}の取得失敗:`, result.reason);
    }
  });
}
```

## まとめ：実務で役立つポイント

- **コールバック地獄の脱却**：Promiseとasync/awaitはネストを平坦化し、コードの可読性を大幅に向上させる
- **エラーハンドリングの統一**：Promiseチェーンでは単一の`catch`、async/awaitでは`try-catch`で一元管理可能
- **パフォーマンス considerations**：直列処理が必要な場面では`await`を、並列処理が可能な場面では`Promise.all`を使用して時間を節約
- **デバッグのしやすさ**：async/awaitはスタックトレースが明確で、エラーの原因追跡が容易

実務では、関数の性質に応じて使い分けることが重要です。APIクライアントやデータ処理パイプラインではasync/awaitが直感的で、一方で複数の非同期処理の成否に依存しない並列実行ではPromise.allが威力を発揮します。まずは既存のコールバックベースのコードをPromiseでラップし、その後async/awaitに移行する段階的アプローチも有効です。
