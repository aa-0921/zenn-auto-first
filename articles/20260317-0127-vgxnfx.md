Git の基礎と実践コマンド集
================================

Git は、ソフトウェア開発の不可欠なバージョン管理システムです。この記事では、Git の基本的な流れと、現場で日常的に使用するコマンドを解説します。特に、駆け出しエンジニアが押さえておくべきポイントを中心に、具体的な操作方法と対処法を紹介します。

## はじめに：Git の役割と基本概念
Git は、ファイルの変更履歴を管理し、複数人での共同作業を円滑にします。主な特徴は次のとおりです：
- **分散型バージョン管理**：各開発者が完全なリポジトリを持つため、サーバーに接続しなくても作業が可能。
- **ブランチモデル**：独立した開発環境を作成し、機能追加やバグ修正を安全に行う。
- **リベース**：ブランチの統合を、マージの代わりに行うことで、履歴を綺麗に保つ。

この記事では、Git の基本的な流れと、最も頻繁に使用するコマンドを解説します。

## 背景・前提：Git の基本的な流れ
Git を使うための基本的な流れは次のとおりです：
1. **リポジトリの取得（clone/pull）**
2. **新しいブランチの作成（branch）**
3. **変更の追跡（add/commit）**
4. **リモートへの反映（push）**
5. **他のブランチへの統合（merge/rebase）**

この流れを実践するためのコマンドを順に説明します。

## コマンド集：実践で役立つ 5 つのコマンド
### 1. clone：リポジトリの取得
```bash
$ git clone https://github.com/user/repo.git
```
- このコマンドで、指定したリモジュトリポジトリをローカル環境にコピーします。
- リポジトリのディレクトリが作成され、初期状態で `main` ブランチがチェックアウトされます。
- コマンドを実行後、`ls` コマンドでディレクトリを確認し、`cd` コマンドで移動します。

### 2. pull：リモートの変更を取り込む
```bash
$ git pull origin main
```
- このコマンドで、リモートリポジトリの `main` ブランチの変更をローカルブランチに取り込みます。
- コマンドを実行前に、`git status` コマンドで変更を確認し、`git add` コマンドで未追跡ファイルを追跡対象に追加します。
- コンフリクトが発生した場合は、`git status` コマンドで詳細を確認し、編集ファイルを手動で解決します。

### 3. branch：新しいブランチの作成
```bash
$ git branch new-feature
$ git checkout new-feature
```
- このコマンドで、`new-feature` という名前の新しいブランチを作成し、そのブランチに切り替えます。
- ブランチを切り替える前に、`git status` コマンドで現在のブランチを確認し、`git checkout` コマンドで切り替えます。
- ブランチ名には、機能名やバグ修正内容を表す名前を付けます（例：`fix-login-bug`、`add-payment-feature`）。

### 4. commit：変更の記録
```bash
$ git add .
$ git commit -m "Fix login validation"
```
- `git add .` コマンドで、現在のディレクトリ以下にあるすべての変更を追跡対象に追加します。
- `git commit` コマンドで、変更を記録します。コミットメッセージには、変更内容を一言で表すメッセージを記述します（例：`Fix login validation`）。
- コミットメッセージの書き方に注意：現在分詞で表すこと（`Fix` ではなく `Fixed`）、変更の目的を明確にする。

### 5. push：リモートへの反映
```bash
$ git push origin new-feature
```
- このコマンドで、ローカルブランチ `new-feature` の変更をリモートリポジトリの `new-feature` ブランチに反映します。
- 初回の `push` では、`--set-upstream` オプションを付ける必要がある場合があります（例：`git push --set-upstream origin new-feature`）。
- リモートブランチが存在しない場合は、`git push` コマンドで自動的に作成されます。

## コンフリクト時の対処：手順を追って解決する
コンフリクトが発生した場合の対処法を次に示します：
1. `git pull` コマンドを実行し、コンフリクトのメッセージを確認します。
2. `git status` コマンドで、コンフリクトしているファイルの一覧を確認します。
3. コンフリクトしているファイルをテキストエディタで開き、Git がマークした箇所を編集します。
4. 編集後、`git add` コマンドでファイルをステージングします。
5. `git commit` コマンドで、変更を記録します（コミットメッセージは自動生成されます）。
6. `git push` コマンドで、変更をリモートに反映します。

## まとめ：Git の基本をマスターするポイント
- **リポジトリの取得**：`clone` コマンドでリポジトリをローカルに取得し、`cd` コマンドでディレクトリを移動します。
- **ブランチの管理**：新しい機能を追加する際は、`branch` コマンドで新しいブランチを作成し、`checkout` コマンドで切り替えます。
- **変更の追跡**：変更を記録する前に、`add` コマンドでファイルを追跡対象に追加します。
- **リモートへの反映**：変更をリモートリポジトリに反映するには、`push` コマンドを実行します。
- **コンフリクトの解決**：コンフリクトが発生した場合は、`status` コマンドでファイルの一覧を確認し、テキストエディタで編集を行います。

Git を効果的に使うためには、これらのコマンドを実践し、日常的な作業に組み込む必要があります。特に、`commit` メッセージの書き方や、`branch` モデルの管理は、チーム開発の品質を高める重要なポイントとなります。初心者の方は、コマンドの説明をしっかりと確認し、小さな変更から始めてみてください。
