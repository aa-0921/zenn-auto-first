## Gitの基本概念と用語
Gitを使い始める前に、基本的な用語を理解しておきましょう。

- **リポジトリ**: プロジェクトのファイルと変更履歴を保存する場所
- **ローカルリポジトリ**: 自分のコンピュータ上のリポジトリ
- **リモートリポジトリ**: サーバー上にある共有リポジトリ（GitHub、GitLabなど）
- **ブランチ**: 開発の流れを分けるための仮想的な「枝」
- **ワーキングディレクトリ**: 実際にファイルを編集する作業場所
- **ステージングエリア**: 次のコミットに含める変更を一時的に保存する場所

これらの概念を理解することで、Gitのコマンドがどのように動作するかがより明確になります。

## 基本的なワークフローとコマンド

### 1. clone: リモートリポジトリからコピーを作成
新しいプロジェクトに参加する時や、自分のコンピュータで開発を始める時に最初に行う操作です。

```bash
git clone https://github.com/username/repository.git
```

このコマンドを実行すると、リモートリポジトリの内容がローカルにコピーされます。ディレクトリ名を指定することもできます。

```bash
git clone https://github.com/username/repository.git my-project
```

### 2. pull: リモートの最新変更を取得
チームの他のメンバーが行った変更を自分のローカルに反映する時に使用します。

```bash
git pull origin main
```

`pull`は内部で`fetch`と`merge`を行っているため、コンフリットが発生する可能性があります。定期的にpullを行うことで、大きなコンフリットを防ぐことができます。

### 3. branch: ブランチの作成と切り替え
新機能の開発やバグ修正を行う時は、メインブランチから新しいブランチを作成します。

```bash
git branch feature/new-function
```

ブランチを作成した後、そのブランチに切り替えます。

```bash
git checkout feature/new-function
```

または、作成と切り替えを同時に行うこともできます。

```bash
git checkout -b feature/new-function
```

### 4. commit: 変更の記録
作業が完了したら、変更をステージングエリアに追加しコミットします。

```bash
git add file1.js file2.js
git commit -m "新機能の追加: ユーザー認証機能を実装"
```

`add`は個別のファイルを指定することも、すべての変更を追加するために`.`を使用することもできます。

```bash
git add .
```

### 5. push: リモートリポジトリに変更を送信
コミットした内容をリモートリポジトリに送信します。

```bash
git push origin feature/new-function
```

新しいブランチを初めてプッシュする場合は、以下のように指定します。

```bash
git push -u origin feature/new-function
```

## コンフリクトの対処方法

### コンフリクトの発生状況確認
コンフリクトが発生した場合、Gitはその旨を通知します。

```bash
git status
```

このコマンドで、競合が発生しているファイルを確認できます。

### コンフリクトの解決手順
1. 競合しているファイルを開き、Gitが自動で挿入するマーカーを確認します。

```text
<<<<<<< HEAD
自分の変更内容
=======
他の変更内容
>>>>>>> branch-name
```

2. どちらの変更を採用するかを決め、マーカーを削除して正しいコードを残します。

3. 解決したファイルをステージングエリアに追加します。

```bash
git add resolved-file.js
```

4. コンフリクトをマークしたコミットを完了します。

```bash
git commit
```

### コンフリクトを避ける方法
- 定期的に`pull`を行い、最新の変更を取り込む
- 大きな変更は小さく分けてコミットする
- チームメンバーと変更するファイルについて事前に連絡する

## まとめ
Gitの基本的なワークフローは、clone → pull → branch → commit → push の順で行います。この流れを理解し、日常的に使用することで、チーム開発におけるバージョン管理の効率が大幅に向上します。また、コンフリートが発生した時の対処方法を知っておくことで、開発の中断を最小限に抑えることができます。

これらの基本をマスターしたら、さらに高度なトピックスであるstash、rebase、cherry-pickなどを学ぶことをお勧めします。Gitは奥が深いツールですが、基本を押さえておけば、ほとんどの場面に対応できます。
