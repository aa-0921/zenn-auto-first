### 背景

JavaScriptは非同期処理をサポートする機能がたくさんあり、コールバック地獄（callback hell）から脱却して、より読みやすく扱いやすい非同期処理を実現します。この記事では、Promiseとasync/awaitの入門から、エラーハンドリングまで、具体的な例を使って解説します。

### Promiseの基礎

PromiseはJavaScriptが提供する非同期処理の機能の一つです。Promiseは値の取得やエラー処理を簡単に実現できるように設計されています。

```javascript
// Promiseの基本例
new Promise((resolve, reject) => {
  // 非同期処理
  setTimeout(() => {
    resolve('非同期処理の結果');
  }, 1000);
})
.then((result) => {
  console.log(result);
});
```

### thenの使い方

thenはPromiseのメソッドで、Promiseの結果を受け取る関数を指定します。thenは1つのPromiseに対して1回しか実行されません。

```javascript
// thenの使い方
new Promise((resolve, reject) => {
  // 非同期処理
  setTimeout(() => {
    resolve('非同期処理の結果');
  }, 1000);
})
.then((result) => {
  console.log(result);
  return 'thenの結果';
})
.then((result) => {
  console.log(result);
});
```

### async/awaitの基礎

async/awaitはJavaScriptが提供する非同期処理の機能の一つです。async/awaitはPromiseのメソッドであるasyncと、awaitというキーワードを使ってPromiseの結果を待つことができます。

```javascript
// async/awaitの基礎
async function sample() {
  // 非同期処理
  const result = await new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('非同期処理の結果');
    }, 1000);
  });
  console.log(result);
}
sample();
```

### async/awaitの使い方

async/awaitはPromiseのthenメソッドと似ていますが、Promiseのthenメソッドの場合、関数を1回しか実行されない制約がありますが、async/awaitの場合は関数を複数回実行できます。

```javascript
// async/awaitの使い方
async function sample() {
  // 非同期処理
  const result1 = await new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('非同期処理の結果1');
    }, 1000);
  });
  console.log(result1);
  const result2 = await new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('非同期処理の結果2');
    }, 1000);
  });
  console.log(result2);
}
sample();
```

### エラーハンドリングの基本

Promiseやasync/awaitの場合、エラーはcatchメソッドでキャッチできます。

```javascript
// エラーハンドリングの基本
new Promise((resolve, reject) => {
  // 非同期処理
  setTimeout(() => {
    reject('エラー');
  }, 1000);
})
.catch((error) => {
  console.error(error);
});
```

### async/awaitのエラーハンドリング

async/awaitの場合、try-catch文を使ってエラーをキャッチできます。

```javascript
// async/awaitのエラーハンドリング
async function sample() {
  try {
    // 非同期処理
    await new Promise((resolve, reject) => {
      setTimeout(() => {
        reject('エラー');
      }, 1000);
    });
  } catch (error) {
    console.error(error);
  }
}
sample();
```

### まとめ

この記事では、Promiseとasync/awaitの入門から、エラーハンドリングまで、具体的な例を使って解説しました。非同期処理は、コールバック地獄から脱却して、より読みやすく扱いやすいコードを実現できます。
