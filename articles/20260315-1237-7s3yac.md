# Tailwind CSS 実践入門：クラス名の考え方からレスポンシブ、カスタム設定まで

駆け出しエンジニアから中級者向けに、Tailwind CSS を実際のプロジェクトで使いこなすための核心を解説します。この記事では、クラス名の構造を理解し、レスポンシブデザインを効率的に書き、自分プロジェクトに合わせてカスタマイズする方法を具体的なコード例とともに紹介します。

## 背景：なぜ Tailwind CSS が実務で選ばれるのか

従来の CSS 開発では、スタイルシートにクラス名を定義し、HTML から参照する「クラスベース」のアプローチが一般的でした。しかし、大規模なプロジェクトでは CSS ファイルが肥大化し、命名規則（BEM など）の維持コストや、スタイルの競合（カスケード）が課題となります。

Tailwind CSS は「ユーティリティファースト」の Framework として、あらかじめ膨大な単機能クラス（例: `p-4`, `text-lg`, `bg-blue-500`）を提供します。HTML に直接クラスを記述することで、CSS ファイルをほとんど書く必要がなくなり、以下のメリットがあります。

- **デザインの一貫性**: 設計システム（スペーシング、色、タイポグラフィ）がクラス名にエンコードされている。
- **開発スピード**: スタイルをHTML上で完結させ、行き来する時間を削減。
- **保守性**: クラス名が意味するスタイルが明確で、削除や変更が安全。

ただし、クラス名が長くなりがちで、最初は「何を選べばいいか」が分かりにくいという声も。そこで、体系的にクラス名の考え方を押さえましょう。

## 本文：実務で使う3つの核心技術

### 1. 基本クラス名の構造と選び方

Tailwind のクラス名は、ほぼ以下のパターンで構成されます。

```
{修飾子}:{プロパティ}-{値}
```

代表例:
- `p-4`: `padding` の `4`（デフォルトでは 1rem = 16px 相当）
- `bg-blue-500`: `background-color` で `blue` カラーの `500` トーン
- `text-lg`: `font-size` の `lg`（large）
- `hover:bg-blue-700`: `hover` 状態で背景色が `blue-700` に

**実務での選び方のコツ:**

1. **スペーシング（margin/padding）**  
   デフォルトでは `0.25rem` 刻み（4px）で `p-{1}〜p-{64}` などが定義されています。`p-4`（16px）を基準に、`p-2`（8px）や `p-6`（24px）を組み合わせるのが一般的です。

2. **カラーパレット**  
   `bg-{色}-{トーン}` で指定。トーンは `50`（薄い）〜 `900`（濃い）まで。  
   例: `bg-gray-100`（薄いグレー）, `bg-red-600`（中程度の赤）  
   状態別: `hover:`, `focus:` などと組み合わせられます。

3. **タイポグラフィ**  
   `text-{サイズ}` で `xs`, `sm`, `base`, `lg`, `xl`, `2xl`... と定義。行間は `leading-{tight/normal/loose}`。

**コード例: カードコンポーネントのスタイル付け**

```html
<div class="max-w-sm rounded-lg bg-white shadow-lg p-6 border border-gray-200">
  <h2 class="text-xl font-bold text-gray-900 mb-2">タイトル</h2>
  <p class="text-gray-600 mb-4">本文がここに入ります。テキストの色は `text-gray-600` で少し薄く。</p>
  <a href="#" class="inline-block bg-blue-500 hover:bg-blue-700 text-white font-semibold py-2 px-4 rounded">
    ボタン
  </a>
</div>
```

各クラスが単一の役割（パディング、タイポグラフィ、色）を持ち、組み合わせている点が分かります。

### 2. レスポンシブデザインの書き方

Tailwind では、**モバイルファースト**のブレークポイントがデフォルトで用意されています。接頭辞として画面サイズを指定し、そのサイズ以上で適用されるスタイルを定義できます。

主な接頭辞:
- `sm:`（640px以上）
- `md:`（768px以上）
- `lg:`（1024px以上）
- `xl:`（1280px以上）
- `2xl:`（1536px以上）

**書き方のルール:**
- モバイル（デフォルト）用のスタイルを最初に書く。
-  larger な画面用のスタイルは `md:` などの接頭辞を付けて後ろに書く。

**コード例: グリッドレイアウトのレスポンシブ対応**

```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <!-- ここにカードやアイテムを繰り返し -->
  <div class="bg-gray-100 p-4">アイテム1</div>
  <div class="bg-gray-100 p-4">アイテム2</div>
  <div class="bg-gray-100 p-4">アイテム3</div>
</div>
```

- モバイル: `grid-cols-1`（1列）
- タブット（md以上）: `md:grid-cols-2`（2列）
- デスクトップ（lg以上）: `lg:grid-cols-3`（3列）

これにより、1つのHTMLで全画面対応が可能です。

### 3. カスタム設定でプロジェクトに合わせる

デフォルトの設定（`tailwind.config.js`）は、多くのプロジェクトで十分ですが、独自の色、フォント、スペーシング値を追加したい場合があります。

**設定ファイルの編集例:**

```javascript
// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{html,js,jsx,ts,tsx}", // クラス名をスキャンするファイルパス
  ],
  theme: {
    extend: {
      // 独自の色を追加
      colors: {
        brand: {
          50: '#eff6ff',
          100: '#dbeafe',
          500: '#3b82f6', // メインカラー
          600: '#2563eb',
        }
      },
      // 独自のフォントファミリー
      fontFamily: {
        sans: ['"Helvetica Neue"', 'Arial', 'sans-serif'],
      },
      // スペーシングの追加（例: 80px）
      spacing: {
        '80': '20rem', // 320px
      }
    },
  },
  plugins: [],
}
```

この設定後、クラス名で `bg-brand-500` や `p-80` が使えるようになります。`extend` ではなく、直接 `theme` を上書きするとデフォルト設定が消えるので注意が必要です。

**実務での応用:**

- ブランドカラーを `colors` に定義し、一貫して使用。
- デザインシステムのスペーシング値を反映（例: 8px 単位ではなく 4px 単位にしたい場合）。
- フォントをプロジェクト用に追加（Webフォントの読み込みも忘れずに）。

## まとめ

Tailwind CSS は、クラス名の構造を理解し、レスポンシブ接頭詞をモバイルファーストで組み合わせ、必要に応じて `tailwind.config.js` でカスタマイズすることで、実務で高い生産性を発揮します。最初はクラス名の多さに圧倒されるかもしれませんが、実際に手を動かして「このスタイルはどのクラスか？」を繰り返すうちに、自然に使えるようになります。まずは本記事で紹介した基本パターンをプロジェクトの一部で試し、徐々にカスタム設定へと広げていきましょう。
