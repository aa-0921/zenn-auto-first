## コールバック地獄の背景と問題点

### 1.1 XMLHTTPRequest の時代

従来の Ajax 開発では、XMLHttpRequest を使用し、`onload` コールバックをネストさせることが多かったです。ファイルの読み込み、JSON パース、UI 更新など、一連の処理を順番に記述すると、以下のようなコードになります。

```js
const xhr = new XMLHttpRequest();
xhr.open('GET', '/api/data');
xhr.onload = () => {
  const data = JSON.parse(xhr.responseText);
  const xhr2 = new XMLHttpRequest();
  xhr2.open('POST', '/api/save');
  xhr2.send(JSON.stringify({ value: data.value }));
};
xhr.send();
```

コールバックのネストはすぐに深くなり、読みにくくなります。これが「コールバック地獄」と呼ばれます。

### 1.2 Promise 登場の理由

Promise を導入することで、複数の非同期操作を平坦化できます。Promise は「将来の値」を表し、`then` を連鎖させて処理を並べることでネストを解消します。ライブラリ（fetch API, axios など）はデフォルトで Promise を返すため、現代の JavaScript 開発では必須の概念です。

## Promise の基本と実践的な使い方

### 2.1 Promise の作成とチェーン

```js
// シンプルな Promise の作成
function fetchData(url) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', url);
    xhr.onload = () => {
      if (xhr.status === 200) {
        resolve(JSON.parse(xhr.responseText));
      } else {
        reject(new Error(`HTTP ${xhr.status}`));
      }
    };
    xhr.send();
  });
}

// Promise をチェーンで繋ぐ例
fetchData('/api/data')
  .then(data => {
    console.log('データ取得成功:', data);
    return fetchData('/api/save', { method: 'POST', body: JSON.stringify(data) });
  })
  .then(() => console.log('保存完了'))
  .catch(err => console.error('エラー発生:', err.message));
```

`then` は前の非同期操作が完了するまで待機し、結果を次のハンドラに渡します。これにより、ネスト構造が解消され、読みやすい流れになります。

### 2.2 Promise.all と並行処理

複数の並行リクエストを一度に処理したい場合、`Promise.all` を使用します。

```js
const urls = ['/api/a', '/api/b', '/api/c'];
const requests = urls.map(url => fetchData(url));

Promise.all(requests)
  .then(results => {
    console.log('すべてのデータ:', results);
  })
  .catch(err => console.error('いずれかのリクエストでエラー:', err));
```

並行処理により、レスポンスを順番に待つ必要がなくなり、UI がブロックされる時間を短縮できます。

## async/await への移行

### 3.1 async/await の構文

`async` で定義された関数は自動的に Promise を返すため、`await` を用いて非同期操作を同期コードのように書けます。

```js
async function processData() {
  try {
    const data = await fetchData('/api/data');
    console.log('取得したデータ:', data);

    const response = await fetchData('/api/save', {
      method: 'POST',
      body: JSON.stringify(data)
    });
    console.log('保存成功');
    return response;
  } catch (err) {
    console.error('処理中にエラーが発生:', err);
    throw err; // 呼び出し元でエラーを伝播させる
  }
}
```

`async/await` は Promise チェーンをより自然な制御フローに変換し、複雑なネストを解消します。

### 3.2 then と async/await の使い分け

| 状況 | 推奨パターン |
|------|----------------|
| **ライブラリ関数が Promise を返し、簡単なエラーハンドリングのみが必要な場合** | `promise.then(...).catch(...)` |
| **複数の非同期処理が絡み合い、読みやすさを重視する場合** | `async function` + `await` |
| **並行処理とタイムアウト制御が必要な場合** | `Promise.all` + `Promise.race`（または `Promise.withTimeout`） |
| **汎用的なヘルパーユーティリティ（例: API 呼び出しラッパー）** | `then/catch`（API では Promise スタイルを維持） |

基本的に、`async/await` はより複雑なフローで可読性を向上させ、単一の Promise チェーンでは `then/catch` を維持します。

## エラーハンドリングのベストプラクティス

### 4.1 根本原因を把握する

`catch` ブロックでは、エラーをそのまま再throwするか、追加の文脈を添えて再throwすることで、スタックトレースを保持できます。

```js
function fetchWithRetry(url, retries = 3) {
  return new Promise((resolve, reject) => {
    const attempt = (n) => {
      fetchData(url)
        .then(resolve)
        .catch(err => {
          if (n > 1) {
            setTimeout(() => attempt(n - 1), 1000);
          } else {
            reject(new Error(`リトライ失敗: ${err.message}`));
          }
        });
    };
    attempt(retries);
  });
}
```

### 4.2 then と catch の組み合わせ

`then`/`catch` のチェーンは `finally` と組み合わせることで、クリーンアップ処理を追加できます。

```js
fetchData('/api/expensive')
  .then(data => console.log(data))
  .catch(err => console.error('エラー:', err))
  .finally(() => console.log('リクエスト完了'));
```

### 4.3 async/await でのエラーハンドリング

`async` 関数は `try/catch` を直接サポートするため、エラー発生時に処理を graceful に停止させ、開発者にとってわかりやすいエラーメッセージを表示できます。

```js
async function safeOperation() {
  try {
    const result = await fetchData('/api/secure');
    return result;
  } catch (err) {
    // UI に通知したり、ロギングしたりする
    console.warn('操作が失敗しました。デフォルト値を返します。');
    return null;
  }
}
```

## まとめ

- コールバック地獄は、XMLHttpRequest によるネストされたコールバックから始まりました。
- Promise は非同期操作を平坦化し、`then/catch` または `async/await` で処理できます。
- `Promise.all` を使用することで、並行リクエストを安全に処理できます。
- `async/await` は複雑なフローで可読性を向上させますが、単一の Promise チェーンはシンプルなエラーハンドリングに適しています。
- エラー処理では、根本原因を把握し、`catch`/`finally` または `try/catch` と組み合わせ、再throw を行うことでデバッグを容易にします。

これらのテクニックを習得することで、JavaScript の非同期処理をより自信を持って管理でき、堅牢で保守性の高いコードを書くことができます。
