## なぜTypeScriptの型を学ぶ必要があるのか

フロントエンド開発をしていると、JavaScriptで書くと「なんかエラー出たけど何でやろうね…」という状況によく直面します。ある日、同同僚が書いたコードを触ることになったらこんなシーンが待っているかもしれません。

```javascript
// JavaScriptでは何が悪いのかわからない
function calculateTotal(items) {
  let total = 0;
  for (let item of items) {
    total += item.price * item.quantity; // これ、priceとquantityどっちも数値やんけ！
  }
  return total;
}

// 呼び出し
calculateTotal([{price: "100", quantity: 2}]); // 実行すると"1002"になっちゃう！
```

TypeScriptならこんなトラブルを防げます。型を指定することで、「priceは文字列だよ」「quantityは数値だよ」とコンパイラーに教えてあげれば、書き間違いをコンパイル時にキャッチできちゃうんです。

## 基本編：型の定義方法をマスターしよう

### 基本型の定義

TypeScriptで最もシンプルな型の指定方法は、変数名の後にコロンと型名を書くことです。

```typescript
// 基本型の定義
let userName: string = "田中太郎";
let age: number = 25;
let isActive: boolean = true;

// 配列の型定義
let scores: number[] = [85, 92, 78];
let names: Array<string> = ["佐藤", "鈴木", "高橋"];

// オブジェクトの型定義
let user: { name: string; age: number } = {
  name: "田中太郎",
  age: 25
};
```

### インターフェースを使った構造の定義

より複雑なオブジェクトの型を定義するときは、インターフェースを活用しましょう。

```typescript
interface User {
  id: number;
  name: string;
  email?: string; // ?をつけると任意属性
  role: "admin" | "user"; // リテラル型
}

function getUserInfo(user: User): string {
  return `${user.name} (${user.role})`;
}

const adminUser: User = {
  id: 1,
  name: "管理者",
  role: "admin"
  // emailは任意なので省略可
};
```

### 関数の型定義

関数の引数と戻り値に型を指定するのも基本です。

```typescript
// 2点間の距離を計算する関数
function distance(x1: number, y1: number, x2: number, y2: number): number {
  return Math.sqrt(Math.pow(x2 - x1, 2) + Math.pow(y2 - y1, 2));
}

// 矢印関数版
const multiply = (a: number, b: number): number => a * b;

// 型別名を使った関数型
type Calculator = (a: number, b: number) => number;
const add: Calculator = (a, b) => a + b;
const subtract: Calculator = (a, b) => a - b;
```

## 忙しくて「any」を使っちゃうんですが…

### anyの問題点

多くのTypeScript初心者がつまずくのが、「型わからんからanyでいいや」という選択です。しかしanyを使うと、結局TypeScriptの恩恵を受けられなくなっちゃいます。

```typescript
// これはTypeScriptとして機能しません！
function processData(data: any) {
  return data.map(item => item.name); // どのプロパティがあるかコンパイラーがわからない
}

// 実行時には正しく動くけど、開発時の補完が効かない
const result = processData([{name: "田中"}, {name: "鈴木"}]);
```

### unknownへの移行：型を持つany

「any」の代わりに「unknown」を使いましょう。unknownは「型が不明である」という意味ですが、anyと違って型のブロードキャストが防がれます。

```typescript
// unknownを使った安全な処理
function processSafeData(data: unknown) {
  // 最初は型を絞り込まないといけない
  if (!Array.isArray(data)) {
    throw new Error("データが配列ではありません");
  }
  
  return data.map((item: unknown) => {
    // 各要素の型を確認してからアクセス
    if (typeof item === "object" && item !== null && "name" in item) {
      return (item as { name: string }).name;
    }
    return "不明なユーザー";
  });
}
```

### 型ガードを活用した型絞り込み

型ガードを使うことで、unknownやanyから安全に型を絞り込むことができます。

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
}

// 型ガード関数
function isProduct(data: unknown): data is Product {
  return (
    typeof data === "object" &&
    data !== null &&
    "id" in data &&
    typeof (data as Partial<Product>).id === "number" &&
    "name" in data &&
    typeof (data as Partial<Product>).name === "string"
  );
}

function displayProducts(rawData: unknown[]) {
  const validProducts: Product[] = [];
  
  for (const item of rawData) {
    if (isProduct(item)) {
      validProducts.push(item); // ここでitemはProduct型として認識される
      console.log(`${item.name}: ${item.price}円`);
    }
  }
  
  return validProducts;
}

// 使用例
const apiResponse = [
  { id: 1, name: "ノート", price: 200 },
  { id: "invalid", name: "ペン" }, // 不正なデータ
  { id: 2, name: "消しゴム", price: 150 }
];

displayProducts(apiResponse);
```

## 型アサーションと型の強制

### as構文を正しく使う

開発を進めていくと、「とりあえず」と思う時に型アサーションを使いますが、使いすぎには注意が必要です。

```typescript
// 間違った使い方
const input = document.getElementById("search-input");
const value = input.value; // エラー：inputはHTMLElement | nullになる

// 正しい方法
const input = document.getElementById("search-input") as HTMLInputElement;
const value = input.value; // OK、しかしnullの可能性がある

// より安全な方法
const maybeInput = document.getElementById("search-input");
if (maybeInput && maybeInput instanceof HTMLInputElement) {
  const value = maybeInput.value;
}

// 非nullアサーション（最終手段）
const input = document.getElementById("search-input")!;
const value = input.value; // nullにならないと保証できる場合のみ使う
```

### ジェネリクスを使った型の活用

型を引数として受け取ることで、より柔軟な型定義が可能になります。

```typescript
// 汎用的なページネーション型
interface PaginatedResponse<T> {
  data: T[];
  total: number;
  page: number;
  pageSize: number;
}

// User型とProduct型で同じ構造を再利用
interface User {
  id: number;
  name: string;
}

interface Product {
  id: number;
  title: string;
  price: number;
}

// 同じ関数で異なる型を扱える
async function fetchPaginated<T>(url: string): Promise<PaginatedResponse<T>> {
  const response = await fetch(url);
  return response.json();
}

// 使用例
const users = await fetchPaginated<User>("/api/users");
const products = await fetchPaginated<Product>("/api/products");

// users.data[0].name // OK
// products.data[0].title // OK
// products.data[0].name // コンパイルエラー
```

## 実務で役立つ型のパターン

### 条件型での型の振り分け

APIレスポンスに応じて型を動的に変更する方法です。

```typescript
// APIレスポンスの共用体型
type ApiResponse<T> = 
  | { success: true; data: T; error?: never }
  | { success: false; data?: never; error: string };

// 成功時のみデータを取り出すヘルパー関数
function unwrapSuccess<T>(response: ApiResponse<T>): T {
  if (response.success) {
    return response.data;
  }
  throw new Error(response.error);
}

// 使用例
const userResponse: ApiResponse<User> = await fetchUser(1);
const user = unwrapSuccess(userResponse); // userはUser型として推論される
```

### PartialとRequiredでの型修飾

送信フォームの検証などでよく使うパターンです。

```typescript
interface UserProfile {
  name: string;
  email: string;
  age: number;
  bio: string;
}

// 部分的に必須にする（フォーム送信時に一部だけ変更する場合）
type PartialProfile = Partial<UserProfile>;

// 部分的に任意にする（一部だけ更新するAPI）
type OptionalProfile = Partial<Required<UserProfile>>;

// 使用例
function updateProfile(changes: PartialProfile) {
  // nameだけでもOK、emailだけでもOK
  console.log("更新内容:", changes);
}

updateProfile({ name: "新しい名前" }); // OK
```

## まとめ：型を正しく使いこなすための習慣

TypeScriptの型システムを正しく使いこなすため、以下の習慣をつけましょう：

1. **anyは最後の手段**。unknownや型ガードを使いましょう。
2. **インターフェースと型エイリアスを使い分ける**。インターフェースはオブジェクト構造に、型エイリアスはユニオン型などに。
3. **型ガード関数を積極的に活用する**。再利用可能な型チェックロジックを関数としてまとめましょう。
4. **ジェネリクスを使って汎化する**。似たような構造を何度も書かずに済むようにします。

TypeScript初心者のうちは、型推論に任せるのも良しませんが、怪しいところは明示的に型を書くようにしましょう。すると、いつの間にか自然に型システムの考え方が身についているでしょう。

最後に、覚えておきたいのは「TypeScriptの型は、間違いを防ぐ道具ではなく、コードの意図を表現する言語」ということです。正しく使えば、チームの共通言語のように機能します。
