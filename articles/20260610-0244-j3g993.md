## 2. REST API とは
REST（Representational State Transfer）は、HTTP プロトコルを利用した軽量なアーキテクチャです。  
- **リソース**：URI で表される各種データ（例：`/users`, `/posts`）  
- **表現**：JSON や XML などでデータを送受信  
- **状態遷移**：クライアントが行う操作（GET、POST など）によってサーバー側の状態が変わる  

このように「名前」と「操作」を分離することで、API の設計が直感的になります。

## 3. HTTP メソッドとステータスコード
| メソッド | 主な目的 | 代表的なステータスコード |
|----------|----------|--------------------------|
| GET      | リソース取得 | 200 OK、404 Not Found |
| POST     | 新規作成 | 201 Created、400 Bad Request |
| PUT/PATCH| 更新 | 200 OK、400 Bad Request |
| DELETE   | 削除 | 204 No Content、404 Not Found |

- **200**：正常にリクエストが処理された  
- **201**：新しいリソースが作成された  
- **204**：正常に処理されたが本文は返さない  
- **400**：クライアントのリクエストが不正  
- **404**：指定リソースが存在しない  
- **500**：サーバー側のエラー  

これらを意識して実装すれば、エラーハンドリングがしやすくなります。

## 4. データ形式：JSON
REST API で最も一般的に使われるデータ形式は **JSON** です。  
- **リクエストボディ**：`Content-Type: application/json`  
- **レスポンスボディ**：通常は JSON 配列やオブジェクト  

```json
{
  "id": 1,
  "name": "Taro",
  "email": "taro@example.com"
}
```

サーバー側は受け取った JSON をパースし、適切な処理を行った後、同様の形式で返します。

## 5. フロントエンドからの呼び出し

### 5.1 fetch API の例
```js
fetch('https://api.example.com/users', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
  },
})
  .then(res => {
    if (!res.ok) throw new Error('Network response was not ok');
    return res.json();               // レスポンスを JSON にパース
  })
  .then(data => console.log(data))   // 取得したデータをコンソールに出力
  .catch(err => console.error('Fetch error:', err));
```

### 5.2 axios の例
```jsaxios.get('https://api.example.com/users', {
  headers: { 'Content-Type': 'application/json' }
})
  .then(res => {
    console.log(res.data);           // データ部分だけを取得
  })
  .catch(err => {
    console.error('Axios error:', err);
  });
```

- **fetch** はブラウザ標準の API で、追加ライブラリが不要です。  
- **axios** は Promise をベースにしたシンプルなインターフェースと、エラーハンドリングがしやすい点で人気があります。

## 6. まとめ
- REST API は HTTP メソッドとステータスコードで「何の操作をしたか」を明示します。  
- データ交換は JSON が主流で、リクエスト・レスポンス共に `Content-Type: application/json` を指定します。  - フロントエンドでは `fetch` や `axios` を使って簡単に API 呼び出しが可能です。  
- ステータスコードを適切に処理し、エラー時のユーザー体験を向上させることが、実務で求められるスキルです。  

これらの基礎を身につければ、実務で必要な API 連携の実装がスムーズに進むはずです。
