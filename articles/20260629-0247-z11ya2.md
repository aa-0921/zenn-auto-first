## 背景・前提
### Tailwind CSS の特徴
- **Utility‑First**: 単一の責任を持つクラスを組み合わせてスタイルを作る。  
- **高速なプロトタイピング**: 何度もクラスを書き直す必要がなく、デザインの変更が即座に反映。  
- **構成ファイルでカスタマイズ**: `tailwind.config.js` で色・フォント・ブレークポイントを簡単に拡張。  

### 前提条件
- Node.js と npm/yarn がインストール済み。  
- 既存のプロジェクト（React/Vue/Next.js など）に導入する想定。  

## 本文
### 1. Tailwind のインストールとセットアップ
```bash
# npm
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```
`tailwind.config.js` が生成され、`postcss.config.js` も自動生成されます。  
`src/index.css`（または任意の CSS ファイル）に以下を追加:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```
ビルドプロセス（例えば `vite` なら `vite.config.js`）で PostCSS を読み込むことで、Tailwind が機能します。

### 2. クラス名の考え方: 「何を表すか」で書く
Tailwind のクラスは「プロパティ→値」の形で表記されます。  
例: `text-center`（テキスト中央寄せ）や `bg-indigo-600`（背景色）。  
ポイントは以下です。

| 意味 | クラス名 | 説明 |
|------|----------|------|
| 幅 | `w-1/2` | 50% 幅 |
| マージン | `mt-4` | 上マージン 1rem |
| パディング | `p-2` | 0.5rem パディング |
| フォント | `font-semibold` | セミボールド |

#### 例: カードコンポーネント
```html
<div class="max-w-sm rounded overflow-hidden shadow-lg">
  <img class="w-full" src="image.jpg" alt="Card image">
  <div class="px-6 py-4">
    <div class="font-bold text-xl mb-2">Card Title</div>
    <p class="text-gray-700 text-base">
      Some description text goes here.
    </p>
  </div>
  <div class="px-6 pt-4 pb-2">
    <span class="inline-block bg-gray-200 rounded-full px-3 py-1 text-sm font-semibold text-gray-700 mr-2">#tag1</span>
    <span class="inline-block bg-gray-200 rounded-full px-3 py-1 text-sm font-semibold text-gray-700 mr-2">#tag2</span>
  </div>
</div>
```
クラス名をそのまま「デザインの意味」に置き換えることで、コードを読むだけでレイアウトが分かります。

### 3. レスポンシブデザインの書き方
Tailwind ではブレークポイントをプレフィックスで指定します。デフォルトは `sm`, `md`, `lg`, `xl`, `2xl` です。

```html
<div class="bg-white sm:bg-indigo-100 md:bg-indigo-200 lg:bg-indigo-300">
  <p class="text-base sm:text-sm md:text-lg lg:text-2xl">
    レスポンシブテキスト
  </p>
</div>
```
- `sm:` は `min-width: 640px` 以上で適用。  
- `md:` は `min-width: 768px` 以上で適用。  
- 同様に `lg:` (`1024px`)、`xl:` (`1280px`)、`2xl:` (`1536px`)。

#### 例: ナビゲーションバー
```html
<nav class="flex flex-col md:flex-row items-center p-4 bg-gray-800 text-white">
  <a href="#" class="mr-4 md:mr-8 mb-2 md:mb-0 hover:text-indigo-300">Home</a>
  <a href="#" class="mr-4 md:mr-8 mb-2 md:mb-0 hover:text-indigo-300">About</a>
  <a href="#" class="mr-4 md:mr-8 mb-2 md:mb-0 hover:text-indigo-300">Contact</a>
</nav>
```
- スマホでは縦並び (`flex-col`)、PC では横並び (`md:flex-row`) に変わります。

### 4. カスタム設定の入口: `tailwind.config.js`
プロジェクト独自の色やフォント、ブレークポイントを追加する方法です。

#### 4‑1. カスタムカラー
```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: '#1D4ED8', // 例: 独自の青色
        secondary: '#F59E0B',
      },
    },
  },
}
```
使い方: `bg-primary`, `text-secondary` で即座に利用可能。

#### 4‑2. カスタムフォント
```js
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        sans: ['Inter', 'ui-sans-serif', 'system-ui'],
        mono: ['Fira Code', 'monospace'],
      },
    },
  },
}
```
使用例: `font-sans`, `font-mono`。

#### 4‑3. ブレークポイントの追加
```js
module.exports = {
  theme: {
    screens: {
      '3xl': '1600px', // 追加ブレークポイント
    },
  },
}
```
使用例: `3xl:text-3xl`。

### 5. 実務でよく使うユーティリティまとめ
| カテゴリ | 主なクラス | 使いどころ |
|----------|------------|------------|
| レイアウト | `flex`, `grid`, `gap-4`, `p-4`, `m-4` | コンテナ、スペース |
| テキスト | `text-center`, `font-bold`, `text-sm`, `line-clamp-3` | 文字整列・サイズ・省略 |
| 背景 | `bg-white`, `bg-primary`, `bg-opacity-50` | 背景色・透明度 |
| ボーダー | `border`, `border-t`, `rounded-lg`, `shadow-lg` | ボーダー・角丸・影 |
| アニメーション | `transition`, `duration-300`, `hover:scale-105` | クリック時のエフェクト |
| レスポンシブ | `sm:`, `md:`, `lg:`, `xl:` | デバイス別デザイン |

## まとめ
Tailwind CSS は「クラス名＝デザインの意味」を直感的に書けるため、実務でのプロトタイピングや機能追加が高速化します。  
- **インストール** → **クラス名の意図** → **レスポンシブ書き方** → **カスタム設定** の流れで進めると、迷わず導入できます。  
- さらに `@apply` を使って再利用可能なコンポーネントを作ると、コードの可読性が向上します。  
- 本記事のコード例をベースに、実際の UI を作る際に「何を表すか」を意識してクラスを組み合わせる習慣をつけてください。  

Tailwind を使いこなせば、デザインと実装がスムーズに連携し、開発速度と保守性の両立が可能になります。
