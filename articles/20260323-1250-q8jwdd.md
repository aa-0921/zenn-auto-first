# Gitの基本と現場での活用

現代の開発では、Gitがほぼ必須のツールとなっており、チームで作業をする際の効率化に大きく貢献しています。この記事では、Gitのコマンドの基本的な使い方と、現場でよく使うコマンドについて、詳しく解説します。

## Gitの背景と考えられる前提

Gitは、コードの管理やバージョン管理を効率化するためのシステムです。  
開発者が複数のメンバで協力する際、ファイルの変更履歴やバージョンの維持が重要になります。  
Gitによって、誰がいつ何を変更したかを明確に追跡できるため、問題が発生した際の対応が速くなります。

この記事では、特に「clone」「pull」「branch」「commit」「push」の流れを中心に、実際の作業に役立つポイントを紹介します。

---

### Gitの基本コマンドの流れ

Gitは、ファイルの管理やチームワークを支えるためのコマンドが豊富です。  
まずは、プロジェクトの初期設定やコードの管理方法を理解しましょう。

#### 1. `clone`コマンド

```
git clone <リポジトリURL>
```

**説明:**  
Gitを使用してリポジトリをダウンロードするコマンドです。  
チームメンバー全員が同じリポジトリにアクセスできるよう、あれば便利です。

#### 2. `pull`コマンド

```
git pull origin <ブランチ名>
```

**説明:**  
リモートリポジトリから最新の変更を取得します。  
あまり使用されないコマンドですが、最新のコードを常に利用したい開発者にとって命中点です。

#### 3. `branch`コマンド

```
git branch <ブランチ名>
git checkout <ブランチ名>
```

**説明:**  
新しいブランチを作成したり、既存のブランチを切り替えるためのコマンドです。  
開発進行中は、複数の機能を並行して開発できるため、非常に重要です。

#### 4. `commit`コマンド

```
git commit -m "変更内容"
```

**説明:**  
コードの変更を記録するコマンドです。  
「commit」の前に「-m」はメッセージを指定する場所に使います。  
変更内容を明確に説明することで、チーム内での理解が深まります。

#### 5. `push`コマンド

```
git push origin <ブランチ名>
```

**説明:**  
ローカルに作成した変更をリモートリポジトリに送信します。  
チームやクライアントとの共有には必須です。

---

### コンフリクト時の対処法

Gitを使用する際は、しばしば複数の開発者が同時に同じファイルを編集します。  
これに伴い、**コンフリクト**が発生することがあります。

#### コンフリクトの対処方法

1. **変更内容の確認**
   - コマンドを実行する前に、変更された部分を確認します。
   - `git diff`を使用して、どの部分が変更されたかを確認しましょう。

2. **手動での修正**
   - 発生した変更を手動で修正します。  
   例: `git add <ファイル>` で変更を蓄積してから `git commit`。

3. **マージの選択**
   - Gitにはマージ機能がありますが、**マージコマンドの結果を確認し、必要に応じて編集**しましょう。
   - 複雑なマージの場合、`git merge` の前にマージツールを使うことも効果的です。

---

### 実務での活用例

開発現場では、単にこれらのコマンドを使うだけでなく、**ワークフローに適応させる**ことが重要です。

- **継続的インテグレーション（CI）**: 自動テストやコード品質チェックを組み込む。
- **コードレビュー**: `pull`コマンドで最新情報を受け取り、チームでレビューする。
- **ブランチ管理**: 開発中は機能ブランチを使い、安定化時はメインブランチにマージ。

---

## まとめ

Gitは、コードの管理やチーム協力を支える強力なツールです。  
`clone`, `pull`, `branch`, `commit`, `push`といった基本コマンドをしっかり理解し、実際の作業に応用することで、開発の効率は大幅に向上します。

特にコンフリクトを適切に対処する力が、経験豊富なエンジニアの違いであり、現代のソフト開発では不可欠です。  
Gitを習得することは、あなたの技術レベルを大きく引き上げる一歩です。

小さな単位でこまめにコミットする習慣を持つと、変更履歴の追跡とレビューがさらに行いやすくなります。
