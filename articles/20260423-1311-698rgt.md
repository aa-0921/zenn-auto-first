## SELECTとWHEREで効率的なフィルタリングを実現する

### SELECTの基本構文と実用的な使い方
SELECT文は、データベースから必要な列を指定して取得する基本的な文法です。実務では、不要な列を取得せずにターゲットのデータだけを抽出することが重要です。例えば、ユーザー名とメールアドレスのみを取得する場合、以下のように書きます。

```sql
SELECT name, email
FROM users;
```

ただし、よくある間違いは`SELECT *`を使うことです。これはテーブル内の全ての列を取得するため、パフォーマンス低下や不必要なデータ転送を引き起こします。特に大規模なテーブルでは、列ごとの取得を明示する方が効率的です。

### WHEREで条件を絞り込むテクニック
WHERE句で条件を指定することで、不要なデータを取得せずに目的のレコードを絞り込むことができます。以下は、特定の条件を満たすレコードを検索する例です。

```sql
SELECT *
FROM orders
WHERE status = 'completed'
  AND created_at >= '2023-01-01';
```

ただし、WHEREに直接`LIKE`や`REGEXP`などの複雑な条件を書くと、インデックスの利用が難しくなります。そのため、可能であれば`BETWEEN`や`IN`を使うのがおすすめです。

```sql
-- 2023年1月～3月の注文を取得
SELECT *
FROM orders
WHERE created_at BETWEEN '2023-01-01' AND '2023-03-31';

-- 特定のユーザーIDの注文を取得
SELECT *
FROM orders
WHERE user_id IN (1, 5, 10);
```

## JOIN演算子で複数テーブルを連携させる

### INNER JOINで共通データを取得
JOIN演算子は、複数のテーブルから関連データを結合するための基本的な機能です。特に`INNER JOIN`は、両方のテーブルにマッチするレコードのみを取得します。例えば、注文とユーザー情報の結合には以下のように書きます。

```sql
SELECT orders.id, orders.total_amount, users.name
FROM orders
INNER JOIN users ON orders.user_id = users.id;
```

このクエリでは、注文テーブルとユーザーテーブルの`user_id`列でマッチするデータのみを取得します。ただし、外部キーが存在しない場合やデータが欠損している場合は、`LEFT JOIN`を使用すると実用的です。

### LEFT JOINで未マッチデータも含む
LEFT JOINは、左側のテーブル（通常は主テーブル）のすべてのレコードを取得し、右側のテーブルとマッチする場合はデータを、マッチしない場合はNULLを埋めます。例えば、注文が存在しないユーザー情報を取得する場合：

```sql
SELECT users.id, users.name, orders.total_amount
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
```

このクエリでは、注文が存在しないユーザーも結果に含まれます。この技術は、データの整合性を確認したり、未処理のケースを分析する際に有効です。

## 集計とサブクエリで複雑な分析を実現する

### GROUP BYで集計処理
GROUP BY句を使うことで、データをグループ化して集計が可能です。例えば、各ユーザーの合計注文額を計算する場合：

```sql
SELECT users.name, SUM(orders.total_amount) AS total_spent
FROM users
JOIN orders ON users.id = orders.user_id
GROUP BY users.name;
```

このクエリでは、ユーザー名ごとに注文額を合計し、結果として各ユーザーの支出額を表示します。ただし、GROUP BYで使用する列は、SELECT句に明示的に含める必要があります。

### サブクエリで複雑な条件を表現
サブクエリは、条件を複雑にする場合に役立ちます。例えば、注文金額が平均以上のユーザーを取得する場合：

```sql
SELECT name, total_amount
FROM users
WHERE id IN (
  SELECT user_id
  FROM orders
  GROUP BY user_id
  HAVING SUM(total_amount) > (
    SELECT AVG(total_amount) FROM orders
  )
);
```

このクエリでは、まず注文テーブルから各ユーザーの合計金額を計算し、その合計が平均より大きいユーザーを選択します。サブクエリは実行順序に注意が必要ですが、柔軟な条件表現が可能です。

## インデックスの活用でクエリパフォーマンスを向上させる

インデックスは、WHERE句やJOINで使用される列に対して作成することで、データの取得速度を大幅に向上させます。例えば、ユーザーIDで頻繁に検索する場合：

```sql
CREATE INDEX idx_user_id ON users(id);
```

ただし、インデックスを多すぎると書き込み性能が低下するため、適切な列に限定して作成することが重要です。また、クエリプランを`EXPLAIN`で確認し、インデックスの利用状況をチェックすることも推奨されます。

## コード例：実務で活用するクエリパターン

### 1. 注文とユーザーの連携
```sql
SELECT orders.id, users.name, orders.created_at
FROM orders
LEFT JOIN users ON orders.user_id = users.id
WHERE orders.total_amount > 1000;
```

### 2. 顧客別最高注文額の取得
```sql
SELECT customer.name, MAX(order.total_amount) AS max_order
FROM customers
JOIN orders ON customers.id = orders.customer_id
GROUP BY customer.id
HAVING MAX(order.total_amount) > 5000;
```

### 3. 過去30日の新規登録ユーザー数
```sql
SELECT COUNT(*) AS new_users
FROM users
WHERE created_at >= DATE_SUB(CURRENT_DATE, INTERVAL 30 DAY);
```

## まとめ

SQLの基本機能であるSELECT、WHERE、JOINを正しく活用することで、効率的なデータ取得が可能になります。集計やサブクエリ、インデックス活用といった実務でよく使うテクニックを理解することで、複雑なクエリを扱いやすくなります。特に、不要な列の取得やインデックスの最適化はパフォーマンス向上において不可欠です。実際のプロジェクトでこれらの技術を試してみることで、SQLの実力が大きく向上するでしょう。
