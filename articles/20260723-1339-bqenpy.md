### REST API の特徴  
- **リソースを扱う**：データ（ユーザー、投稿など）を「リソース」として扱い、URLで識別します（例：`/users/1`）。  
- **HTTPメソッドで操作を指定**：GET（取得）、POST（新規作成）、PUT/PATCH（更新）、DELETE（削除）でリソースを操作。  
- **JSON形式でデータ交換**：ブラウザやモバイルアプリが扱いやすい形式。  

こうした仕組みにより、フロントとバックが独立して開発・拡張しやすくなります。

## HTTP メソッドとステータスコード：知っておくべき基本  

### HTTP メソッドの役割  
| メソッド | 用途 | 例 |  
|----------|------|----|  
| **GET** | データの取得 | ユーザー一覧の取得：`GET /users` |  
| **POST** | 新規データの作成 | ユーザー作成：`POST /users` |  
| **PUT** | 全体を置き換える更新 | ユーザー全情報の更新：`PUT /users/1` |  
| **PATCH** | 部分的な更新 | ユーザー名のみを変更：`PATCH /users/1` |  
| **DELETE** | データの削除 | ユーザー削除：`DELETE /users/1` |  

> **補足**：ブラウザの開発者ツール（F12）で「Network」タブにアクセス履歴が表示され、メソッドやステータスコードを確認できます。  

### ステータスコードの意味  
| コード | 意味 | 例 |  
|--------|------|----|  
| **200 OK** | 成功 | データ取得完了時 |  
| **201 Created** | 新規作成成功 | `POST` でリソース作成時 |  
| **400 Bad Request** | リクエストエラー | パラメータ不足時 |  
| **401 Unauthorized** | 認証失敗 | トークンなしでアクセス |  
| **404 Not Found** | リソース不存在 | 存在しないURLにアクセス |  
| **500 Internal Error** | サーバーエラー | データベース接続失敗時 |

## JSON 通信の流れ：リクエストとレスポンスの構造  

### リクエストの例（POST メソッド）  
```http
POST /users HTTP/1.1  
Content-Type: application/json  
Authorization: Bearer <トークン>  

{
  "name": "John Doe",
  "email": "john@example.com"
}
```  

### レスポンスの例（201 Created）  
```http
HTTP/1.1 201 Created  
Content-Type: application/json  

{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com",
  "createdAt": "2023-10-01T12:00:00Z"
}
```  

### リクエストパラメータの扱い  
- **Query String**（GETメソッド）：URLにパラメータを追加（例：`/users?name=John`）。  
- **Body**（POST/PUT/PATCH）：JSON形式で本文にデータを送信。

## フロントエンドからAPIを叩く：JavaScriptでの実装例  

### 1. `fetch` を使ったシンプルなGETリクエスト  
```javascript
// ユーザー一覧の取得  
fetch('https://api.example.com/users')  
  .then(response => {  
    if (!response.ok) {  
      throw new Error('ネットワークエラー');  
    }  
    return response.json();  
  })  
  .then(data => console.log('ユーザー一覧:', data))  
  .catch(error => console.error(error));  
```  

### 2. ページネーション付きのGETリクエスト  
```javascript
// 2ページ目のユーザーを取得（1ページに10件）  
fetch(`https://api.example.com/users?page=2&limit=10`)  
  .then(response => response.json())  
  .then(data => { /* ページング処理 */ });  
```  

### 3. `POST`でデータを送信する例  
```javascript
// 新規ユーザー作成  
const user = { name: 'Alice', email: 'alice@example.com' };  

fetch('https://api.example.com/users', {  
  method: 'POST',  
  headers: {  
    'Content-Type': 'application/json',  
  },  
  body: JSON.stringify(user),  
})  
  .then(response => response.json())  
  .then(data => console.log('作成完了:', data));  
```  

### 4. `axios` を使ったキャッシュ切り替えの例  
```javascript
// axiosを使いメソッドを柔軟に扱う  
import axios from 'axios';  

axios.get('/users')  
  .then(response => console.log('ユーザー一覧:', response.data))  
  .catch(error => console.error('エラー:', error.response.data));  

// PUTメソッドで更新  
axios.put('/users/1', { name: 'Alice Smith' })  
  .then(response => console.log('更新成功:', response.data));  
```  

### 5. 認証ヘッダーの追加（Bearerトークン）  
```javascript
// ヘッダーに認証情報を追加  
fetch('https://api.example.com/private-data', {  
  headers: {  
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'  
  }  
});  
```

## まとめ：フロントからAPIを実践的に使うポイント  

1. **HTTPメソッドの役割を明確に**: GETは取得、POSTは新規作成など、一貫した使い方を。  
2. **ステータスコードでエラーを判断**：`response.ok`で簡単にチェック可能。  
3. **JSONの構造を確認**：APIドキュメントでリクエスト/レスポンスの形式を確認。  
4. **ツール選択**：簡単なプロジェクトなら`fetch`、拡張性が必要なら`axios`を採用。  

この流れを理解しておけば、フロントエンドとバックエンドの連携開発がスムーズになります。実際のプロジェクトでは、APIのドキュメント（SwaggerやPostmanなど）を参考にしながら実装してください。
