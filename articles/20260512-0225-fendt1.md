## 背景・前提

VS Code はデフォルトでも十分使えますが、適切な拡張機能と設定を追加することで作業効率が大幅に向上します。特に以下のようなシーンで役立ちます。

- コードの自動補完やシンタックスハイライトの強化
- Git 操作の簡略化
- デバッグやテストの効率化
- チームでの開発環境の統一

これらを実現するための具体的な方法を以下に示します。

## 本文

### 1. 入れておきたい拡張機能

#### 基本機能強化系

**1.1 Prettier - Code formatter**

コードフォーマッターとして最も人気のある拡張機能です。保存時に自動でコードを整形してくれます。

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true
}
```

**1.2 ESLint**

JavaScript/TypeScript のコーディング規約をチェックし、リアルタイムでエラーを表示します。

**1.3 GitLens**

Git の履歴や差分を視覚的に確認できる拡張機能。誰がいつコードを変更したかを瞬時に把握できます。

**1.4 Bracket Pair Colorizer**

対応する括弧を色分けして表示し、コードの構造を視覚的に理解できます。

#### 言語別推奨拡張機能

**2.1 Python**

Microsoft 製の公式 Python 拡張機能。デバッグ、インテリセンス、ジャンプ機能を提供します。

**2.2 Docker**

Docker コンテナの管理やイメージのビルドを VS Code から行えるようにする拡張機能。

**2.3 REST Client**

VS Code 上で API のテストを行うことができる拡張機能。curl コマンドを書かずに済みます。

#### 生産性向上系

**3.1 Auto Rename Tag**

HTML/XML の開始タグと終了タグを同時に変更してくれる拡張機能。

**3.2 Path Intellisense**

ファイルパスを自動補完してくれる拡張機能。import 文の入力が劇的に楽になります。

**3.3 TODO Highlight**

コード内の TODO や FIXME をハイライトしてくれる拡張機能。忘れずにタスクを管理できます。

### 2. キーバインドの設定

効率的なキーバインドは作業速度を大幅に向上させます。以下はおすすめのキーバインド設定です。

```json
[
  {
    "key": "ctrl+shift+p",
    "command": "editor.action.formatDocument",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+shift+t",
    "command": "workbench.action.files.newUntitledFile"
  },
  {
    "key": "ctrl+shift+n",
    "command": "workbench.action.newWindow"
  }
]
```

#### 必須キーバインド

| キー | 機能 | 説明 |
|------|------|------|
| Ctrl+P | ファイル検索 | ファイル名を入力して素早くファイルを開く |
| Ctrl+Shift+O | シンボル検索 | 関数や変数などのシンボルにジャンプ |
| Ctrl+` | ターミナル表示 | 統合ターミナルを表示/非表示 |
| F2 | リファクタリング | 変数名や関数名の一括変更 |

### 3. 設定の共有方法

チーム開発では設定を共有することで一貫性を保ちます。以下の方法があります。

#### 3.1 settings.json の共有

プロジェクトルートに `.vscode/settings.json` を作成し、チーム共通の設定を記述します。

```json
{
  "editor.tabSize": 2,
  "editor.insertSpaces": true,
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}
```

#### 3.2 拡張機能の推奨設定

`.vscode/extensions.json` に推奨拡張機能を記述します。

```json
{
  "recommendations": [
    "esbenp.prettier-vscode",
    "dbaeumer.vscode-eslint",
    "ms-python.python",
    "ms-azuretools.vscode-docker"
  ]
}
```

#### 3.3 設定の同期

VS Code の設定同期機能を使用すると、Microsoft アカウントや GitHub で設定を同期できます。

1. コマンドパレット（Ctrl+Shift+P）を開く
2. 「Settings Sync: Turn On」を実行
3. 使用するアカウントを選択

### 4. 実践的な設定例

#### 4.1 ターミナルのカスタマイズ

```json
{
  "terminal.integrated.fontSize": 14,
  "terminal.integrated.fontFamily": "Fira Code",
  "terminal.integrated.cursorBlinking": true,
  "terminal.integrated.scrollback": 10000
}
```

#### 4.2 ファイル保存時の自動フォーマット

```json
{
  "editor.formatOnSave": true,
  "editor.formatOnPaste": true,
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[python]": {
    "editor.defaultFormatter": "ms-python.python"
  }
}
```

#### 4.3 ワークスペースの設定

プロジェクトごとの設定を `.vscode/settings.json` に記述します。

```json
{
  "python.pythonPath": "./venv/bin/python",
  "python.testing.pytestEnabled": true,
  "python.testing.unittestEnabled": false
}
```

## まとめ

VS Code の拡張機能や設定を適切に活用することで、開発効率を大幅に向上させることができます。本記事で紹介した拡張機能や設定方法を参考に、自分に合った開発環境を構築してみてください。

特に以下の点に注意してください：

1. **必要最小限の拡張機能**: 多すぎる拡張機能は動作を遅くする可能性があります
2. **チームでの設定共有**: プロジェクトルートの `.vscode` フォルダで設定を管理
3. **定期的な見直し**: 使用しなくなった拡張機能はアンインストールしてパフォーマンスを維持

これらの知識を活かして、より快適なコーディングライフを送りましょう。
